# Review Grade

根据结构化题目对象、用户答案和 profile 资料进行判题。

## 输出要求

先给结果，再解释：

```markdown
## 判题结果

✅ 正确

## 正确答案

{正确答案}

## 解析

{Level 1 解析}
```

如果答错，使用 `❌ 错误`，并指出具体误区。

## 判题规则

- 优先依据题目对象中的 `correct_answer` 和 `explanation_l1`。
- 简答题按关键要点判定，不要求逐字匹配。
- 如果题目对象和 profile 资料冲突，说明不一致，并优先相信 profile 资料。
- 不要用模糊文本匹配推断正确性。归档时必须提供显式布尔值 `is_correct`。
- 第一层解析保持简洁。更宽的概念对比留到讨论阶段。

## 语气

- 不要说“你错了”。
- 答对时，用一句话确认用户抓住了关键点。
- 答错时，指出混淆的概念，并直接给出纠正。
