# Review 迭代计划：流程减噪、开局配置、学习画像基座

## Summary

本轮不做资料包自动改写，先把复习体验打顺：题后菜单只保留高频续航动作，提示/追问/难度调整前移到合适阶段；开局允许选择难度策略；summary/session 从“单次归档”升级为“科目学习画像”的输入，每次 **/review** 前读取最近记录，帮助 agent 知道用户当前薄弱点、偏好和复习建议。

默认策略：

* 题后菜单采用“只留续航项”。
* summary 优先作为学习画像，不默认修改资料包。
* 资料优化仍通过用户确认后的 **/review-fix** 进入 draft 流程。

## Key Changes

### 1. 修正题后动作菜单时机

* 调整 **review_turn_action**：
  * 保留：**next_question**、**show_card**、**show_chapter**、**summary**、**exit**。
  * 移除题后默认项：**hint**、**discuss**、**increase_difficulty**。
* 将“提示/追问”移入答题阶段：
  * **review_answer** 增加答题中动作：提交答案、请求提示、追问/澄清、退出。
  * 对选择题/判断题仍优先保持选择交互；简答题允许用户输入特殊动作或通过 TUI 菜单触发。
* 将“提高难度”移到开局配置：
  * **/review** 选择范围后增加难度选择。
  * 选项固定为：自动、S-R、S-U、M-U、M-A、C-A。
  * 自动模式继续使用当前 **selectDifficulty()**，手动模式写入 session 并进入 prompt。

### 2. 复习开局配置产品化

* **/review** 启动流程调整为：
  * 选择 profile。
  * 展示该 profile 的最近学习画像摘要。
  * 选择模式、范围、题型、难度。
  * 初始化 session 时记录 **mode**、**difficulty_policy**、**question_type_policy**、**profile_id**。
* **buildReviewStartPrompt()** 明确注入：
  * 当前难度策略。
  * 最近 summary 提炼出的薄弱点、已掌握点、建议复习方向。
  * 若无历史 summary，说明这是新会话，不要臆造用户画像。
* 章节学习模式中，**review_chapter** 后只出现“出题/下一节/跳过/退出”，题后不再重复出现低价值动作。

### 3. Summary / Session 升级为学习画像基座

* 新增学习画像文件，建议放在用户数据目录：
  * **state/learning_profiles/{subject_id}.json**
* **review_summary** 保存报告后，同步更新该科目画像：
  * 最近 session 列表。
  * 总题数、正确率。
  * 最近薄弱知识点。
  * 高频错误类型。
  * 用户偏好或遗留问题。
  * 下一次复习建议。
* **/review** 每次开始前读取该画像，并在 prompt 中注入“学习者画像”。
* 不把 summary/session 自动写回 profile 内容；只在 **/review-fix** prompt 中提示：
  * 如用户要求优化资料包，可参考 learning profile、recent summaries、wrong book 创建 revision draft。
* 保留原有 **archive/sessions/** 和 **archive/summaries/**，新增画像只是索引和提炼层，不破坏历史归档。

### 4. 待排查问题纳入本轮修复优先级

* 修复 **review_answer** 长题目省略问题：
  * 长题干使用可滚动/分页展示。
  * 代码块不能被替换成“已省略 N 行”。
  * 工具返回给 agent 的文本可压缩，但 TUI 给用户看的题目必须完整。
* 继续排查 TUI 滚动回弹：
  * 避免 render/cache 更新时重置 scroll。
  * 卡片、章节、题目三个面板使用统一滚动状态逻辑。
* 修复 **review_chapter** 展示不稳定：
  * 调用工具时必须能看到章节内容面板。
  * 找不到内容时显示明确缺失路径和建议修复方式。
* revision draft 命名问题作为低优先级修复：
  * 多次修订应基于原始 **revisionOf** 生成 **subject__draft_YYYYMMDD_vN**。

### 5. 文档更新

* 更新产品迭代路线：
  * 增加“学习画像”和“今日推荐”的基座设计。
  * 明确 summary 当前不是资料包内容，而是长期学习状态。
* 更新 DESIGN：
  * **review_turn_action** 的动作集合改为续航菜单。
  * **review_answer** 说明答题中可请求提示/追问。
  * 增加 **learning_profiles/{subject_id}.json** 状态文件说明。
* 更新待排查问题：
  * 已修复项标记状态、修复位置、验证方式。
  * 未修复项保留复现路径。

## Test Plan

* 单元测试：
  * **review_turn_action** 不再包含 **hint/discuss/increase_difficulty**。
  * **/review** prompt 包含手动难度或自动难度策略。
  * **initSession()** 正确保存 profile、mode、difficulty policy。
  * **review_summary** 会写 summary 文件并更新 learning profile。
  * 无 summary 时 **/review** prompt 不注入虚假的历史画像。
  * 有 recent summary / wrong book 时，学习画像能提取薄弱点和下一步建议。
  * 长题目 normalization 不丢失代码块内容。
* 集成检查：
  * 模式 1：卡片 → 出题 → 答题中提示/追问 → 归档 → 简化题后菜单。
  * 模式 2：考点总结 → 出题 → 归档 → 下一题/总结/退出。
  * 模式 3：章节材料 → 出题 → 归档 → 下一节或退出。
  * 生成 summary 后重新 **/review**，开局 prompt 能读取该科目的学习画像。
* 手动验收：
  * 章节学习结束后不再反复出现“提供方向/提高难度”等低频题后项。
  * 开始复习时可以直接指定难度。
  * 长代码题在 TUI 中完整可读。
  * summary 保存后，下次复习能体现用户近期薄弱点和建议方向。

## Assumptions

* 本轮不实现自动修改资料包内容。
* **/review-fix** 仍是唯一资料包修订入口，且必须创建 draft 后由用户确认启用。
* 学习画像只存用户数据，不进入可分享 profile。
* 暂不新增 **/review-today**，但本轮产出的 learning profile 要为后续今日推荐复用。
