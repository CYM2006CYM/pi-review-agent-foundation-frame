## 一、阻塞性问题（不上无法开源）

### 1.1 node_modules/ 被提交到 git

- 17,936 个第三方文件被 git 跟踪。
- 问题：仓库臃肿、安全扫描无法区分项目代码和第三方代码。
- 解决：`git rm --cached -r workspace/node_modules/` + `.gitignore` + BFG 清洗历史。

### 1.2 没有 .gitignore

- 仓库根目录没有 `.gitignore` 文件。
- 除 `node_modules/` 外，`state/*.json`、`archive/`、`__pycache__/` 等运行产物均未被忽略。
- 解决：添加 `.gitignore`，至少忽略：
  - `node_modules/`
  - `state/*.json`
  - `archive/`
  - `__pycache__/`
  - `*.pyc`

### 1.3 没有 LICENSE 文件

- `package.json` 中无 `license` 字段，仓库根目录无 `LICENSE` 文件。
- 没有许可证 = 默认保留所有权利，他人无法合法使用、修改或分发。
- 建议：MIT 许可证（最宽松，适合工具类项目），或 GPL-3.0（如希望衍生项目也保持开源）。

### 1.4 私有依赖 `@earendil-works/pi-coding-agent`

- 项目核心依赖了 `@earendil-works/pi-coding-agent`（v0.78.0），该包未在 npm 公开 registry 发布。
- 外部用户 `npm install` 会返回 404。
- 待决策方案：

| 方案 | 工作量 | 影响 |
|------|--------|------|
| A. 将 pi-agent 开源发布到 npm | 依赖 pi-agent 作者的配合 | 项目可独立运行 |
| B. 抽取核心逻辑，去掉 pi-agent 依赖 | 大（相当于重写 CLI 入口） | 彻底独立 |
| C. 先不开源，等 pi-agent 就绪 | 无 | 项目保持私有不发布 |

## 二、必须补（否则不像开源项目）

### 2.1 没有 CI/CD

- 无 `.github/workflows/` 目录，无 GitHub Actions 配置。
- PR 无法自动跑 `npm test` 和 `npm run check`。
- 解决：创建 `.github/workflows/ci.yml`，在 push/PR 时运行：

```yaml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm test
```

### 2.2 没有 CONTRIBUTING.md

- 潜在贡献者不知道如何提交 PR、代码风格、测试要求。
- 建议包含：开发环境搭建、运行测试、提交规范、PR 流程。

### 2.3 测试覆盖 < 20%

| 模块 | 行数 | 覆盖情况 |

| `review_profiles.mjs` | ~234 | 部分覆盖（CRUD、路径安全） |
| `review_engine.mjs` | ~167 | 部分覆盖（核心函数） |
| `review_question.mjs` | ~121 | 部分覆盖（问题解析） |
| `state.mjs` | ~415 | **零覆盖** |
| `terminal.mjs` | ~182 | **零覆盖** |

- `state.mjs` 和 `review_cli.mjs` 是业务逻辑最密集的模块，无测试是上线前的最大风险。

## 三、锦上添花（有了更好）

### 3.1 语言问题

- 全部文档、注释、用户交互字符串为中文。
- 国际开源：至少需要有英文 README 和英文代码注释。
- 中文开源项目可以接受中文文档，但英文大门会损失国际贡献者。

### 3.2 硬编码 C++ 默认值

`lib/review_config.mjs` 和 `lib/state.mjs` 中存在硬编码的 C++ 路径和默认值。虽然 profile 系统已抽象好（legacy-bridge 模式），但框架层面仍有残留假设。

- `review_config.mjs` 默认 `courseName: "C++ 面向对象程序设计"`
- `state.mjs` 默认 `REFERENCE` 路径指向 `../reference`

解决方向：将默认值抽取到 profile 配置中。

### 3.3 没有 linter / formatter

- 无 `.eslintrc`、无 `.prettierrc`、无 `.editorconfig`。
- 多个贡献者时代码风格难以统一。
- 建议：至少添加 `.editorconfig` 和 `eslint` 基础配置。

### 3.4 没有 CHANGELOG / CODE_OF_CONDUCT

- CHANGELOG：用户和贡献者无法了解版本间变化。
- CODE_OF_CONDUCT：社区规范缺失，不符合 GitHub 社区标准。
