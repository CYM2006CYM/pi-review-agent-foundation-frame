# pi-review-agent-foundation-frame 待排查问题

记录日期：2026-06-09

## 1. review_chapter 章节材料展示

- 现象：章节学习模式下，早期 `review_chapter` 只返回动作，TUI 中没有稳定展示章节材料。
- 状态：已修复，已纳入 `showMaterialPanel` 统一滚动材料面板。
- 复验路径：`/review` -> 章节笔记学习 -> 选择章节 -> 确认可看到章节材料并可滚动。

## 2. TUI 长内容与滚动

### 2a. 长行左侧截断

- 现象：卡片、章节材料、题目中存在长行时，终端可见区域会丢内容。
- 状态：已修复，长内容统一进入可滚动面板。
- 仍需观察：不同 Windows Terminal / PowerShell 字体宽度下的中文、代码块换行表现。

### 2b. 滚动位置回弹和进度条异常跳动

- 现象：滚动后自动回到顶部，进度条持续跳动，严重时终端崩溃。
- 状态：已修复，已增加 custom UI closed guard，避免 `done()` 后继续 requestRender。
- 仍需观察：真实 pi TUI 中连续打开卡片、章节、题目面板时是否还有重复渲染。

## 3. review_answer 长题目省略

- 现象：含代码块的长简答题被显示为“已省略 N 行”，用户无法看到完整题干。
- 状态：已修复，题干先进入可滚动题目面板，再进入选择/输入控件。
- 复验路径：构造含 50 行代码块的简答题，确认 J/K 可滚动阅读，Enter 后可继续作答。

## 4. 题后菜单冗余

- 现象：题后菜单出现“提示/追问/提高难度”等时机不合适的选项。
- 状态：已修复，`review_turn_action` 已简化为续航菜单：
  - 下一题
  - 看卡片
  - 看章节
  - 总结
  - 退出
- 说明：提示/追问放在答题过程中；难度选择前置到 `/review` 开局配置。

## 5. 学习画像阻塞启动流程

- 现象：每次选择 profile 后弹出一整屏学习画像，并要求 Enter 继续。
- 状态：已修复，改为非阻塞通知/提示，不再打断 `/review` 启动流程。
- 关联变化：学习画像现在优先从资料包私有 `_user/learning_profile.json` 读取。

## 6. revision draft 命名层层嵌套

- 现象：多次修订后 subjectId 变成 `cpp-oop__draft_20260608__draft_20260608`。
- 状态：已修复。
- 当前规则：
  - 首次修订：`{root}__draft_{YYYYMMDD}`
  - 当天再次修订：`{root}__draft_{YYYYMMDD}_v2`
  - 继续递增：`_v3`、`_v4`
- 说明：命名会沿 `revisionOf` / `revisionRoot` 回溯到原始 subjectId，并兜底剥离已有 draft 后缀。

## 7. 草稿和归档目录累积

- 现象：每次修订启用后，旧 active 会归档但目录保留，`review_profiles/` 下目录逐渐增多。
- 状态：保留但隐藏。
- 当前策略：
  - `/review` 只显示 active。
  - `/review-fix` 只显示 active 和 draft。
  - archived 不进入普通使用流。
- 后续规划：增加 `/review-doctor` 或 `review_profile_prune`，默认 dry-run，用户确认后再删除。

## 8. summary / learning profile 全局化

- 现象：旧实现把 summaries 和学习画像放在全局 `archive/`、`state/learning_profiles/`，多资料包使用时容易混杂。
- 状态：已修复。
- 当前规则：
  - 新 summary 写入 `review_profiles/{subjectId}/_user/summaries/`。
  - 新学习画像写入 `review_profiles/{subjectId}/_user/learning_profile.json`。
  - 旧全局 `state/learning_profiles/{subjectId}.json` 保留 fallback。
- 说明：`_user/` 是用户私有运行数据，不属于可分享资料包内容。

## 9. 出题 / 读取 skill 阶段主输出区进度条顶到顶部

- 现象：进入出题或 agent 读取 skill 时，进度条顶到终端最上方，滚轮无法把主对话区翻下来。
- 判断：这不是卡片/章节/题目 custom UI 内部滚动问题，而是主对话输出区在 agent 流式阶段被超长 prompt、超长工具调用行或可见题目 JSON 撑开。
- 本轮修复：
  - `injectReviewCore()` 不再把完整 `review-core/SKILL.md` 复制进 `sendUserMessage()`，改为注入短契约摘要。
  - `review_answer.renderCall()` 不再渲染完整题干，只显示题型、难度、知识点和短摘要。
  - 出题 prompt 明确要求 agent 不要在主对话区可见输出完整题目 JSON，应直接把 Question JSON 作为 `review_answer` 工具参数。
- 仍需真实 pi 复验：
  - `/review` -> 任意 profile -> 模式 1/2/3 -> 触发出题。
  - 观察 agent 读取 skill、生成题目、调用 `review_answer` 前后，主输出区是否仍被进度条锁住。
- 若仍复现：
  - 下一步应检查 pi-agent 本体 streaming/status 渲染是否在长 tool-call 参数下仍有布局问题。
  - 可进一步把 `/skill:*` 自然语言提示改为完全由短契约摘要接管，减少 agent 主动读 skill 的频率。
