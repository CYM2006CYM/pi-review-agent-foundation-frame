# 真题资料包能力设计

更新日期：2026-06-07

## 背景

当前 Review profile 已经覆盖三类核心资料：知识点卡片、章节材料、考点总结。下一步如果要提升复习价值，最重要的资料不是更多笔记，而是真题。

真题的价值不只是“拿来做题”。它还可以反向生成：

- 高频知识点和考点分布。
- 真实题型结构和命题习惯。
- 题干表达模板和选项干扰方式。
- 难度分布和综合题组合方式。
- 用于随机抽取的真实练习题库。
- 用于 agent 仿真出题的风格约束。

因此，真题能力不应只是把 Markdown 文件放进资料包，而应成为 profile 的一等资料层。

## 产品目标

### 面向学生

学生不需要理解真题分析流程。只要 profile 中包含真题，就可以在 `/review` 中选择：

- 随机抽一道真题。
- 按章节抽真题。
- 按知识点抽真题。
- 按错题薄弱点抽相似真题。
- 先看真题考法，再做仿真题。

### 面向 profile 制作者

制作者只需要提供 Markdown 版真题，系统负责：

- 切分题目。
- 识别题型。
- 归类知识点。
- 生成真题风格分析。
- 生成题型模板。
- 生成质量报告和待复核项。

### 面向 agent

agent 在出题时不再只依赖抽象知识点，而可以读取：

- 历年真题样例。
- 题型模板。
- 命题风格报告。
- 某知识点的真实考法。

这能显著减少“题目像教材问答，不像考试题”的问题。

## 新增资料包结构

建议在 profile 中新增以下目录和文件：

```text
review_profiles/{subjectId}/
  past_papers/
    raw/
      2024-final.md
      2023-final.md
    questions/
      q_2024_final_001.md
      q_2024_final_002.md
    past_questions.json
    style_report.md
    templates/
      choice_definition_compare.md
      short_answer_reasoning.md
      code_reading_output.md
    quality_report.md
```

### `past_papers/raw/`

保存用户提供的 Markdown 真题原文。首版只支持 `.md` 和 `.txt`，不直接处理 PDF / Word / 图片。

要求尽量保留原始结构：

```markdown
# 2024 年期末考试

## 一、判断题

1. 纯虚函数不能有函数体。（ ）

2. 析构函数可以是虚函数。（ ）

## 二、选择题

1. 下列关于友元函数的说法正确的是：
   A. ...
   B. ...
```

### `past_papers/questions/`

保存切分后的单题 Markdown。每题一个文件，便于人工复核和版本管理。

推荐格式：

```markdown
---
id: q_2024_final_001
paper_id: 2024-final
question_no: 1
type: judgment
points: 2
difficulty: S-U
knowledge_points:
  - pure_virtual_function
source_range: "2024-final.md#一-判断题-1"
status: reviewed
---

# 题目

纯虚函数不能有函数体。（ ）

# 答案

错误

# 解析

纯虚函数可以有类外定义，但只要类中存在纯虚函数，该类仍然是抽象类。
```

### `past_papers/past_questions.json`

保存结构化真题索引，供代码快速检索。

建议 schema：

```json
{
  "version": 1,
  "subject_id": "cpp-oop",
  "papers": [
    {
      "id": "2024-final",
      "title": "2024 年期末考试",
      "exam_type": "final",
      "year": 2024,
      "source_file": "past_papers/raw/2024-final.md"
    }
  ],
  "questions": [
    {
      "id": "q_2024_final_001",
      "paper_id": "2024-final",
      "question_no": "一-1",
      "type": "judgment",
      "points": 2,
      "difficulty": "S-U",
      "knowledge_points": ["pure_virtual_function"],
      "chapter_ids": ["chapter_abstract_class"],
      "template_ids": ["judgment_concept_exception"],
      "has_answer": true,
      "has_explanation": true,
      "file": "past_papers/questions/q_2024_final_001.md",
      "source_basis": "2024-final.md#一-判断题-1",
      "confidence": 0.86,
      "review_status": "auto"
    }
  ]
}
```

### `past_papers/style_report.md`

保存真题风格分析。它不是给学生背的知识点，而是给 agent 出题和给用户了解考试风格用的命题分析。

建议包含：

- 题型分布。
- 分值分布。
- 高频知识点。
- 高频混淆点。
- 命题表达习惯。
- 选项干扰方式。
- 简答题常见追问方式。
- 综合题组合模式。
- 与章节知识点的覆盖关系。
- 出题时必须遵循的风格约束。

### `past_papers/templates/`

保存从真题中抽象出的命题模板。模板不是具体题目，而是“如何生成类似真题”的结构化说明。

示例：

```markdown
---
id: judgment_concept_exception
type: judgment
source_questions:
  - q_2024_final_001
knowledge_pattern:
  - concept_definition
  - exception_case
difficulty: S-U
---

# 模板说明

考查一个概念的定义边界，并故意加入一个常见例外情况。

# 生成结构

给出一个绝对化判断句：

「{概念} 一定/不能/必须 {行为}。」

# 干扰策略

- 使用“不能”“必须”“一定”等绝对词。
- 将常规情况和例外情况混在一起。

# 判题要点

- 说明一般规则。
- 指出例外。
- 回到概念定义本身。
```

## 与现有资料层的关系

真题资料层不是替代 `cards/`、`chapters/`、`exam_points/`，而是增强它们。

```text
chapters/       解释知识结构
cards/          帮助记忆和概念辨析
exam_points/    总结考试导向
past_papers/    提供真实题型、真实考法、真实风格
```

知识点关系：

```text
knowledge_index.json
  chapters.*.knowledge_points[]
        ↑
        | 归类
        |
past_papers/past_questions.json
```

每道真题必须尽量映射到已有 `knowledge_points`。如果映射不到，应进入质量报告，提示 profile 需要补充知识点或标记为跨章节综合题。

## 新增流程

### `/review-init` 中的真题处理

如果源资料目录中存在真题文件，init 流程应增加以下步骤：

1. 识别真题文件。
2. 读取真题 Markdown。
3. 按题号、题型标题、分值标记切分题目。
4. 抽取题干、选项、答案、解析。
5. 生成单题 Markdown。
6. 生成 `past_questions.json`。
7. 将题目映射到知识点。
8. 生成 `style_report.md`。
9. 生成题型模板。
10. 在 `quality_report.md` 中列出低置信度题目。

### `/review-fix` 中的真题修订

用户可以自然语言反馈：

- “2023 年选择题切错了。”
- “这些题都属于第 8 章多态。”
- “第 5 题不是简答题，是代码阅读题。”
- “这个模板太泛，应该拆成两个。”

agent 应只修改 draft profile，不直接修改 active profile。

### `/review` 中的真题使用

建议新增入口或模式：

```text
1. 概念卡片 + 练习
2. 直接练习
3. 章节学习
4. 真题练习
```

真题练习支持：

- 随机真题。
- 按年份。
- 按章节。
- 按知识点。
- 按题型。
- 只练错过的真题。
- 先做真题，再生成相似题。

模式 2 直接练习也可以读取真题风格，用于生成更像考试的题目。

## 新增工具建议

### `review_past_question`

代码读取并展示一题真实真题。

输入：

```json
{
  "subject_id": "cpp-oop",
  "question_id": "q_2024_final_001"
}
```

输出：

```json
{
  "question_id": "q_2024_final_001",
  "action": "answer",
  "type": "judgment",
  "knowledge_points": ["pure_virtual_function"]
}
```

### `review_past_question_pick`

代码按条件抽题。

输入：

```json
{
  "subject_id": "cpp-oop",
  "scope": {
    "chapter_id": "chapter_abstract_class",
    "question_type": "judgment"
  },
  "strategy": "random"
}
```

策略：

- `random`
- `unseen_first`
- `wrong_first`
- `low_confidence`
- `by_year`

### `review_past_style`

代码展示真题风格报告或某知识点的真题考法。

输入：

```json
{
  "subject_id": "cpp-oop",
  "knowledge_point_id": "pure_virtual_function"
}
```

### `review_template_generate_question`

让 agent 基于真题模板生成仿真题。这个工具可以先不做成代码生成题，而是提供模板上下文给 prompt，最终仍由 `review_answer` 负责答题 UI。

## 真题切分策略

首版不要追求完全自动正确。应采用“自动切分 + 质量报告 + 用户复核”的策略。

### 可识别标记

优先识别：

- 一级/二级标题中的年份和考试名称。
- “一、判断题”“二、选择题”“三、简答题”。
- `1.`、`1、`、`（1）`、`【题 1】`。
- `A.`、`B.`、`C.`、`D.` 选项。
- “答案：”“参考答案：”“解析：”。
- 分值标记，如 “（2 分）”。

### 低置信度情况

以下情况必须进入 `past_papers/quality_report.md`：

- 题号不连续。
- 选项数量异常。
- 找不到答案。
- 答案和题型不匹配。
- 一道题疑似包含多个小问。
- 无法映射知识点。
- 同一题重复出现。
- 题干过短或过长。

## 知识点归类策略

真题归类应结合三类证据：

1. 题干关键词。
2. 答案/解析中的概念。
3. `knowledge_index.json` 中的 aliases、tags、chapter 信息。

输出时必须给出置信度：

```json
{
  "knowledge_points": [
    {
      "id": "friend_function",
      "confidence": 0.92,
      "basis": "题干出现 friend function，解析讨论非成员访问私有成员"
    }
  ]
}
```

低于阈值的归类不应静默写死，应进入待复核项。

## 真题风格分析

风格分析应回答五个问题：

1. 考什么：知识点频率和章节分布。
2. 怎么考：题型、题干结构、分值和答题要求。
3. 怎么误导：常见陷阱、干扰项、绝对化表述。
4. 怎么综合：哪些知识点经常组合出现。
5. 怎么仿真：生成新题时应遵守哪些模板和约束。

示例输出：

```markdown
# 真题风格分析

## 高频考法

- 抽象类常通过“纯虚函数是否能定义”“不能实例化是否等于抽象类”进行判断题考查。
- 友元常通过“是否是成员函数”“是否破坏封装”进行概念辨析。

## 干扰项风格

- 喜欢使用“必须”“只能”“不能”等绝对词。
- 常把语法允许和设计建议混在一起。

## 出题约束

- 判断题应优先考边界条件。
- 选择题应至少包含一个“看似正确但偷换概念”的选项。
- 简答题应要求解释原因，而不是只背定义。
```

## 与出题系统的结合

### 使用真题复习

真实真题应原样展示，不由 agent 改写题干。用户作答后，agent 可以解释和扩展，但题目本身保持原貌。

流程：

```text
选择真题范围
-> review_past_question_pick 抽题
-> review_past_question 展示题目
-> review_answer 收集答案
-> agent 判题
-> review_archive 归档
-> review_turn_action
```

### 使用真题生成仿真题

仿真题不是复制真题，而是使用真题模板生成新题。

流程：

```text
选择知识点
-> 读取相关真题
-> 读取 templates
-> 读取 style_report
-> agent 生成 Question JSON
-> review_answer
```

`Question JSON` 建议增加字段：

```json
{
  "source_basis": "template: judgment_concept_exception; past_questions: q_2024_final_001",
  "is_past_question": false,
  "past_question_style": true
}
```

真实真题则：

```json
{
  "source_basis": "past_question: q_2024_final_001",
  "is_past_question": true,
  "past_question_style": false
}
```

## 学习状态

建议新增：

```text
state/past_question_progress.json
```

结构：

```json
{
  "q_2024_final_001": {
    "seen_count": 1,
    "practice_count": 1,
    "correct_count": 0,
    "last_seen_at": "2026-06-07T12:00:00.000Z",
    "last_practiced_at": "2026-06-07T12:00:00.000Z",
    "last_answer": "正确",
    "confidence": 0.42
  }
}
```

真题状态应和知识点状态联动：

- 真题答错，降低相关知识点 confidence。
- 同一知识点多道真题答对，提高 confidence。
- 未做过的真题优先进入考前复习队列。

## Profile 质量指标

有真题的 profile 应额外统计：

- 真题文件数量。
- 题目切分数量。
- 有答案题目比例。
- 有解析题目比例。
- 知识点映射覆盖率。
- 低置信度题目数量。
- 模板数量。
- 每章真题覆盖率。

这些指标应写入 `past_papers/quality_report.md`，并汇总到 profile 的 `quality_report.md`。

## 分阶段实施计划

### 阶段 1：真题资料结构和读取

目标：profile 能包含真题，并能被代码读取。

实现：

- 定义 `past_papers/` 目录规范。
- 新增 `past_questions.json` 校验。
- 新增读取模块，例如 `lib/past_papers.mjs`。
- 新增单元测试：读取真题索引、按章节/知识点/题型过滤。

验收：

- demo profile 可包含 3-5 道真题。
- `/review` 能检测 profile 是否有真题。

### 阶段 2：真题展示和随机练习

目标：用户能随机抽真题做。

实现：

- 新增 `review_past_question_pick`。
- 新增 `review_past_question`。
- 新增真题练习入口或在模式 2 中增加“真题优先”。
- 答题仍复用 `review_answer` 和 `review_archive`。

验收：

- 能按章节随机抽一题真题。
- 真题答题记录能归档。
- 真题做题进度能更新。

### 阶段 3：自动切分和质量报告

目标：`/review-init` 能从 Markdown 真题生成结构化真题库。

实现：

- 增加真题切分 skill。
- 增加受控写入 `past_papers/` 的规则。
- 自动生成 `questions/` 和 `past_questions.json`。
- 生成 `past_papers/quality_report.md`。

验收：

- 给定一份规范 Markdown 真题，可以自动切成多道题。
- 切分低置信度项进入质量报告。

### 阶段 4：知识点归类和风格分析

目标：真题能反哺知识点和命题风格。

实现：

- agent 根据 `knowledge_index.json` 为真题归类。
- 生成 `style_report.md`。
- 生成 `templates/`。
- `/review-fix` 支持修订真题归类和模板。

验收：

- 每道真题有知识点映射或待复核原因。
- 某知识点能展示相关真题和常见考法。

### 阶段 5：仿真出题

目标：使用真题模板生成更像考试的新题。

实现：

- `buildReviewStartPrompt()` 在直接练习模式注入相关真题模板摘要。
- 题目 JSON 增加真题风格来源字段。
- `review-question` skill 增加真题模板使用规范。

验收：

- 同一知识点能生成“教材问答型”和“真题风格型”两种题。
- 用户能看到题目来源依据。

## 风险和控制

### 版权和公开风险

如果真题来自学校课程或考试资料，公开仓库中是否能包含需要单独判断。建议支持两种模式：

- 公开 demo：使用自制模拟真题。
- 班级私用：允许本地导入真实真题，不默认发布到公开仓库。

### 切分错误风险

真题格式非常不统一，自动切分必然会错。必须保留人工复核路径，不允许低置信度切分直接 active。

### 过拟合真题风险

产品不应只训练背题。真题应服务于理解考试风格和定位薄弱点。仿真题应基于模板和知识点变体，而不是复述原题。

### Agent 生成题目偏离真题风格

真题风格报告必须包含可执行约束，而不是泛泛总结。出题 prompt 应明确引用模板 id 和 source questions。

## 推荐近期落地顺序

1. 先手工为 demo profile 加 3-5 道模拟真题，验证目录结构。
2. 实现 `past_papers.mjs` 和真题索引读取测试。
3. 实现随机真题练习工具，复用现有答题和归档。
4. 再做 `/review-init` 自动切分。
5. 最后做风格分析和仿真出题。

## 一句话总结

真题能力的核心不是“存真题”，而是把真题转化为可抽取、可归类、可分析、可仿真的 profile 资产。
