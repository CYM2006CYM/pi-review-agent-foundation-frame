# 多重继承名字冲突

[[多重继承名字冲突]] 是多个基类中存在同名成员，派生类直接访问该名字时产生二义性的现象。

```cpp
class C : public A, public B {
public:
    void k() {
        A::f();       // 明确调用 A 中的 f
    }
};
```

可以用作用域限定或 `using` 缓解，但数据成员冲突仍会真实存在于对象布局中。
