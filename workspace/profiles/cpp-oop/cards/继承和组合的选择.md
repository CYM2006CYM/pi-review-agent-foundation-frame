# 继承和组合的选择

[[继承和组合的选择]] 的核心问题是：当前类间关系到底是“是一种”，还是“有一个/用一个”。

判断规则：

| 关系 | 推荐表达 |
|---|---|
| 子类是一种父类 | [[公有继承]] |
| 一个类拥有另一个类作为组成部分 | [[组合]] |
| 一个类临时使用另一个类的功能 | [[依赖]] |
| 一个类长期知道另一个对象 | [[关联]] |
| 只是为了复用实现 | 优先组合，谨慎继承 |

> [!important] 高频考点
> 能说出“子类是一个父类”才使用公有继承；只能说“类中有一个对象”时，优先组合。
