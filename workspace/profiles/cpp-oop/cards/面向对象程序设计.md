# 面向对象程序设计

[[面向对象程序设计]]是一种重要的[[编程范型]]。课程画面把它概括为：对象间的相互作用，以及抽象、封装、信息隐藏、继承、复用的思想。

## 核心关键词

| 关键词 | 复习理解 |
|---|---|
| [[抽象]] | 提取问题中的关键属性和行为 |
| [[封装]] | 将数据和操作组织在同一对象或类中 |
| [[信息隐藏]] | 隐藏实现细节，只暴露必要接口 |
| [[继承]] | 用已有类型构造新类型，表达层次关系 |
| [[复用]] | 复用已有设计和代码，减少重复 |
| [[多态]] | 同一接口在不同对象上表现出不同行为 |

## 初学者解释

过程式程序常问：“先做什么，再做什么？”  
面向对象程序常问：“有哪些对象？它们各自负责什么？它们怎样协作？”

## 易错点

> [!warning] 易错
> 面向对象不是“把所有代码都放进类里”。真正重要的是对象职责、接口边界、封装和对象关系。
