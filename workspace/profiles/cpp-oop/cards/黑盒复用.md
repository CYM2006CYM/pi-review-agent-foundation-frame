# 黑盒复用

[[黑盒复用]] 是把已有类当作不透明的“盒子”来使用：使用者只依赖它公开出来的接口，不关心内部数据、私有成员和具体实现算法。

在 C++ 类设计中，黑盒复用通常通过[[依赖]]、[[关联]]、[[聚合]]、[[组合]]等水平关系实现。

```cpp
class B {
public:
    void f(A& a) {
        a.func1(); // 只通过 A 的公开接口复用功能
    }
};
```

黑盒复用的本质是[[功能复用]]。如果被复用类内部实现改变，但公开接口和语义不变，使用者通常不需要修改。
