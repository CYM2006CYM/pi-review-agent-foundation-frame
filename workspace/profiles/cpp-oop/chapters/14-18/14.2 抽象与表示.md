# 14.2 抽象与表示

[[抽象与表示]] 是类设计中最基础的一对概念。

- [[抽象]]：先从问题描述中提取类型、行为、数据。
- [[具体表示]]：再用某种语言机制把抽象结果写成类、成员函数、数据成员。

抽象先于表示。表示可以依赖 C++ 语法，但抽象本身不应该一开始就被代码细节绑住。

## 抽象关注什么

面向对象抽象通常关注三件事：

| 抽象对象 | 需要回答的问题 |
|---|---|
| 类型 | 这个问题域中有哪些重要事物？ |
| 行为 | 这些事物对外能做什么？ |
| 数据 | 为了支持这些行为，需要记录哪些状态？ |

> [!important] 高频考点
> 初学类设计时，不要一上来就想“数据成员怎么写”。应先用自然语言说清楚类型和行为，再考虑数据表示。

## 小球碰撞例

问题描述：屏幕中有小球，小球运动，碰到边界或挡板会反弹。

### 类型抽象

从描述中可以先抽出：

- `Ball`：小球。
- `Border`：边界或挡板。

边界和挡板是否要继续细分，不必一开始就决定。先抓住当前问题中最主要的类型。

### 行为抽象

小球可能有这些行为：

- 设置位置。
- 设置速度。
- 移动一步。
- 检测碰撞。
- 反弹。
- 设置能量损耗系数或反弹系数。

边界也可能有这些行为：

- 设置位置。
- 设置反弹系数。
- 检测是否发生碰撞。
- 反弹处理。

这些行为先用自然语言说清楚，不必立刻决定参数列表和函数体。

### 数据抽象

数据可以包括：

- 位置。
- 速度。
- 半径。
- 反弹系数。
- 边界位置。

至于位置使用笛卡尔坐标、极坐标、复数表示，属于后续具体表示阶段。

### 具体表示

抽象完成后，才进入 C++ 表示：

```cpp
class Ball {
public:
    void setPosition(int x, int y);
    void setSpeed(int speed);
    void stepMove();
    void rebound();

private:
    int mx;
    int my;
    int speed;
    float reboundScale;
};

class Border {
public:
    void setPosition(int x, int y);
    void setScale(float scale);
    bool hit(const Ball& ball) const;

private:
    int mx;
    int my;
    float scale;
};
```

这里的代码只是某一种表示，不是唯一答案。抽象合理，表示才有意义；抽象不合理，语法写对也会影响后续维护和扩展。

## 怪物战斗例

另一个例子是怪物战斗：多个怪物一对一战斗，每个怪物有生命值、攻击力、防御力，双方轮流攻击直到一方生命值为零。

先做自然语言压缩：一个怪物攻击另一个怪物。

因此可以抽象出：

- 类型：`Monster`。
- 行为：`fight(Monster& other)`、攻击、读取或设置属性。
- 数据：生命值、攻击力、防御力等。

```cpp
class Monster {
public:
    bool fight(Monster& other);

private:
    bool attack(Monster& other);
    int hp;
    int attackPower;
    int defensePower;
};
```

`fight` 的实现可以先用自然语言描述：

1. 当前怪物攻击对方。
2. 如果对方生命值为零，当前怪物获胜。
3. 对方攻击当前怪物。
4. 如果当前怪物生命值为零，当前怪物失败。
5. 否则继续循环。

这个过程说明：代码应尽量接近清楚的自然语言描述，让阅读者容易理解。

## 登录信息例

登录函数可能需要用户名、密码、验证码、时间、客户端 ID、IP 地址等大量参数。

可以有三种设计：

```cpp
void login(const char* username,
           const char* password,
           const char* validCode,
           DateTime dt,
           int clientID,
           const char* ip);

void login(LoginInfo& info);

void login(UserInfo& user, Address& client);
```

这三种都可能正确，取决于问题域和维护需要。

如果一组参数经常一起出现，并且在概念上共同表达“登录信息”，就可以抽象为一个类。抽象对象不一定必须是现实世界中独立存在的实体，只要它对理解和维护有意义，就可以抽象。

> [!warning] 易错点
> “现实中有没有这个东西”不是能否抽象的唯一标准。只要在程序设计中有意义，就可以抽象成类型。

## 本节小结

- 抽象先于表示。
- 抽象关注类型、行为、数据。
- 表示才关心 C++ 中的类、成员函数、参数和数据成员。
- 行为实现前应先用自然语言描述清楚。
- 任何有意义、能降低复杂度的概念都可以抽象成类型。
