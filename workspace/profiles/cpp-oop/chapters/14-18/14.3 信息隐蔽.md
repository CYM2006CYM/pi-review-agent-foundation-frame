# 14.3 信息隐蔽

[[信息隐蔽]] 指在类的设计中，把不需要让外部知道的行为和数据隐藏起来，只公开必要接口。

一个类可以隐藏：

- 内部行为或操作。
- 内部数据。
- 具体实现过程。

## 行为、操作与成员函数

在抽象层面，一个行为可能由多个操作构成。到了 C++ 表示层面，一个操作通常由一个或多个[[成员函数]]实现。

因此，信息隐蔽不仅针对数据，也针对成员函数。

例如小球移动一步这个公开行为，内部可能包含：

- 检测是否碰撞。
- 计算反弹方向。
- 更新坐标。

外部只需要调用 `stepMove()`，不一定需要直接调用 `hitBorder()` 之类的内部函数。内部函数可以设为 `private`。

```cpp
class Ball {
public:
    void stepMove();

private:
    bool hitBorder() const;
    void rebound();
    int mx;
    int my;
};
```

`stepMove` 是公开接口，`hitBorder` 和 `rebound` 是实现细节。

## 为什么数据通常私有

面向对象关注对象的本质行为，而不是让外部直接操作数据。

如果数据成员公开，外部代码会直接依赖对象的内部表示。以后只要数据结构变化，外部代码就可能全部受影响。

因此常见设计是：

- 数据成员尽量 `private`。
- 通过 `public` 成员函数提供必要访问。
- 不公开不必要的 getter/setter。

> [!warning] 易错点
> 把所有数据都写成 `private` 后再机械生成全部 getter/setter，并不一定是好的信息隐蔽。真正要控制的是外部能依赖什么。

## 信息隐蔽的价值

信息隐蔽的本质是减少外部可见的行为和数据，让对象之间只能通过公开行为交互。

它带来三个直接价值：

| 价值 | 原因 |
|---|---|
| 软件更安全可靠 | 外部不能随意调用内部函数或修改内部数据 |
| 维护更方便 | 内部实现可以修改，外部调用方不必跟着改 |
| 便于复用 | 使用者只依赖公开接口，不需要理解全部内部实现 |

例如一个类有 10 个成员函数，其中只有 1 个是公开接口，另外 9 个是内部辅助函数。外部只能走公开接口，错误使用的可能性会明显降低。

## 本节小结

- 信息隐蔽既隐藏数据，也隐藏内部行为。
- 一个行为可以由多个操作构成，一个操作可以由成员函数实现。
- 外部应该通过公开接口使用对象。
- 数据通常设为私有，避免外部依赖内部表示。
- 信息隐蔽提升安全性、维护性和复用性。
