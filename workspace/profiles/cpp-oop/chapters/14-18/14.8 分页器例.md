# 14.8 分页器例

[[分页器]] 是网页和应用中常见的控件，用来在大量内容中按页浏览。例如论坛帖子、搜索结果、商品列表，都可能需要分页。

本节重点不是图形界面实现，而是从分页器例子中练习类的行为抽象、数据抽象和接口设计。

## 分页器的行为抽象

分页器常见行为包括：

```cpp
class Paginate {
public:
    void go(int page);
    void prev();
    void next();
    void prevN();
    void nextN();
};
```

这些行为对应：

| 行为 | 含义 |
|---|---|
| `go(page)` | 跳转到指定页 |
| `prev()` | 前一页 |
| `next()` | 后一页 |
| `prevN()` | 向前跳一段页数 |
| `nextN()` | 向后跳一段页数 |

页码越界不应简单导致程序错误。例如总页数是 90，用户输入 200，可以把结果限制到 90。这种范围检查应封装在类内部。

## 分页器的数据抽象

分页器可能需要维护：

```cpp
class Paginate {
private:
    int num;        // 当前页
    int total;      // 总页数
    int rangeStart; // 当前显示段起始页
    int rangeEnd;   // 当前显示段结束页
};
```

数据可以更多，也可以更少，取决于具体实现。由于数据是私有的，外部不应直接依赖内部表示。只要公开接口语义稳定，内部数据组织可以调整。

## 还要考虑哪些成员

分页器类不只需要核心翻页行为，还要考虑常规类设计问题：

| 成员 | 是否需要 | 判断依据 |
|---|---|---|
| 构造函数 | 通常需要 | 设置总页数、当前页、显示范围等初始状态 |
| 拷贝构造函数 | 视情况 | 如果只有简单值类型，默认拷贝可能足够 |
| 赋值函数 | 视情况 | 如果允许分页器赋值，默认赋值可能足够 |
| 析构函数 | 视情况 | 如果没有动态资源，默认析构通常足够 |
| 显示函数 | 视平台 | 控制台、窗口、网页显示方式不同 |
| getter/setter | 按需提供 | 不应机械公开全部数据 |

如果类中只有 `int` 这类简单值类型，编译器默认的拷贝构造和赋值函数通常足够。若以后引入动态内存、文件句柄、图形资源等，再考虑自定义。

## 一个可能的接口草案

```cpp
class Paginate {
public:
    explicit Paginate(int total, int page = 1);

    void go(int page);
    void prev();
    void next();
    void prevN(int n);
    void nextN(int n);

    int currentPage() const;
    int totalPages() const;
    void show() const;

private:
    int clamp(int page) const;
    void updateRange();

    int num;
    int total;
    int rangeStart;
    int rangeEnd;
};
```

`clamp` 和 `updateRange` 是内部辅助行为，不需要公开。外部只需要知道“跳转、上一页、下一页、显示”等高层行为。

## 本节小结

- 分页器例说明类设计要先抽象行为和数据。
- 翻页、跳页、段跳转是分页器核心行为。
- 当前页、总页数、显示范围是常见内部数据。
- 范围检查应封装在类内部。
- 构造、拷贝、赋值、显示、getter/setter 都要按需要判断，不机械添加。
