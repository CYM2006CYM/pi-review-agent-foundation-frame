# 15.1 黑盒复用和白盒复用

本节进入第 15 章“继承”的主线。前面课程通过[[抽象]]把问题中的事物表示成[[类]]，而类设计的重要目的之一就是[[软件复用]]：已经写好、测试过、语义稳定的类，应该能在后续程序中继续使用。

复用不只是“少写代码”，更重要的是减少修改已有代码带来的风险。

## 不推荐的低层次复用

最常见的复用方式是[[拷贝粘贴修改复用]]：

```text
复制已有代码 -> 粘贴到新位置 -> 按新需求修改
```

它确实复用了已有代码的一部分，但问题很明显：

- 修改后的代码不再等价于原代码；
- 哪怕只改一条语句，也可能破坏原有语义；
- 原代码是否仍正确，需要重新测试；
- 多份复制代码会增加维护成本。

从面向对象的角度看，更理想的复用方式是：已有类的代码一个字、一个标点都不改，仍然通过某种机制把它用起来。

## 两种类级复用

课程把不改动已有类代码的复用分成两大类：

| 复用方式 | 核心比喻 | 使用者知道什么 | C++ 中常见形式 |
|---|---|---|---|
| [[黑盒复用]] | 不透明盒子 | 只知道公开接口 | [[依赖]]、[[关联]]、[[聚合]]、[[组合]] |
| [[白盒复用]] | 透明盒子 | 能看到并继承类内部成员 | [[继承]] |

黑盒和白盒不是说“有没有复用”，而是说“以什么方式看待已有类”。

## 黑盒复用：通过公开接口使用功能

黑盒复用把已有类看作一个不透明盒子。盒子内部有什么数据、私有函数、算法实现，使用者不需要知道；使用者只需要知道它提供了哪些公开功能。

```cpp
class A {
public:
    void func1();
    void func2();
private:
    void func3();
    int data;
};

class B {
public:
    void bfunc(A& a) {
        a.func1();
    }
};
```

`B` 通过参数 `A& a` 调用 `a.func1()`，复用了 `A` 的功能。但 `B` 不知道 `A::func1()` 的内部实现，也不能直接访问 `A` 的私有数据。

这就是[[依赖]]形式的黑盒复用。

## 关联、聚合、组合也是黑盒复用

黑盒复用也可以通过数据成员形式出现：

```cpp
class C {
public:
    void cfunc() {
        pa->func1();
    }

private:
    A* pa;
};
```

这里 `C` 保存了指向 `A` 的指针，通过 `pa->func1()` 使用 `A` 的公开功能。`C` 与 `A` 之间是[[关联]]关系；如果生命周期关系更强，还可能构成[[聚合]]或[[组合]]。

无论形式是参数、指针成员还是对象成员，只要复用者只通过公开接口使用被复用类，就属于黑盒复用。

## 黑盒复用是功能复用

黑盒复用真正关心的是“这个类提供什么功能”，而不是“这个类内部如何实现”。

这带来一个重要好处：只要公开接口和语义稳定，被复用类的内部实现可以调整，而使用者代码往往不需要改变。

> [!important] 高频考点
> 黑盒复用依赖公开接口，体现[[信息隐蔽]]。类内部数据和算法可以改变，只要外部接口语义不变，复用者通常不受影响。

## 白盒复用：通过继承吸收成员

白盒复用把已有类看作透明盒子。新类可以看见并继承已有类的成员，把它们作为自身的一部分。

在 C++ 中，白盒复用主要由[[继承]]实现：

```cpp
class B : public A {
public:
    void bfunc() {
        func1();
    }
};
```

`B` 从 `A` 派生后，`A` 中允许继承和访问的成员成为 `B` 的一部分。`B` 的成员函数可以像使用自己成员一样使用基类成员。

这与黑盒复用不同：黑盒复用是“我拿一个 A 来用它的功能”；白盒复用是“我从 A 派生出 B，A 的成员成为 B 的组成部分”。

## 本节小结

| 问题 | 答案 |
|---|---|
| 为什么不建议拷贝粘贴修改？ | 修改后正确性不再自动保证，维护成本高 |
| 黑盒复用靠什么？ | 公开接口 |
| 黑盒复用常见类间关系 | 依赖、关联、聚合、组合 |
| 黑盒复用复用什么？ | 功能 |
| 白盒复用靠什么？ | 继承 |
| 继承在 C++ 中的作用 | 实现类代码的白盒复用 |

> [!summary] 考前速记
> 复用的目标是不改动已有类代码。黑盒复用只看公开接口，常由依赖、关联、聚合、组合实现；白盒复用能看到并吸收基类成员，在 C++ 中主要由继承实现。
