# 15.2 继承和派生

上一节说到，C++ 用[[继承]]实现[[白盒复用]]。本节先学习继承的基本语法和术语。

## 继承的基本语法

C++ 中派生类的基本格式是：

```cpp
class 派生类名 : 继承方式 基类名1, 继承方式 基类名2 {
    // 派生类自己的成员
};
```

更具体的例子：

```cpp
class Child : public Parent1, private Parent2 {
public:
    Child(int n);
    void childFunc();

private:
    int data1;
    OtherClass* ptr;
};
```

`Child` 是[[派生类]]，`Parent1`、`Parent2` 是[[基类]]。冒号后面的部分是[[继承列表]]。

## 继承方式

继承方式有三种：

| 继承方式 | 关键字 | 大体含义 |
|---|---|---|
| [[公有继承]] | `public` | 基类公开接口在派生类中继续公开，常用于表达父类/子类 |
| [[保护继承]] | `protected` | 基类公开和保护成员在派生类中变成保护成员 |
| [[私有继承]] | `private` | 基类公开和保护成员在派生类中变成私有成员 |

如果 `class` 的继承列表中省略继承方式，默认是私有继承：

```cpp
class Child : Parent {
};
```

上面等价于：

```cpp
class Child : private Parent {
};
```

> [!warning] 易错点
> `class` 默认私有继承；不要看到继承列表里只写了基类名，就误认为是公有继承。

## 单继承和多重继承

如果一个派生类只有一个直接基类，称为[[单继承]]：

```cpp
class Child : public Parent {
};
```

如果一个派生类有多个直接基类，称为[[多重继承]]：

```cpp
class Child : public Parent1, public Parent2 {
};
```

多重继承语法允许，但设计复杂度更高，后续会涉及二义性、构造顺序等问题。初学阶段先把单继承、公有继承理解扎实。

## 继承层次不宜过深

派生类还可以继续派生出新的类，从而形成继承树：

```text
Base
├── Derived1
│   └── MoreDerived1
└── Derived2
    └── MoreDerived2
```

语法上，继承层数没有硬性限制；但设计上不应过深。

课程给出的经验：

- 常规设计中，三层左右比较容易理解；
- 三到五层通常已经足够；
- 七到九层已经接近人理解能力的上限，不适合普通工程设计。

继承层次过深会带来两个问题：

- 自己难以把握每层职责和访问关系；
- 同伴维护时也很难理解代码。

## 基类/派生类与父类/子类

[[基类]]和[[派生类]]是 C++ 继承语法中的一般术语。无论公有继承、保护继承、私有继承，都可以说某类是基类，某类是派生类。

[[父类]]和[[子类]]更强调类型语义。课程中说明：通常只有在[[公有继承]]下，才更适合把基类称为父类，把派生类称为子类。

这是因为公有继承更接近“子类是一种父类”的类型关系。

## 本节小结

| 术语 | 必背结论 |
|---|---|
| 继承列表 | 写在派生类名后，指出继承方式和基类名 |
| 继承方式 | public、protected、private |
| 默认继承方式 | `class` 省略时默认 private |
| 单继承 | 一个直接基类 |
| 多重继承 | 多个直接基类 |
| 继承层次 | 不宜过深，通常控制在三到五层 |
| 父类/子类 | 通常用于公有继承语义 |

> [!summary] 考前速记
> `class Derived : public Base { ... };` 是继承基本语法。冒号后是继承列表。继承方式有 public、protected、private；`class` 省略继承方式时默认 private。单继承一个基类，多重继承多个基类。继承层次可以继续派生，但设计上不宜过深。
