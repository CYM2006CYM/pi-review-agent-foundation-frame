# 15.3 派生类中的成员及访问控制

继承语法看起来很简单，但派生类内部到底拥有哪些成员、对象占多大空间、继承来的成员能不能访问，是本节的重点。

## 派生类中有哪些成员

[[派生类]]中的成员大体有三类：

| 成员来源 | 内容 |
|---|---|
| 派生类特殊函数 | 构造函数、析构函数、拷贝构造函数、赋值函数 |
| 派生类自己定义的成员 | 新增数据成员、新增成员函数 |
| 从基类继承来的成员 | 基类中的普通成员函数和数据成员 |

其中要特别注意：基类的构造函数、析构函数、拷贝构造函数和赋值函数不会作为普通成员自动继承给派生类。

> [!important] 高频考点
> 派生类有自己的构造、析构、拷贝构造和赋值函数；如果用户不写，编译器会按规则提供默认版本。但这不等于继承了基类的这些特殊函数。

## 继承来的成员是派生类的一部分

例如：

```cpp
class A {
public:
    void afunc1();
    void afunc2();
private:
    void afunc3();
    int data;
};

class B : public A {
public:
    void bfunc1() {
        afunc1();
    }
private:
    void bfunc2();
    int value;
};
```

`B` 从 `A` 派生后，`A` 的成员成为 `B` 的一部分。`B::bfunc1()` 中可以直接调用允许访问的 `afunc1()`。

这体现了[[白盒复用]]：基类成员不是像黑盒复用那样通过某个对象参数来调用，而是成为派生类自身结构的一部分。

## 派生类对象的大小

派生类对象通常由两部分组成：

```text
派生类对象 = 基类子对象 + 派生类新增数据
```

对象所占内存主要与非静态数据成员有关，而不是与普通成员函数数量直接相关。

例如 `B` 继承 `A`：

- `A` 中的数据成员形成[[基类子对象]]部分；
- `B` 自己新增的数据成员形成派生类新增部分；
- 两部分合起来构成完整的 `B` 对象。

如果基类没有数据成员，单独创建基类对象时也不能大小为 0；但当它作为派生类的基类子对象时，编译器可能对空基类做优化，使其不额外占用空间。

后续涉及[[虚函数]]时，派生类对象还可能包含虚函数表指针等额外内容。

## 派生类访问控制表

继承来的成员在派生类中呈现什么访问权限，取决于两个因素：

- 该成员在基类中的原访问权限；
- 采用的继承方式。

| 基类成员原权限 | public 继承 | protected 继承 | private 继承 |
|---|---|---|---|
| `public` | `public` | `protected` | `private` |
| `protected` | `protected` | `protected` | `private` |
| `private` | 不可直接访问 | 不可直接访问 | 不可直接访问 |

这张表是本节最重要的考试表之一。

## private 成员不是没有继承

基类的 `private` 成员在派生类中不可直接访问，但并不是“派生类对象里没有它们”。

```cpp
class Base {
private:
    int num;

public:
    void setNum(int n) {
        num = n;
    }
};

class Derived : public Base {
public:
    void f() {
        setNum(10); // 合法
        // num = 10; // 非法
    }
};
```

`Derived` 不能直接访问 `Base::num`，但继承来的 `Base::setNum()` 仍然可以访问 `Base::num`。因此这个私有数据必须存在于派生类对象的基类部分中。

> [!warning] 易错点
> “不可直接访问”不是“不存在”。基类私有成员仍属于派生类对象的基类子对象，只是派生类代码不能用名字直接访问。

## protected 的意义

`protected` 的作用是让成员能沿继承体系向下传递，但不暴露给普通类外代码。

与 `public` 相比，`protected` 不向所有外部使用者公开，能保留一定信息隐蔽性。

与 `private` 相比，`protected` 又允许派生类访问，适合给子类扩展留下接口。

可以这样理解：

| 访问权限 | 类外可访问 | 派生类可访问 | 典型用途 |
|---|---|---|---|
| `public` | 可以 | 可以 | 对外接口 |
| `protected` | 不可以 | 可以 | 给继承体系内部使用 |
| `private` | 不可以 | 不可以直接访问 | 本类内部实现 |

## 本节小结

| 考点 | 必背结论 |
|---|---|
| 派生类成员来源 | 特殊函数、自定义成员、继承来的基类成员 |
| 基类特殊函数 | 构造、析构、拷贝构造、赋值函数不作为普通成员继承 |
| 派生类对象布局 | 基类子对象 + 派生类新增数据 |
| private 基类成员 | 派生类不可直接访问，但对象中仍存在 |
| protected 作用 | 允许沿继承体系访问，同时不向普通类外公开 |

> [!summary] 考前速记
> 派生类不仅有自己新增的成员，也有从基类继承来的成员。对象中包含基类子对象。访问控制表必须会背：public 成员随继承方式变 public/protected/private；protected 成员随继承方式变 protected/protected/private；private 成员在派生类中不可直接访问。
