# 15.4 派生类的构造和析构函数

派生类对象由[[基类子对象]]和派生类新增部分共同组成。因此构造派生类对象时，必须先把基类部分构造出来；析构时则按相反顺序销毁。

## 派生类也有自己的构造和析构

派生类的构造函数、析构函数属于派生类自己。如果用户没有提供，编译器会按规则生成默认版本。

基类中即使有构造函数、析构函数，也不会自动继承为派生类的构造函数、析构函数。

> [!warning] 易错点
> 派生类对象需要构造基类部分，但这不等于派生类“继承了基类构造函数”。正确说法是：派生类构造过程中会调用基类构造函数。

## 构造顺序

[[派生类构造顺序]] 是：

1. 先构造基类部分；
2. 再构造派生类自己的数据成员；
3. 最后执行派生类构造函数体。

```cpp
class Base {
public:
    explicit Base(int n) : number(n) {
        ++number;
        print();
    }

private:
    void print() const;
    int number;
};

class Derived : public Base {
public:
    Derived(int n1, int n2) : Base(n1), value(n2) {
    }

private:
    int value;
};
```

`Derived(int n1, int n2) : Base(n1), value(n2)` 中：

- `Base(n1)` 说明如何构造基类部分；
- `value(n2)` 说明如何构造派生类新增数据；
- 函数体 `{}` 执行时，基类部分和 `value` 已经完成初始化。

## 基类没有无参构造函数时必须显式调用

如果基类没有无参构造函数，而派生类构造函数初始化列表中又没有写如何构造基类，编译器会尝试调用基类无参构造函数。

找不到时就会编译错误。

```cpp
class Base {
public:
    explicit Base(int n);
};

class Derived : public Base {
public:
    Derived(int x) {
        // 错误：没有说明 Base 怎么构造，Base 又没有无参构造函数
    }
};
```

正确写法：

```cpp
class Derived : public Base {
public:
    Derived(int x) : Base(x) {
    }
};
```

> [!important] 高频考点
> 派生类构造函数初始化列表不仅可以初始化自己的成员，也可以指定基类构造函数。

## 多个基类的构造顺序

如果存在[[多重继承]]，多个基类的构造顺序由继承列表中的声明顺序决定，而不是由初始化列表中的书写顺序决定。

```cpp
class D : public B1, public B2 {
public:
    D() : B2(), B1() {
    }
};
```

实际仍先构造 `B1`，再构造 `B2`，因为继承列表写的是 `public B1, public B2`。

## 析构顺序

[[派生类析构顺序]] 与构造顺序相反：

1. 先执行派生类析构函数体；
2. 销毁派生类自己的成员；
3. 最后自动析构基类部分。

记忆方式：

```text
构造：基类 -> 派生类
析构：派生类 -> 基类
```

这样做的原因是：派生类析构函数执行时，派生类代码可能仍要依赖基类部分；所以基类部分必须最后销毁。

## 输出顺序例

假设：

- `Base(1)` 构造时把 `number` 从 1 加到 2，并输出 `2`；
- `Derived(1, 99)` 构造时初始化 `value = 99`；
- `Derived` 析构时输出 `99`；
- `Base` 析构时把 `number` 从 2 减到 1，并输出 `1`。

则局部对象离开作用域时，完整输出顺序是：

```text
2 99 1
```

其中 `2` 来自基类构造，`99` 来自派生类析构，`1` 来自基类析构。

## 本节小结

| 问题 | 必背结论 |
|---|---|
| 派生类是否有自己的构造/析构？ | 有，用户不写时编译器提供默认版本 |
| 基类构造函数是否被普通继承？ | 不会 |
| 派生类构造顺序 | 先基类，再派生类成员，再构造函数体 |
| 基类无无参构造怎么办？ | 派生类初始化列表必须显式调用基类构造函数 |
| 派生类析构顺序 | 先派生类，再基类 |

> [!summary] 考前速记
> 派生类对象构造时先构造基类部分，析构时最后析构基类部分。基类没有无参构造函数时，派生类构造函数必须在初始化列表中写 `Base(args)`。构造和析构顺序是继承章节的高频选择题、判断题考点。
