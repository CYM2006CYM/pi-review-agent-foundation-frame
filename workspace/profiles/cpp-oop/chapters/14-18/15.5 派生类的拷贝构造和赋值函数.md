# 15.5 派生类的拷贝构造和赋值函数

派生类对象由[[基类子对象]]和派生类新增部分组成。因此自定义[[派生类拷贝构造函数]]和[[派生类赋值函数]]时，不能只处理派生类自己的数据，也要正确处理基类部分。

## 编译器默认版本

如果用户不提供，编译器会为派生类提供默认拷贝构造函数和默认赋值函数。

默认行为通常是逐成员拷贝：

- 基类部分调用基类对应的拷贝构造或赋值函数；
- 派生类新增成员逐个拷贝或赋值。

如果类中只有简单值类型，例如 `int`，默认版本通常可以工作。若涉及动态资源、所有权、深拷贝、引用计数等，就可能需要自定义。

## 自定义拷贝构造函数

假设 `B` 公有继承自 `A`：

```cpp
class A {
public:
    A(int d = 0) : data(d) {}
    A(const A& other) : data(other.data) {}

private:
    int data;
};

class B : public A {
public:
    B(int d, int n) : A(d), number(n) {}

    B(const B& other) : A(other), number(other.number) {
    }

private:
    int number;
};
```

`B(const B& other) : A(other), number(other.number)` 的关键是 `A(other)`。

`other` 是一个 `B` 对象，但 `B` 对象开头包含 `A` 的基类子对象，所以可以用它来构造 `A` 部分。编译器会把其中的基类部分交给 `A` 的拷贝构造函数。

> [!important] 高频考点
> 派生类拷贝构造函数中，应在初始化列表中显式调用基类拷贝构造函数，例如 `A(other)`。

## 不建议直接改基类私有数据

有时初学者会想在派生类拷贝构造中直接写：

```cpp
data = other.data;
```

如果 `data` 是基类的 `private` 成员，这样写不合法。派生类不能直接访问基类私有数据。

即使当前例子中把 `data` 写成 `protected` 能访问，也不意味着这是最好的方式。更稳妥的做法是让基类自己负责复制自己的部分：

```cpp
B(const B& other) : A(other), number(other.number) {
}
```

这样即使以后 `A` 的内部数据改成 `private`，派生类也不需要直接依赖 `A` 的内部表示。

## 自定义赋值函数

派生类赋值函数需要处理两部分：

1. 调用基类赋值函数，完成基类部分赋值；
2. 赋值派生类自己新增的成员。

```cpp
class B : public A {
public:
    B& operator=(const B& other) {
        if (this != &other) {
            A::operator=(other);
            number = other.number;
        }
        return *this;
    }

private:
    int number;
};
```

这里 `A::operator=(other)` 调用的是基类 `A` 的赋值函数。虽然 `other` 是 `B` 对象，但它可以作为其中的 `A` 基类部分参与赋值。

## 自赋值检查仍然需要

自定义赋值函数通常要做[[自赋值检查]]：

```cpp
if (this != &other) {
    A::operator=(other);
    number = other.number;
}
```

对于只有 `int` 的简单例子，自赋值检查看起来影响不大；但一旦涉及动态资源，先释放再复制自身就可能造成严重错误。

## 拷贝构造与赋值的区别

| 项目 | 拷贝构造函数 | 赋值函数 |
|---|---|---|
| 对象状态 | 新对象尚未存在完整内容 | 左侧对象已经存在 |
| 基类部分处理 | 初始化列表中 `A(other)` | 函数体中 `A::operator=(other)` |
| 派生类新增成员 | 初始化列表或函数体中构造 | 函数体中赋值 |
| 是否返回对象 | 不返回 | 通常返回 `*this` |

## 本节小结

| 考点 | 必背结论 |
|---|---|
| 派生类拷贝构造 | 要构造基类部分和派生类新增部分 |
| 基类拷贝构造调用 | 在初始化列表写 `A(other)` |
| 派生类赋值函数 | 先自赋值检查，再调用 `A::operator=(other)`，再赋值新增成员 |
| 不要直接依赖基类私有数据 | private 成员不可直接访问，应让基类函数处理基类部分 |
| 默认版本是否一定够用 | 简单值类型通常够，资源管理类不一定够 |

> [!summary] 考前速记
> 派生类拷贝构造和赋值函数的核心是“别忘了基类部分”。拷贝构造用 `A(other)` 初始化基类子对象；赋值函数用 `A::operator=(other)` 赋值基类部分，然后再处理派生类新增成员。
