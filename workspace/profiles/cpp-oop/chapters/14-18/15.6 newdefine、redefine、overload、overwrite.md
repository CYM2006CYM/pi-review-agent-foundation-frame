# 15.6 newdefine、redefine、overload、overwrite

派生类中的成员函数来源较多：可以是派生类自己新定义的函数，也可以是从基类继承来的函数，还可能涉及静态成员函数、重载函数、虚函数。为了避免混淆，本节区分几个容易混用的术语。

## 五个术语总览

| 术语 | 中文理解 | 判断标准 |
|---|---|---|
| [[newdefine]] | 新定义 | 派生类定义了基类中没有同名函数的新函数 |
| [[redefine]] | 重定义 | 派生类定义了与基类同名同原型的普通函数 |
| [[函数重载]] overload | 重载 | 同一作用域内函数名相同、参数列表不同 |
| [[同名隐藏]] overwrite/hide | 隐藏 | 派生类定义同名函数后，隐藏基类所有同名函数 |
| [[override]] | 覆盖/重写 | 派生类重写基类虚函数 |

其中 `overwrite` 在课程中强调的是 `hide`，即名字隐藏；它不是后续 C++ 标准关键字 `override`。

## newdefine：派生类新增函数

如果派生类定义了一个基类中完全没有同名函数的成员函数，就是 newdefine。

```cpp
class A {
public:
    void f();
};

class B : public A {
public:
    void bf(); // newdefine
};
```

`bf` 在 `A` 中没有同名函数，所以它是 `B` 新增的成员函数。

## redefine：普通函数重定义

如果基类中已有某个普通成员函数，派生类中又定义了同名、同参数列表的函数，课程中称为 redefine。

```cpp
class A {
public:
    void g();
};

class B : public A {
public:
    void g(); // redefine
};
```

这里 `B::g()` 与 `A::g()` 函数原型相同，但函数体可以不同。它不是后续多态意义上的 override，因为没有强调虚函数机制。

## overload：函数重载

[[函数重载]] 指同名函数通过不同参数列表区分。

```cpp
void f();
void f(int);
void f(double);
```

重载在普通函数、成员函数中都常见。但在继承体系中，重载和名字查找会叠加出[[同名隐藏]]问题。

## overwrite/hide：同名隐藏

课程中的 `overwrite` 更准确理解为 hide：派生类中只要定义了某个与基类同名的函数，就会隐藏基类中所有同名函数，不要求参数列表相同。

```cpp
class A {
public:
    void f();
    void f(int);
};

class B : public A {
public:
    void f(double);

    void test() {
        // f();    // 基类 A::f() 被隐藏
        // f(10);  // 基类 A::f(int) 被隐藏
        f(3.14);   // 调用 B::f(double)
    }
};
```

这个行为容易违背直觉：很多人以为 `B::f(double)` 只是给 `f` 增加一个重载版本，但实际上它可能让基类已有的 `f()`、`f(int)` 在派生类作用域中不可直接访问。

如果确实要恢复基类同名函数，可以使用[[using声明]]：

```cpp
class B : public A {
public:
    using A::f;
    void f(double);
};
```

> [!warning] 易错点
> 同名隐藏只看名字，不看参数列表。派生类新增 `f(double)` 也可能隐藏基类的 `f()` 和 `f(int)`。

## override：重写虚函数

[[override]] 与[[虚函数]]有关。基类中有虚函数，派生类中给出相同原型的新实现，就构成 override。

```cpp
class A {
public:
    virtual void h();
};

class B : public A {
public:
    void h() override;
};
```

本节只做概念预告，后续讲虚函数和多态时会详细展开。

## 本节小结

| 易混点 | 正确理解 |
|---|---|
| newdefine vs redefine | newdefine 是基类没有同名函数；redefine 是基类已有同名同原型普通函数 |
| overload vs hide | overload 是同一作用域的重载；hide 是派生类同名函数隐藏基类同名函数 |
| redefine vs override | redefine 不强调虚函数；override 强调重写基类虚函数 |
| using 的作用 | 可把基类同名函数引入派生类作用域，缓解隐藏问题 |

> [!summary] 考前速记
> 派生类成员函数同名问题最容易考。派生类定义同名函数会隐藏基类所有同名函数，即使参数列表不同。`override` 要和虚函数一起理解；普通同名同原型函数更接近 redefine。
