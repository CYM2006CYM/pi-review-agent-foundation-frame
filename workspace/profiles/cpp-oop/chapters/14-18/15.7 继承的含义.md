# 15.7 继承的含义

继承不只是语法，也不是“能复用代码就用”。不同继承方式表达的设计含义不同，必须先理解含义，再决定是否使用继承。

## 黑盒复用与白盒复用的本质差异

前面说过：

- [[黑盒复用]]常通过[[关联]]、[[依赖]]、[[组合]]实现，强调[[功能复用]]；
- [[白盒复用]]常通过[[继承]]实现，强调代码和成员结构的复用。

二者没有绝对优劣，只有是否适合当前问题。

使用继承有一个前提：基类代码对派生类是可见、可用的。对于只提供二进制库、不给源代码的商品化组件，通常更适合用公开接口做黑盒复用，而不是依赖继承做白盒复用。

## 三种继承方式表达不同含义

同样写成继承，`public`、`private`、`protected` 表达的逻辑含义不同。

| 继承方式 | 设计含义 |
|---|---|
| [[公有继承]] | 表示 [[is-a关系]]，即“子类是一种父类” |
| [[私有继承]] | 更接近 [[has-a关系]] 或 implemented-in-terms-of，即“用基类实现自己” |
| [[保护继承]] | 接近私有继承，但允许保护成员沿继承层次继续传递 |

> [!important] 高频考点
> 不同继承方式不只是访问权限不同，更重要的是设计语义不同。

## public 继承：is-a

公有继承中，基类的 public 接口在派生类中仍然是 public。也就是说，派生类对象至少拥有基类对象对外承诺的公开行为。

因此公有继承应该表达：

```text
子类是一种父类
```

例如：

```text
手动挡轿车是一种轿车
自动挡轿车是一种轿车
```

可以写成：

```cpp
class Car {
public:
    void run();
    void brake();
};

class ManualCar : public Car {
public:
    void shiftGear();
};

class AutoCar : public Car {
public:
    void autoShift();
};
```

如果你无法用自然语言说出“派生类是一个基类”，就不应使用公有继承。语法能编译，不代表设计正确。

## public 继承的真正价值

公有继承的好处不只是复用代码，还在于类型替代能力：需要基类对象或引用的地方，可以使用派生类对象。

```cpp
void drive(Car& car) {
    car.run();
}

ManualCar mcar;
AutoCar acar;

drive(mcar);
drive(acar);
```

`drive` 需要的是 `Car&`，传入 `ManualCar` 或 `AutoCar` 都合理，因为它们都是一种 `Car`。

这种能力是后续[[多态]]和[[动态绑定]]的基础，也是公有继承最有价值的地方。

## private 继承：用基类实现自己

私有继承中，基类的 public 成员在派生类中变成 private。也就是说，基类公开接口不再作为派生类的公开接口暴露给外部。

这时两个类通常不再满足 is-a。

例如：

```cpp
class Bike {
public:
    void move();
    void stop();
    void repair();
};

class Player : private Bike {
public:
    void startRace() {
        move();
    }

    void endRace() {
        stop();
    }
};
```

`Player` 不是一种 `Bike`。这里只是 `Player` 的实现中借用了 `Bike` 的已有代码。

所以私有继承的含义更接近：

```text
Player 用 Bike 来实现某些功能
Player 有一个 Bike
```

这与[[组合]]非常接近。

## protected 继承：让实现沿继承树传递

保护继承的外部接口表现与私有继承相近：基类的 public 接口不会继续公开为派生类的 public 接口。

它的特殊价值在于：基类 public 和 protected 成员会变成派生类 protected 成员，可以沿后续继承层次继续传递。

也就是说，保护继承可以让某些基类功能在继承树内部继续可用，但不暴露给普通类外代码。

不过它仍然不表示典型 is-a 关系，使用时要谨慎。

## 本节小结

| 问题 | 必背结论 |
|---|---|
| 公有继承表达什么？ | 子类是一种父类，即 is-a |
| 什么时候用公有继承？ | 能自然说出“派生类是一个基类” |
| 私有继承表达什么？ | 用基类实现自己，接近 has-a |
| 保护继承的特点 | 类似私有继承，但保护成员可继续沿继承树传递 |
| 继承的主要价值 | 公有继承带来类型替代和后续多态基础 |

> [!summary] 考前速记
> 公有继承才是继承中最重要的设计形式，它表达 is-a。私有继承和保护继承更像实现复用，不适合随便理解成父类/子类关系。判断公有继承是否正确，先用自然语言说：“子类是一个父类。”
