# 16.1 继承下的类型转换

本节讨论[[继承]]关系中的[[类型转换]]。继承把两个类放到同一条类层次中，因此类型转换不再只是内置类型之间的转换，也会发生在[[基类]]和[[派生类]]之间。

## 两个方向

在继承体系中，类型转换有两个基本方向：

| 方向 | 含义 | 常用英文 |
|---|---|---|
| [[向上类型转换]] | 派生类转成基类 | upcast |
| [[向下类型转换]] | 基类转成派生类 | downcast |

假设有：

```cpp
class Base {
};

class Derived : public Base {
};
```

从 `Derived` 到 `Base` 是向上类型转换；从 `Base` 到 `Derived` 是向下类型转换。

## 为什么继承方式会影响转换意义

继承方式不只是语法，它还表达类之间的逻辑关系。

| 继承方式 | 公有接口关系 | 类型转换意义 |
|---|---|---|
| [[公有继承]] | 派生类保留基类 public 行为 | 向上转换通常有意义 |
| [[保护继承]] | 基类 public 行为变为 protected | 对外不再保持同一种公共接口 |
| [[私有继承]] | 基类 public 行为变为 private | 对外不再表示 is-a |

[[私有继承]]和[[保护继承]]更多表示实现复用，而不是“派生类是一种基类”。因此在这两种继承方式下，基类和派生类的公共接口不再一致，直接进行基类与派生类之间的类型转换通常没有实际设计意义。

## private/protected 继承下的转换

以 `Player` 私有继承 `Bike` 为例：

```cpp
class Bike {
public:
    void move();
    void stop();
    void repair();
protected:
    int changeColor(int);
private:
    int mColor;
};

class Player : private Bike {
public:
    void startRace();
    void endRace();
private:
    int mMaxStrength;
    int mAge;
};
```

`Player` 内部可以复用 `Bike` 的功能，但不能说“运动员是一辆自行车”。它们的 public 接口完全不同，所以把 `Player*` 当作 `Bike*` 使用不符合对象模型的含义。

这种场景如果确实需要转换，通常应当显式设计转换方式：

```cpp
class Derived : private Base {
public:
    operator Base() const;          // 转换函数
    Derived(const Base& other);     // 转换构造函数
};
```

但这已经不是继承天然带来的 is-a 转换，而是程序员额外定义的转换语义。

## public 继承下的转换

[[公有继承]]表达 [[is-a关系]]。如果 `ManualCar` 公有继承 `Car`，那么“手动挡汽车是一种汽车”成立。

```cpp
class Car {
public:
    void run();
    void brake();
};

class MTCar : public Car {
public:
    void funcMT();
};
```

在这种关系中，从 `MTCar` 转换到 `Car` 是自然的：一个更具体的对象可以当作更一般的对象使用。

## 向下转换为什么要谨慎

从基类到派生类的转换不是总能成立。

```text
Animal 可以指向 Dog
Animal 也可以指向 Cat
Animal 还可能只是 Animal
```

因此把一个 `Animal*` 转换成 `Dog*` 时，必须知道它运行时实际指向的对象确实是 `Dog`。否则转换后的指针或引用就会表达错误对象，使用它会带来危险。

这也是后面要学习 [[dynamic_cast]] 和 [[RTTI]] 的原因：向下转换需要运行时确认对象的真实类型。

## 本节小结

| 问题 | 结论 |
|---|---|
| 继承下类型转换有几个方向 | 向上转换和向下转换 |
| private/protected 继承适合自动转换吗 | 通常不适合，因为对外接口不再表示 is-a |
| public 继承为什么适合向上转换 | 派生类是一种基类，公共行为兼容 |
| 向下转换为什么危险 | 基类对象未必真的是目标派生类对象 |

> [!summary] 考前速记
> 继承下的类型转换要先看继承含义。public 继承表达 is-a，派生类向基类转换通常安全；private/protected 继承主要是实现复用，转换意义弱。基类向派生类转换必须确认真实对象类型，不能只凭静态类型判断。
