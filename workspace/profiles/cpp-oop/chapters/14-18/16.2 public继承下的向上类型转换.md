# 16.2 public继承下的向上类型转换

本节详细分析 [[公有继承]] 下的[[向上类型转换]]。核心结论是：派生类对象可以安全地当作基类对象使用，但不同转换形式的代价不同。

## public 继承的语义

public 继承表示 [[is-a关系]]：

```text
派生类对象是一种基类对象
```

例如：

```cpp
class Car {
public:
    void run();
    void brake();
protected:
    int addWeight(int);
    int reduceWeight(int);
private:
    int mWeight;
    int mWheels;
};

class MTCar : public Car {
public:
    void funcMT();
};
```

`MTCar` 是更具体的汽车，`Car` 是更一般的汽车类型。因此 `MTCar` 转成 `Car` 在逻辑上成立。

## 形式一：派生类指针转基类指针

```cpp
MTCar mtCar;
Car* pc = &mtCar;
```

`pc` 的静态类型是 `Car*`，但它指向的实际对象仍然是 `MTCar` 对象中的[[基类子对象]]部分。

这种转换安全，原因是：

- 每个 `MTCar` 对象内部都包含一个 `Car` 基类部分；
- `Car*` 只要求能访问 `Car` 的接口；
- `MTCar` 作为 `Car` 使用时，基类接口一定存在。

这也是多态设计中常见的写法：

```cpp
Car* car = new MTCar;
car->run();
```

## 形式二：派生类引用转基类引用

```cpp
MTCar mtCar;
Car& rc = mtCar;
```

引用本质上可以理解为对象的别名。从效果上看，`rc` 是 `mtCar` 中 `Car` 基类部分的别名。

这种转换同样安全，并且不会复制对象，也不会丢失派生类对象本身。

```cpp
void drive(Car& car) {
    car.run();
}

MTCar mtCar;
drive(mtCar);
```

函数参数写成 `Car&`，调用时可以传入任何公有继承自 `Car` 的对象。这让函数面向更一般的类型编程。

## 形式三：派生类对象转基类对象

对象形式也可以向上转换：

```cpp
MTCar mtCar;
Car c = mtCar;
```

这会创建一个新的 `Car` 对象，只把 `mtCar` 中的 `Car` 基类部分复制过去。派生类自己新增的数据成员会被舍弃，这个现象称为[[对象切片]]。

```text
MTCar 对象 = Car 部分 + MTCar 自己的部分
Car c = mtCar 只保留 Car 部分
```

对象形式的转换在语法上安全，但会丢掉派生类部分的信息，因此实际代码中通常更推荐使用指针或引用。

## 三种形式对比

| 转换形式 | 是否安全 | 是否复制对象 | 是否可能丢失派生类部分 |
|---|---|---|---|
| `Car* pc = &mtCar;` | 安全 | 否 | 否 |
| `Car& rc = mtCar;` | 安全 | 否 | 否 |
| `Car c = mtCar;` | 安全 | 是 | 是，发生对象切片 |

## 向上转换的两个常见用途

### 作为数据成员保存

类可以保存基类指针，从而关联某个基类对象或任意派生类对象：

```cpp
class Example {
public:
    Example() : mp(new Child) {
    }

    ~Example() {
        delete mp;
    }

private:
    Parent* mp;
};
```

`mp` 的类型是 `Parent*`，但可以指向 `Parent` 的任意派生类对象。这种写法让对象关系面向抽象的父类。

### 作为函数参数接收

函数参数写成基类引用，可以接收任何派生类对象：

```cpp
void func(Parent& object) {
    object.doFunc();
}
```

只要传入对象是 `Parent` 或其公有派生类，就可以调用。以后新增新的派生类时，`func` 不必修改。

## 本节小结

public 继承下，向上类型转换体现了“更具体类型可以当作更一般类型使用”。

| 重点 | 结论 |
|---|---|
| 指针向上转换 | 安全且不复制对象 |
| 引用向上转换 | 安全且不复制对象 |
| 对象向上转换 | 安全但会对象切片 |
| 实际推荐 | 多用基类指针或引用，少用对象形式转换 |

> [!summary] 考前速记
> public 继承下，派生类向基类的指针、引用、对象转换都安全；指针和引用不丢数据，对象转换会裁剪派生类部分。考试中看到 `Base b = derived;` 要立刻想到对象切片。
