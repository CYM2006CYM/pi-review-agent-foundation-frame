# 16.3 类型转换操作符

本节总结 C++ 中的类型转换方式，重点介绍四个显式类型转换操作符：[[static_cast]]、[[const_cast]]、[[reinterpret_cast]]、[[dynamic_cast]]。

## C++ 中的类型转换方式

C++ 中常见类型转换包括：

| 转换方式 | 说明 |
|---|---|
| 内置类型自动转换 | 如 `int` 到 `float` |
| [[转换构造函数]] | 用一个类型的对象构造另一个类型对象 |
| [[转换函数]] | 类中定义 `operator T()` |
| public 继承下的向上转换 | 派生类到基类 |
| 显式转换操作符 | `static_cast`、`const_cast`、`reinterpret_cast`、`dynamic_cast` |

旧式 C 风格强制转换写法短，但意图不清：

```cpp
int n = (int)x;
```

C++ 提供的四个转换操作符把不同风险的转换拆开，让读代码的人能看出程序员到底想做哪一种转换。

## static_cast：静态转换

`static_cast` 用于编译期可判断的类型转换。

```cpp
double d = 3.14;
int n = static_cast<int>(d);
```

它常用于：

- 内置类型之间的显式转换；
- 有明确转换规则的类类型转换；
- public 继承中的向上转换；
- 部分需要程序员确认的向下转换。

`static_cast` 比 C 风格转换更清楚，因为它直接表达“这里做的是静态类型转换”。

## const_cast：添加或去除 const/volatile

`const_cast` 只处理类型限定符，主要是 `const` 和 `volatile`。

```cpp
const A& ca = a;
A& ra = const_cast<A&>(ca);
```

`ca` 是常引用，不能通过它修改对象；`ra` 去掉了常限定，之后就可以通过 `ra` 调用非常成员函数或修改对象。

需要注意：

- 如果原始对象本来就是非常对象，去掉 `const` 后修改通常能工作；
- 如果原始对象本身就是常对象，强行修改会产生未定义行为；
- `const_cast` 破坏了 `const` 的约束，应当慎用。

### volatile 的含义

`volatile` 表示对象的值可能在程序控制之外随时改变。编译器不能随意把多次读取优化成一次读取。

```cpp
volatile int k;
return k * k;
```

两次读取 `k` 的值可能不同。因此 `volatile` 常见于底层、硬件寄存器或并发相关场景。本课程只要求理解它是 `const_cast` 可以处理的限定符之一。

## reinterpret_cast：重新解释

`reinterpret_cast` 用于把一段数据或地址按另一种类型重新解释。

```cpp
using FuncType = void (*)(char*);

FuncType f = reinterpret_cast<FuncType>(myFunc);
```

它常见于：

- 指针类型之间的低层转换；
- 函数指针类型的重新解释；
- 与底层接口、系统接口打交道。

这种转换风险最高，因为它不保证转换后的类型语义正确，只是要求编译器按目标类型解释原表达式。普通面向对象代码中应尽量少用。

## dynamic_cast：动态转换

`dynamic_cast` 在运行时检查对象真实类型，最典型用途是[[向下类型转换]]。

```cpp
class Animal {
public:
    virtual ~Animal() = default;
};

class Dog : public Animal {
};

Animal* p = new Dog;
Dog* dog = dynamic_cast<Dog*>(p);
```

基类指针 `p` 的静态类型是 `Animal*`，但它实际可能指向 `Dog`、`Cat` 或其他派生类对象。能否转成 `Dog*`，只有运行时才知道。

### 指针形式

```cpp
Dog* dog = dynamic_cast<Dog*>(p);
if (dog != nullptr) {
    // 转换成功
}
```

指针形式转换失败时返回空指针，因此可以用 `nullptr` 判断是否成功。

### 引用形式

```cpp
Dog& dog = dynamic_cast<Dog&>(animal);
```

引用必须绑定到真实对象，不能像指针那样用空值表示失败。因此引用形式转换失败时会抛出异常。

## dynamic_cast 与 RTTI

`dynamic_cast` 属于[[RTTI]]，也就是运行时类型识别。它通常依赖类中的[[虚函数表]]或相关运行时类型信息实现。

因此用于 `dynamic_cast` 的基类通常需要至少有一个[[虚函数]]，常见做法是给多态基类定义虚析构函数：

```cpp
class Base {
public:
    virtual ~Base() = default;
};
```

## 四个转换操作符对比

| 操作符 | 主要用途 | 风险 |
|---|---|---|
| [[static_cast]] | 编译期明确的转换 | 中 |
| [[const_cast]] | 添加或去除 `const`/`volatile` | 中高 |
| [[reinterpret_cast]] | 低层重新解释 | 高 |
| [[dynamic_cast]] | 运行时安全向下转换 | 较低，但有运行时开销 |

## 本节小结

| 问题 | 结论 |
|---|---|
| 为什么不用 C 风格转换 | 意图不清，风险混在一起 |
| `static_cast` 什么时候用 | 编译期能确认规则的转换 |
| `const_cast` 能改变什么 | 只能改变 `const`/`volatile` 限定 |
| `reinterpret_cast` 的特点 | 重新解释底层表示，风险很高 |
| `dynamic_cast` 的特点 | 运行时检查真实类型，常用于安全向下转换 |

> [!summary] 考前速记
> 四个转换操作符要按用途记：`static_cast` 静态转换，`const_cast` 改限定符，`reinterpret_cast` 重新解释，`dynamic_cast` 运行时检查。向下转换优先想到 `dynamic_cast`，指针失败返回空，引用失败抛异常。
