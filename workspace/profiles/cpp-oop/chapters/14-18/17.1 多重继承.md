# 17.1 多重继承

本节介绍[[多重继承]]的基本语法、构造析构顺序和对象存储结构。

## 什么是多重继承

一个派生类只有一个直接基类，称为[[单继承]]；一个派生类有两个或更多直接基类，称为[[多重继承]]。

```cpp
class Derived : public Base1, protected Base2, private Base3 {
public:
    // 派生类自己的成员
};
```

多重继承和单继承的主要差别在继承列表：继承列表中出现多个基类。

## 多重继承的直观用途

例如：

```cpp
class Teacher {
public:
    void teach();
};

class Student {
public:
    void study();
};

class InServiceDoctor : public Teacher, public Student {
public:
    void doResearch();
};
```

`InServiceDoctor` 同时具有教师和学生的行为，又可以增加自己的行为。这说明多重继承可以快速组合多个已有类型的公共行为。

但这种方便会带来两个重要问题：

- [[多重继承名字冲突]]；
- [[菱形结构]]导致共同基类重复出现。

## 构造顺序

多重继承下，构造派生类对象时，先构造所有直接基类，再构造派生类自己的成员和构造函数体。

基类构造顺序严格由继承列表中的声明顺序决定，而不是由构造函数初始化列表的书写顺序决定。

```cpp
class D : public B1, public B2 {
public:
    D() : B2(), B1() {
    }
};
```

即使初始化列表写成 `B2(), B1()`，实际仍先构造 `B1`，再构造 `B2`，最后执行 `D` 的构造函数体。

## 析构顺序

析构顺序与构造顺序相反：

```text
先析构 D 自己
再按继承列表逆序析构 B2、B1
```

这是对象生命周期的基本原则：先构造的部分后析构，后构造的部分先析构。

## 对象存储结构

多重继承的派生类对象中，会包含多个基类子对象。

```text
D 对象
├─ B1 基类子对象
├─ B2 基类子对象
└─ D 自己的数据成员
```

对象中存放的是各个类的[[实例变量]]，不直接存放成员函数。成员函数代码通常位于代码区，所有对象共享同一份函数代码。

如果类涉及[[虚函数]]，对象中还可能包含[[虚函数表指针]]，后续章节会详细讨论。

## 本节小结

| 主题 | 结论 |
|---|---|
| 多重继承语法 | 继承列表中列出多个基类 |
| 构造顺序 | 按继承列表顺序构造基类，再构造派生类 |
| 析构顺序 | 与构造顺序相反 |
| 对象布局 | 含多个基类子对象和派生类自己的数据 |
| 主要风险 | 名字冲突、菱形结构、对象布局复杂 |

> [!summary] 考前速记
> 多重继承不是难在语法，而是难在二义性和对象布局。构造顺序看继承列表，析构顺序反过来；对象中会保存每个直接基类的数据部分。
