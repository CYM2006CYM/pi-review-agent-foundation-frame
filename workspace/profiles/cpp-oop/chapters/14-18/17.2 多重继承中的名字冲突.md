# 17.2 多重继承中的名字冲突

本节讨论[[多重继承]]中最直接的问题：不同基类中存在同名成员时，派生类访问这些成员会出现二义性。

## 同名成员导致二义性

```cpp
class A {
public:
    void f();
protected:
    int number;
};

class B {
public:
    void f();
protected:
    int number;
};

class C : public A, public B {
public:
    void k() {
        f();           // 错误：不知道调用 A::f 还是 B::f
        number = 10;   // 错误：不知道访问 A::number 还是 B::number
    }
};
```

`C` 同时从 `A` 和 `B` 继承了 `f` 和 `number`。当代码只写 `f()` 或 `number` 时，编译器无法判断应该选择哪一个。

这种错误本质上是[[二义性]]：程序语义不唯一。

## 方法一：作用域限定

最直接的解决方式是明确指定来源：

```cpp
class C : public A, public B {
public:
    void k() {
        A::f();
        B::number = 10;
    }
};
```

`A::f()` 表示调用从 `A` 继承来的 `f`，`B::number` 表示访问从 `B` 继承来的 `number`。

这种方法语义明确，但要求使用者知道每个成员来自哪个基类。如果复用者必须读完整个基类源码才能避免冲突，复用成本会很高。

## 方法二：using 声明

可以在派生类中用 `using` 把某个基类成员引入当前作用域：

```cpp
class C : public A, public B {
public:
    using A::f;
    using B::number;

    void k() {
        f();           // 使用 A::f
        number = 10;   // 使用 B::number
    }
};
```

`using A::f;` 的含义是把 `A` 中名为 `f` 的成员引入 `C` 的作用域，从而让未限定的 `f()` 有明确选择。

## using 的局限

`using` 可以消除使用时的二义性，但并没有从对象结构上消除重复成员。

对于数据成员来说，`C` 对象中仍然同时存在：

```text
C 对象
├─ A::number
├─ B::number
└─ C 自己的数据
```

也就是说，`using B::number` 只是规定默认访问哪一个 `number`，并不会删掉另一个 `number`。

这也是课程强调的重点：函数名冲突相对好处理，数据成员冲突更麻烦，因为数据成员真实占用对象空间，并会继续影响后续派生类和关联使用。

## 本节小结

| 解决方式 | 作用 | 局限 |
|---|---|---|
| `A::f()` | 每次访问时明确来源 | 使用处写法繁琐，要求知道继承细节 |
| `using A::f;` | 在类中统一指定默认来源 | 不能消除对象中重复的数据成员 |

> [!summary] 考前速记
> 多重继承名字冲突主要考二义性。函数和数据成员都可能冲突，`A::f()` 可以显式限定，`using A::f` 可以汇入名字；但 `using` 不会改变对象里实际存在的多个基类数据成员。
