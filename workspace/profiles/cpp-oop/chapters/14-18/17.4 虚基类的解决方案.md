# 17.4 虚基类的解决方案

本节介绍 C++ 对[[菱形结构]]的一种语言级解决方案：[[虚基类]]。

## 虚基类语法

在中间层继承共同基类时，加上 `virtual`：

```cpp
class A {
public:
    void fA();
private:
    int mA;
};

class B : virtual public A {
private:
    int mB;
};

class C : virtual public A {
private:
    int mC;
};

class D : public B, public C {
private:
    int mD;
};
```

`virtual public A` 表示 `A` 作为虚基类被继承。这样 `D` 中经 `B` 和 `C` 到达的 `A` 会合并为一份。

## 虚基类解决了什么

未使用虚基类时：

```text
D 对象
├─ B 子对象
│  └─ A 子对象
├─ C 子对象
│  └─ A 子对象
└─ D 自己的数据
```

使用虚基类后：

```text
D 对象
├─ B 子对象
│  └─ 指向共享 A 的信息
├─ C 子对象
│  └─ 指向共享 A 的信息
├─ D 自己的数据
└─ 共享的 A 子对象
```

最终派生类 `D` 只保留一份 `A` 子对象，因此：

```cpp
D d;
d.fA();   // 不再因为两份 A 而二义
```

## 对象布局的代价

虚基类通常需要额外指针或类似的运行时/编译器辅助信息，用来定位共享的虚基类子对象。

这会带来两个代价：

- 对象布局比普通继承复杂；
- 派生类向基类的类型转换可能需要调整指针位置，效率低于普通继承。

普通 public 继承中，向上转换往往只是把派生类指针看作基类指针；虚基类场景下，共享基类部分不一定在对象最前面，转换时可能需要计算偏移。

## 设计上的缺点

虚基类还有一个更重要的设计问题：它要求中间层类提前预判未来会不会参与菱形结构。

例如 `B` 的作者写：

```cpp
class B : virtual public A {
};
```

这意味着 `B` 的作者必须想到未来可能出现：

```cpp
class D : public B, public C {
};
```

但在实际复用中，`B` 的作者未必知道以后会有哪些类共同继承它。为了未来某个可能的 `D`，要求 `B` 和 `C` 在定义时都写虚继承，这对类设计者要求很高。

## 本节小结

| 方面 | 结论 |
|---|---|
| 语法 | `class B : virtual public A` |
| 解决的问题 | 共同基类在最终派生类中只保留一份 |
| 对象布局 | 需要额外定位共享虚基类子对象 |
| 转换成本 | 可能需要指针调整，效率低于普通继承 |
| 设计问题 | 要求中间层类预判未来继承结构 |

> [!summary] 考前速记
> 虚基类能解决菱形结构中共同基类重复的问题，关键字写在中间层继承共同基类的位置：`B : virtual public A`、`C : virtual public A`。但它带来对象布局复杂、转换效率下降和设计预判成本。
