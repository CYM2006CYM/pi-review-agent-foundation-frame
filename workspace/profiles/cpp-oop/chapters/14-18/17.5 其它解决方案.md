# 17.5 其它解决方案

本节讨论除[[虚基类]]之外，语言和设计层面对多重继承问题的其他解决思路。

## 方案一：只允许单继承

最直接的方案是语言层面禁止多重继承，只允许一个直接基类。

优点：

- 不会出现多重继承名字冲突；
- 不会出现菱形结构；
- 对象布局简单。

缺点：

- 类型复用能力受限制；
- 一个类很难同时表达多种“可作为某接口使用”的身份。

## 方案二：多个基类中至多一个有实例变量

更常见的现代做法是：允许一个类继承多个基类，但多个基类中至多只有一个基类有[[实例变量]]，其余基类只提供行为接口或静态工具。

```cpp
class Person {
public:
    void work();
private:
    int salary;
};

class Utility {
public:
    static void doIt();
};

class MyException {
};

class SpecialPerson : public Person, public Utility, public MyException {
};
```

`Person` 有实例数据；`Utility` 和 `MyException` 没有实例数据。这样对象中真正需要保存的数据主要来自一个有状态基类，可以避免多个有状态基类造成的数据成员冲突。

## 为什么关注实例变量

函数名冲突可以通过作用域限定或 `using` 缓解；数据成员冲突更麻烦，因为数据真实占对象空间。

因此限制“只有一个基类有实例变量”的核心目的，是避免对象中出现多个来源复杂、命名冲突的数据部分。

静态数据成员不属于对象实例的一部分，所以不算实例变量。

## 无实例变量类的用途一：类型标记

有些类没有数据，只用类型名称表达语义。

```cpp
class NormalException {
};

class SpecialException {
};

void f(bool condition) {
    if (condition) {
        throw NormalException();
    } else {
        throw SpecialException();
    }
}
```

比起 `throw 1`、`throw 2`，抛出不同类型的对象更能表达语义。这里类的主要价值不是存数据，而是用类型区分不同情况。

## 无实例变量类的用途二：工具类

[[工具类]]通常只包含静态成员函数和静态数据成员：

```cpp
class PackUtility {
public:
    static void packBinary();
    static void packText();
    static void packPicture();
    static int getVersion();
private:
    static int version;
};
```

这些静态成员函数类似 C 语言中的全局函数，只是被放进类作用域中统一管理。工具类通常不需要实例变量。

## 无实例变量类的用途三：接口类

[[接口类]]描述一组公共行为，而不是保存对象状态。

```cpp
class IComputer {
public:
    virtual ~IComputer() = default;
    virtual void runApp() = 0;
    virtual void showVideo() = 0;
};

class IPaper {
public:
    virtual ~IPaper() = default;
    virtual void writeText() = 0;
};

class Telephone {
public:
    void callTo();
    void callBy();
};

class Mobile : public Telephone, public IComputer, public IPaper {
public:
    void runApp() override;
    void showVideo() override;
    void writeText() override;
};
```

`Mobile` 是一种 `Telephone`，同时实现 `IComputer` 和 `IPaper` 两组接口。它的主要状态可以来自 `Telephone` 或自己，而接口类只规定“必须能做什么”。

## 接口继承的意义

[[接口继承]]把“状态继承”和“行为契约”分开：

- 有状态基类负责对象数据和主要身份；
- 接口类只规定公共行为；
- 多个接口可以组合，不容易造成对象数据冲突。

这也是许多现代面向对象语言采用的思路：类只能单继承实现，但可以实现多个接口。

## 本节小结

| 方案 | 思路 | 评价 |
|---|---|---|
| 只允许单继承 | 从语言层面消除问题 | 简单但不灵活 |
| 虚基类 | C++ 语言级解决菱形结构 | 能解决重复基类，但代价较高 |
| 一个有状态基类 + 多个接口类 | 保留多身份，避免多份数据 | 更符合现代设计习惯 |

> [!summary] 考前速记
> 多重继承问题不一定非靠虚基类解决。现代设计常用“一个有实例变量的基类 + 多个无实例变量接口类”。接口类定义公共行为，不保存实例数据；工具类放静态函数；类型标记类用类型名表达语义。
