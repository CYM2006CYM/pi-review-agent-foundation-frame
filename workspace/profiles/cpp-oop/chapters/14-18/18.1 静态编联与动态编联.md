# 18.1 静态编联与动态编联

本节从一个图形面积的例子引出[[静态编联]]和[[动态编联]]。这是理解[[虚函数]]和[[虚机制]]的入口。

## 问题：基类接口调用了错误实现

假设有基类 `Shape`，以及两个派生类 `Rectangle` 和 `Circle`。

```cpp
class Shape {
public:
    void show() {
        cout << "area = " << area() << endl;
    }

    float area() {
        return 0;
    }
};

class Rectangle : public Shape {
public:
    Rectangle(float w, float h) : width(w), height(h) {
    }

    float area() {
        return width * height;
    }

private:
    float width;
    float height;
};

class Circle : public Shape {
public:
    Circle(float r) : radius(r) {
    }

    float area() {
        return 3.14f * radius * radius;
    }

private:
    float radius;
};
```

如果把派生类对象当作基类引用使用：

```cpp
Rectangle rect(1, 2);
Circle circle(1);

Shape& sp1 = rect;
Shape& sp2 = circle;

sp1.show();
sp2.show();
```

从设计意图看，希望输出矩形面积 `2` 和圆面积 `3.14`。但如果 `area()` 不是虚函数，`show()` 内部调用的仍然是 `Shape::area()`，结果会输出 `0`。

## 静态编联

[[静态编联]]也称静态绑定、早绑定，指编译期间就确定函数调用关系。

```text
编译期看到 this 的静态类型是 Shape*
所以 area() 被绑定为 Shape::area()
```

在普通成员函数调用中，编译器根据表达式的静态类型决定调用哪个函数。`Shape::show()` 中的 `area()` 等价于：

```cpp
this->area();
```

而 `this` 在 `Shape::show()` 中的静态类型是 `Shape*`，所以静态编联会选择 `Shape::area()`。

## 动态编联

[[动态编联]]也称动态绑定、晚绑定，指运行时根据对象的真实类型决定调用哪个函数。

对于上面的例子，动态编联希望：

```text
this 实际指向 Rectangle 对象 -> 调用 Rectangle::area()
this 实际指向 Circle 对象 -> 调用 Circle::area()
```

这种能力正是[[多态]]的基础：同样调用 `area()`，不同实际对象表现出不同实现。

## 为什么需要虚机制

静态编联只能按编译期类型处理，不能解决“基类引用实际绑定到哪个派生类对象”的问题。

要让程序在运行时根据对象真实类型选择函数，需要语言和编译器提供额外机制。C++ 的常见实现就是[[虚机制]]，程序员通过声明[[虚函数]]来启用它。

## 本节小结

| 概念 | 决定时机 | 决定依据 | 典型结果 |
|---|---|---|---|
| [[静态编联]] | 编译期 | 表达式静态类型 | `Shape::area()` |
| [[动态编联]] | 运行期 | 对象真实类型 | `Rectangle::area()` 或 `Circle::area()` |

> [!summary] 考前速记
> 静态编联看“变量声明的类型”，动态编联看“对象实际的类型”。如果基类函数内部调用希望随派生类变化，就要把被调用函数设计成虚函数。
