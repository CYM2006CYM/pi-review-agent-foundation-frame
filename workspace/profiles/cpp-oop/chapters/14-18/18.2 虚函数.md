# 18.2 虚函数

本节介绍 [[虚函数]] 的声明格式、适用范围、虚析构函数、派生类中的虚函数规则，以及 [[override]]。

## 虚函数的基本写法

在成员函数声明前加 `virtual`：

```cpp
class Shape {
public:
    void show() {
        cout << "area = " << area() << endl;
    }

    virtual float area() {
        return 0;
    }
};
```

派生类给出同名同参数的实现：

```cpp
class Rectangle : public Shape {
public:
    float area() override {
        return width * height;
    }
private:
    float width;
    float height;
};
```

当基类引用或指针实际绑定到 `Rectangle` 对象时，调用 `area()` 会在运行时选择 `Rectangle::area()`。

## 虚函数必须是成员函数

虚函数一定是成员函数，而且是实例方法。

不能成为虚函数的成员包括：

| 成员 | 原因 |
|---|---|
| 静态成员函数 | 没有 `this` 指针，不依附具体对象 |
| 构造函数 | 对象尚未完整形成，无法按最终动态类型派发 |
| 拷贝构造函数 | 同样属于构造阶段 |

[[析构函数]]可以是虚函数，而且含有虚函数的类通常应该定义[[虚析构函数]]。

## 为什么需要虚析构函数

如果一个类用于多态，并可能通过基类指针删除派生类对象：

```cpp
Shape* p = new Circle(3);
delete p;
```

基类析构函数应当是虚的：

```cpp
class Shape {
public:
    virtual ~Shape() = default;
    virtual float area() = 0;
};
```

这样 `delete p` 时会先调用 `Circle` 的析构函数，再调用 `Shape` 的析构函数，资源释放顺序才正确。

## 派生类中的虚函数

如果基类中的函数已经是虚函数，派生类中同原型函数即使不写 `virtual`，仍然是虚函数。

```cpp
class Base {
public:
    virtual void f();
};

class Derived : public Base {
public:
    void f();   // 仍然是虚函数
};
```

实际工程中建议显式写 `override`：

```cpp
class Derived : public Base {
public:
    void f() override;
};
```

这样如果函数签名不匹配，编译器会报错。

## override 的条件

派生类函数要构成 `override`，通常需要满足：

- 基类函数是虚函数；
- 函数名相同；
- 参数列表相同；
- `const` 等函数限定匹配；
- 返回类型相同，或满足协变返回类型规则。

协变返回类型的典型情况是：基类虚函数返回 `Base*`，派生类重写时返回 `Derived*`。

```cpp
class Parent {
public:
    virtual Parent* clone();
};

class Child : public Parent {
public:
    Child* clone() override;
};
```

`Child*` 可以安全向上转换为 `Parent*`，所以这类返回类型相容。

## override 与同名隐藏

如果派生类定义了同名函数，但参数列表不同，它不是重写，而是[[同名隐藏]]。

```cpp
class Base {
public:
    virtual void f(int);
    virtual void f(double);
};

class Derived : public Base {
public:
    void f();   // 隐藏 Base 中所有 f，不是 override
};
```

这会让 `Base::f(int)` 和 `Base::f(double)` 在 `Derived` 的未限定查找中被隐藏。需要时可用：

```cpp
class Derived : public Base {
public:
    using Base::f;
    void f();
};
```

## 本节小结

| 问题 | 结论 |
|---|---|
| 虚函数关键字 | `virtual` |
| 虚函数本质 | 让调用支持动态编联 |
| 静态函数能否 virtual | 不能 |
| 构造函数能否 virtual | 不能 |
| 析构函数能否 virtual | 可以，多态基类通常应当 virtual |
| 派生类是否必须重复写 virtual | 不必须，但建议写 `override` |

> [!summary] 考前速记
> 虚函数一定是实例成员函数。构造函数不能虚，析构函数可以虚；只要类用于多态，析构函数通常就应该虚。派生类重写虚函数时最好写 `override`，避免把重写误写成同名隐藏。
