# 18.3 虚函数表

本节解释 [[虚函数表]] 和 [[虚函数表指针]]，说明 C++ 常见编译器如何用它们实现[[动态编联]]。

## 虚函数表是什么

虚函数表可以理解为一个函数指针数组。

```text
虚函数表
├─ 指向虚析构函数的入口地址
├─ 指向虚函数 f 的入口地址
└─ 指向虚函数 k 的入口地址
```

一个类只要有至少一个虚函数，就通常会有一张虚函数表。虚函数可以是自己定义的，也可以是从基类继承来的。

## 表项保存什么

虚函数表中的每一项保存一个函数入口地址，也就是对应虚函数代码的位置。

```text
Parent 的虚函数表
├─ Parent::~Parent
├─ Parent::f
└─ Parent::k

Child 的虚函数表
├─ Child::~Child
├─ Child::f        // override 后替换 Parent::f
├─ Parent::k       // 未重写则沿用基类实现
└─ Child::h        // 派生类新增虚函数
```

## 虚函数表顺序

派生类虚函数表前半部分要与基类虚函数表保持对应关系。

如果基类第 2 项是 `f`，派生类第 2 项也对应 `f`。区别只是表项指向的实现可能从 `Parent::f` 换成 `Child::f`。

这样运行时调用不必按名字查找函数，只需要按固定位置取表项，速度更快。

## 虚函数表指针

含虚函数的对象中通常会增加一个隐藏指针，称为 [[虚函数表指针]] 或 `vptr`。

```text
Rectangle 对象
├─ vptr -> Rectangle 的虚函数表
├─ width
└─ height
```

同一个类的多个对象共享同一张虚函数表；每个对象内部的 `vptr` 指向自己所属类的虚函数表。

## 动态调用过程

以 `Shape* p = new Circle(3); p->show();` 为例：

1. `p` 的静态类型是 `Shape*`，但实际指向 `Circle` 对象；
2. `show()` 如果不是虚函数，调用的是 `Shape::show()`；
3. `Shape::show()` 内部调用虚函数 `area()`；
4. 程序通过当前对象的 `vptr` 找到 `Circle` 的虚函数表；
5. 在虚函数表中找到 `area` 对应表项；
6. 表项指向 `Circle::area()`，所以最终调用圆的面积计算。

这就是“基类代码中调用虚函数，却能执行派生类实现”的原因。

## 虚析构如何工作

如果通过基类指针删除派生类对象：

```cpp
Shape* p = new Circle(3);
delete p;
```

当析构函数是虚函数时，`delete` 会通过 `vptr` 找到 `Circle` 的析构函数，先析构派生类部分，再析构基类部分。

这解释了为什么多态基类需要虚析构函数。

## 是否所有函数都要 virtual

不需要。把函数声明为虚函数会进入虚函数表，调用时可能有额外间接开销。

如果一个函数不需要根据实际对象类型改变行为，就不必声明为 `virtual`。例如 `show()` 本身可以不虚，只要它内部调用的 `area()` 是虚函数，最终面积计算仍然会动态派发。

## 本节小结

| 概念 | 作用 |
|---|---|
| [[虚函数表]] | 保存虚函数入口地址 |
| [[虚函数表指针]] | 对象中指向所属类虚函数表的隐藏指针 |
| 表项顺序 | 派生类前半部分与基类保持对应 |
| override | 用派生类实现替换对应虚函数表项 |
| 动态编联 | 运行时通过 `vptr` 和虚函数表找到真实实现 |

> [!summary] 考前速记
> 虚函数表是函数指针数组，表项指向虚函数代码；对象中有 `vptr` 指向所属类虚表。虚函数调用不是按函数名慢慢找，而是按虚表固定序号取函数地址。重写虚函数就是把派生类虚表对应位置换成派生类实现。
