# 18.4 虚函数的作用机制

本节从[[静态类型]]、[[动态类型]]出发，解释编译器如何把虚函数调用翻译成通过[[虚函数表]]查找函数地址的过程。

## 静态类型

静态类型是在编译期间就能确定的类型。

```cpp
Child child;
Parent* p = &child;
Parent& r = child;
Parent object = child;
```

| 表达式 | 静态类型 |
|---|---|
| `child` | `Child` |
| `p` | `Parent*` |
| `r` | `Parent&` |
| `object` | `Parent` |

编译器看到 `p->f()` 时，首先只知道 `p` 的静态类型是 `Parent*`，因此会到 `Parent` 中查找 `f`。

## 动态类型

动态类型是在运行时才能确定的对象真实类型。

```cpp
Parent* p = &child;
```

`p` 的静态类型是 `Parent*`，但它实际指向的是 `Child` 对象，所以它的动态类型是 `Child`。

对象形式转换不同：

```cpp
Parent object = child;
```

这里发生[[对象切片]]，新建了一个真正的 `Parent` 对象，所以 `object` 的静态类型和动态类型都是 `Parent`。

## 编译阶段先检查静态类型

对于：

```cpp
p->func();
```

编译器先做两件事：

1. 判断 `p` 的静态类型；
2. 在静态类型中查找是否存在匹配的 `func`。

如果 `Parent` 中没有匹配的 `func`，即使 `p` 运行时可能指向某个有 `func` 的派生类对象，编译也会失败。

因此，虚函数机制不是“随便调用动态类型里的函数”，它仍然要求静态类型中存在对应接口。

## 非虚函数的调用

如果 `func` 不是虚函数，编译器直接按静态类型绑定：

```cpp
p->Parent::func();
```

也就是不管 `p` 运行时指向 `Child` 还是 `GrandChild`，调用目标都由 `Parent*` 这个静态类型决定。

## 虚函数的调用

如果 `func` 是虚函数，编译器会把调用转换成类似下面的过程：

```text
1. 通过 p 找到对象
2. 通过对象中的 vptr 找到虚函数表
3. 根据 func 在虚函数表中的固定序号 index 取表项
4. 得到函数入口地址
5. 把 p 作为 this 指针传入并调用该函数
```

可以抽象理解为：

```cpp
(*(p->vptr[index]))(p);
```

这里的 `index` 在编译期已经根据静态类型中的虚函数表布局确定；运行时变化的是 `p->vptr` 指向哪一张虚函数表。

如果 `p` 指向 `Child` 对象，`p->vptr` 指向 `Child` 的虚函数表；如果指向 `Circle` 对象，就指向 `Circle` 的虚函数表。

## 为什么虚函数表要按固定顺序

虚函数调用依赖表项序号，而不是运行时按函数名慢慢查找。

因此派生类虚函数表的前半部分必须与基类虚函数表对应：

```text
Parent 虚表第 2 项：func
Child  虚表第 2 项：func
```

如果 `Child` 重写了 `func`，第 2 项会指向 `Child::func`；如果没有重写，就仍然指向 `Parent::func`。

这种按序号查表的方式速度快，是 C++ 虚机制常见实现的原因。

## 基类中必须有对应虚函数

以 `Shape::show()` 为例：

```cpp
class Shape {
public:
    void show() {
        cout << area() << endl;
    }

    virtual double area();
};
```

即使实际执行时永远希望调用 `Rectangle::area()` 或 `Circle::area()`，`Shape` 中也必须声明 `area()`。因为 `show()` 的 `this` 静态类型是 `Shape*`，编译器必须先在 `Shape` 中找到 `area`，才能生成虚函数调用代码。

## 本节小结

| 阶段 | 做什么 |
|---|---|
| 编译期 | 根据静态类型检查函数是否存在 |
| 编译期 | 判断函数是否为虚函数 |
| 编译期 | 确定虚函数在虚表中的序号 |
| 运行期 | 根据对象的 `vptr` 找到动态类型虚表 |
| 运行期 | 按序号取函数地址并调用 |

> [!summary] 考前速记
> 虚函数调用不是跳过静态类型检查。先在静态类型中找函数，找不到就编译错误；找到了再判断是不是虚函数。若是虚函数，运行时通过对象 `vptr` 找到动态类型虚表，再按编译期确定的序号取函数地址。
