# 18.5 虚函数的作用机制（例）

本节通过 A/B 类和 Date 类两个例子，把[[虚函数表]]、[[静态类型]]、[[动态类型]]和[[虚析构函数]]串起来。

## A 与 B 的虚表例子

设基类 `A` 有虚析构函数、虚函数 `f2()`、虚函数 `g()`：

```cpp
class A {
public:
    virtual ~A();
    virtual void f2();
    virtual void g();
};
```

派生类 `B` 继承 `A`，自己增加 `f1()`、重写 `f2()`、增加 `f3()`：

```cpp
class B : public A {
public:
    virtual ~B();
    virtual void f1();
    void f2() override;
    virtual void f3();
};
```

可以把两张虚表抽象理解为：

| 类 | 虚表内容 |
|---|---|
| `A` | `A::~A`、`A::f2`、`A::g` |
| `B` | `B::~B`、`B::f2`、`A::g`、`B::f1`、`B::f3` |

其中 `B::f2` 放在与 `A::f2` 对应的虚表槽位上，因为它是对 `A::f2` 的[[override]]；`B::f1`、`B::f3` 是新增虚函数，所以追加在后面。

## A 指针指向 A 对象

```cpp
A* p1 = new A;
p1->f2();
p1->g();
p1->f1();   // 编译错误
```

`p1` 的静态类型是 `A*`，动态类型也是 `A`。

调用 `p1->f2()` 时：

1. 编译期先在 `A` 中查找 `f2`；
2. 发现 `f2` 是虚函数；
3. 编译器确定 `f2` 在 `A` 虚表中的固定下标；
4. 运行期通过 `p1` 指向对象的 `vptr` 找到 `A` 的虚表；
5. 调用 `A::f2`。

`p1->g()` 同理，调用 `A::g`。

`p1->f1()` 不能通过编译，因为 `p1` 的静态类型是 `A*`，而 `A` 中没有 `f1`。虚函数机制不会跳过静态类型检查。

## A 指针指向 B 对象

```cpp
A* p2 = new B;
p2->f2();
p2->g();
p2->f1();   // 编译错误
delete p2;
```

`p2` 的静态类型是 `A*`，动态类型是 `B`。

| 表达式 | 编译期检查 | 运行期结果 |
|---|---|---|
| `p2->f2()` | `A` 中有虚函数 `f2` | 通过 `B` 的虚表调用 `B::f2` |
| `p2->g()` | `A` 中有虚函数 `g` | `B` 没有重写，调用 `A::g` |
| `p2->f1()` | `A` 中没有 `f1` | 编译失败，轮不到运行期 |
| `delete p2` | `A` 的析构函数是虚的 | 先执行 `B::~B`，再执行 `A::~A` |

> [!warning] 易错点
> 动态类型是 `B` 不代表可以通过 `A*` 调用 `B` 独有的函数。编译器先看静态类型，`A` 中没有 `f1`，`p2->f1()` 就是编译错误。

## Date 例子：同一个接口，不同显示方式

基类 `Date` 保存年月日，并提供虚函数 `display()`：

```cpp
class Date {
public:
    Date(int y, int m, int d);
    virtual ~Date();
    virtual void display();
protected:
    int year;
    int month;
    int day;
};
```

`ChineseDate` 按“年月日”显示，`AmericanDate` 按“月/日/年”显示：

```cpp
class ChineseDate : public Date {
public:
    void display() override;
};

class AmericanDate : public Date {
public:
    void display() override;
};
```

使用基类指针：

```cpp
Date* pDate = new AmericanDate(2008, 12, 31);
pDate->display();
delete pDate;
```

执行 `pDate->display()` 时，编译器只要求 `Date` 中存在 `display`，并且它是虚函数。运行时 `pDate` 指向 `AmericanDate` 对象，因此通过 `AmericanDate` 的虚表调用 `AmericanDate::display()`。

这里的关键不是“绕一圈多此一举”，而是让函数参数可以面向公共基类编程：

```cpp
void func(Date* p) {
    p->display();
}
```

`func` 可以接收 `ChineseDate*`、`AmericanDate*`，以后也可以接收新增加的 `BritishDate*`。函数本身不需要为每一种日期格式写一份分支，实际显示方式由对象的动态类型决定。

## delete 为什么也需要虚机制

如果 `Date` 的析构函数不是虚函数：

```cpp
Date* pDate = new AmericanDate(2008, 12, 31);
delete pDate;
```

`delete pDate` 只会按静态类型调用 `Date::~Date()`，派生类资源可能无法正确释放。

把析构函数声明为虚函数后，`delete` 会先通过虚机制找到动态类型的析构函数，先析构派生类部分，再自动析构基类部分。

## 本节小结

| 问题 | 结论 |
|---|---|
| 虚函数调用是否检查静态类型 | 必须检查 |
| 静态类型没有该函数怎么办 | 编译错误 |
| 派生类重写基类虚函数 | 占用同一个虚表槽位 |
| 派生类新增虚函数 | 追加到派生类虚表 |
| 基类指针删除派生类对象 | 基类析构函数应为虚函数 |

> [!summary] 考前速记
> `A* p = new B; p->f();` 的判断顺序是：先看 `A` 中有没有 `f`，没有就编译错误；有且是非虚函数就调用 `A::f`；有且是虚函数就按 `p` 指向对象的虚表调用动态类型版本。
