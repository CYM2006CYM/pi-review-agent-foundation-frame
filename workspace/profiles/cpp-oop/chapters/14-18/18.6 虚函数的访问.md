# 18.6 虚函数的访问

本节讨论成员函数内部调用虚函数时如何绑定，重点区分普通成员函数中的调用与[[构造析构中的虚调用]]。

## 成员函数中调用成员函数的几种情况

在成员函数内部写：

```cpp
f();
```

可以理解为：

```cpp
this->f();
```

因此仍然要看 `this` 的静态类型、被调函数是否为虚函数，以及运行时 `this` 指向的对象。

| 调用位置 | 被调函数 | 绑定方式 |
|---|---|---|
| 虚函数中 | 非虚函数 | 静态编联，调用本类可见版本 |
| 非虚函数中 | 虚函数 | 动态编联 |
| 虚函数中 | 虚函数 | 动态编联 |
| 构造函数中 | 虚函数 | 调用本地版本 |
| 析构函数中 | 虚函数 | 调用本地版本 |

## 普通成员函数内部仍按 this 判断

假设基类 `A` 中有虚函数 `f()`、虚函数 `g()` 和非虚函数 `k()`；派生类 `B` 重写 `f()`，重定义非虚函数 `k()`，并增加 `h()`。

```cpp
A* p2 = new B;
p2->f();
p2->g();
p2->k();
p2->h();    // 编译错误
```

判断规则仍然分成两步：

1. 编译期看 `p2` 的静态类型 `A*`；
2. 如果函数存在，再看它是不是虚函数。

所以：

| 调用 | 结果 |
|---|---|
| `p2->f()` | `A` 中有虚函数 `f`，运行期调用 `B::f` |
| `p2->g()` | `A` 中有虚函数 `g`，`B` 没重写则调用 `A::g` |
| `p2->k()` | `A::k` 是非虚函数，静态绑定到 `A::k` |
| `p2->h()` | `A` 中没有 `h`，编译错误 |

如果 `B::f()` 内部调用 `g()` 或 `k()`，就要把它们理解为 `this->g()`、`this->k()`。在 `B::f()` 中，`this` 的静态类型是 `B*`，因此查找和绑定规则会从 `B` 的角度重新进行。

## 非虚函数中的虚调用

一个容易混淆的情况是：外层函数本身不是虚函数，但它内部调用了虚函数。

```cpp
class A {
public:
    virtual void f();
    void k() {
        f();
    }
};

class B : public A {
public:
    void f() override;
};
```

如果执行：

```cpp
A* p = new B;
p->k();
```

`p->k()` 因为 `k` 是非虚函数，所以先静态绑定到 `A::k()`。但进入 `A::k()` 后，里面的 `f()` 等价于 `this->f()`，`this` 实际指向 `B` 对象，`f` 又是虚函数，所以最终会调用 `B::f()`。

> [!important] 高频考点
> “外层函数是否虚”只决定外层调用怎么进入；进入函数体之后，函数体里的每一次成员调用都要按那一次调用自己的规则重新判断。

## 构造函数中的虚调用

构造派生类对象时，先构造基类部分，再构造派生类部分。

```cpp
class A {
public:
    A() {
        f();
    }
    virtual void f();
};

class B : public A {
public:
    void f() override;
};
```

创建 `B` 对象时，会先执行 `A::A()`。在 `A::A()` 中调用 `f()`，不会动态调用 `B::f()`，而是调用 `A::f()`。

原因是：执行 `A` 的构造函数时，`B` 的成员还没有构造完成。如果此时允许调用 `B::f()`，`B::f()` 可能访问尚未初始化的派生类数据，行为不可靠。

所以 C++ 规定：构造函数内部调用虚函数时，采用当前构造层次的本地版本。

## 析构函数中的虚调用

析构派生类对象时，先析构派生类部分，再析构基类部分。

```cpp
class A {
public:
    virtual ~A() {
        f();
    }
    virtual void f();
};

class B : public A {
public:
    void f() override;
};
```

释放 `B` 对象时，会先执行 `B::~B()`，再执行 `A::~A()`。当执行到 `A::~A()` 内部的 `f()` 时，`B` 的派生类部分已经析构完毕，因此也不会动态调用 `B::f()`，而是调用 `A::f()`。

因此：析构函数内部调用虚函数时，也采用当前析构层次的本地版本。

## 完整判断口诀

对一次成员函数调用：

```cpp
x->func();
```

按下面顺序判断：

1. 看 `x` 的静态类型；
2. 在静态类型中查找 `func`；
3. 找不到则编译错误；
4. 找到但不是虚函数，静态绑定；
5. 找到且是虚函数，通常动态绑定；
6. 如果这次调用发生在构造函数或析构函数内部，调用当前类的本地版本。

## 本节小结

| 场景 | 最重要的判断 |
|---|---|
| 通过基类指针调用虚函数 | 看动态类型 |
| 通过基类指针调用非虚函数 | 看静态类型 |
| 非虚函数内部调用虚函数 | 内部虚调用仍可动态绑定 |
| 构造函数内部调用虚函数 | 不动态派发，调用本地版本 |
| 析构函数内部调用虚函数 | 不动态派发，调用本地版本 |

> [!summary] 考前速记
> 普通成员函数内部调用虚函数，仍可通过 `this` 发生动态绑定；但构造函数和析构函数内部调用虚函数时，只调用本地版本。原因是构造时派生类部分尚未形成，析构时派生类部分已经销毁。
