# 18.7 具体类和抽象类

本节介绍[[具体类]]、[[抽象类]]、[[纯虚函数]]、[[纯虚定义]]和[[接口类]]，重点理解“类是否能实例化”和“类是否用于提供抽象接口”的关系。

## 具体类

具体类是可以实例化出具体对象的类。

```cpp
Rectangle r(3, 4);
Circle c(5);
```

`Rectangle`、`Circle` 都表示清楚的对象类型，构造对象后有明确含义，所以它们是具体类。

## 不能实例化不等于抽象类

有些类不能实例化，但原因可能只是构造函数不可访问：

```cpp
class A {
private:
    A();
};
```

这个类在类外不能直接创建对象，但课程中讨论的“抽象类”不是泛指所有不能实例化的类。

抽象类强调的是：它处在继承层次的较高位置，用来给后续派生类提供高层次接口或抽象约束。

> [!warning] 易错点
> “不能实例化”是抽象类的结果之一，但不是抽象类的充分定义。C++ 中只要类含有至少一个纯虚函数，它就是抽象类。

## 抽象类的设计动机

考虑 `Shape`：

```cpp
class Shape {
public:
    virtual double area();
};
```

`Circle` 可以求面积，`Rectangle` 可以求面积；但“一个普通形状”本身是什么，面积是多少，并不清楚。直接实例化 `Shape` 没有太大意义。

因此 `Shape` 更适合做抽象类：它规定“所有形状都应该能求面积”，但具体面积公式留给派生类实现。

类似地，`Animal` 可以规定动物共有接口，`Dog`、`Cat`、`Fish` 再分别给出具体行为。

## 纯虚函数

纯虚函数的声明格式是在虚函数声明末尾写 `= 0`：

```cpp
class Shape {
public:
    virtual double area() = 0;
};
```

只要一个类中有至少一个纯虚函数，这个类就是抽象类，不能直接实例化：

```cpp
Shape s;        // 错误
Shape* p;       // 可以，只是声明指针
```

纯虚函数的访问控制可以是 `public`、`protected` 或 `private`，但最常见的是 `public`，因为它通常用于提供公共接口。

## 派生类何时成为具体类

抽象类的派生类如果没有实现所有继承来的纯虚函数，仍然是抽象类。

```cpp
class Polygon : public Shape {
    // 没有实现 area()
};
```

`Polygon` 仍然不能实例化。

如果派生类实现了所有纯虚函数，就可以成为具体类：

```cpp
class Rectangle : public Shape {
public:
    double area() override {
        return width * height;
    }
private:
    double width;
    double height;
};
```

此时 `Rectangle` 可以实例化。

## 纯虚定义

纯虚函数通常不写实现，但 C++ 允许给纯虚函数提供定义，这称为纯虚定义。

```cpp
class Shape {
public:
    virtual double area() = 0;
};

double Shape::area() {
    return 0;
}
```

纯虚定义的作用是给某些派生类提供一个默认实现。例如“线”“抛物线”等非封闭形状可能都没有面积，如果每个类都重复写 `return 0;`，会造成重复；这时可以让它们复用 `Shape::area()` 的默认实现。

但要注意：

- 纯虚定义只能写在类外；
- 即使纯虚函数有定义，只要它仍然被声明为 `= 0`，所在类仍然是抽象类；
- 派生类若想成为具体类，仍需让该纯虚函数在派生层次中变成非纯虚实现。

## 纯抽象类和接口类

如果一个类除了构造、析构等特殊成员外，主要成员函数都是公有纯虚函数，它就接近接口类。

```cpp
class AreaComputable {
public:
    virtual ~AreaComputable() = default;
    virtual double area() = 0;
};
```

这个类的含义是：所有可实例化的派生类都必须提供 `area()` 这个公共行为。

在 C++ 中没有专门的 `interface` 关键字，但可以用只含公共纯虚函数的抽象类表达接口。

接口类也可以继续派生出新的接口：

```cpp
class Country : public AreaComputable {
public:
    virtual long population() = 0;
};
```

这里 `Country` 表示：一个具体国家既要能计算面积，又要能计算人口。它仍然是抽象接口，具体国家类再实现这些行为。

## 本节小结

| 概念 | 判断标准 |
|---|---|
| 具体类 | 可以实例化出具体对象 |
| 抽象类 | 至少含有一个纯虚函数 |
| 纯虚函数 | `virtual 函数声明 = 0` |
| 纯虚定义 | 给纯虚函数在类外提供默认实现 |
| 接口类 | 主要由公有纯虚函数组成，用于规定公共行为 |

> [!summary] 考前速记
> 类中只要有一个纯虚函数就是抽象类，不能实例化。派生类只有实现所有纯虚函数后才可能成为具体类。纯虚函数可以有类外定义，但有定义不改变“纯虚”身份，原类仍然是抽象类。
