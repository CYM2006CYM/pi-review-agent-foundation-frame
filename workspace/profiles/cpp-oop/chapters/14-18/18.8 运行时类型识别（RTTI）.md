# 18.8 运行时类型识别（RTTI）

本节介绍[[RTTI]] 的基本思想、实现依赖、`typeid` 操作符，以及 [[dynamic_cast]]。

## 为什么需要 RTTI

在运行期间，程序有时需要判断一个对象的真实类型。

例如：

```cpp
Animal* p = getAnimal();
```

`p` 的静态类型是 `Animal*`，但运行时可能指向 `Cat`、`Dog` 或其他派生类对象。若程序必须判断它到底是不是 `Cat`，就需要运行时类型识别。

RTTI 全称是 Runtime Type Information，即运行时类型信息。

## RTTI 通常依赖虚机制

课程中强调：C++ 中使用 RTTI 的类通常需要有[[虚函数]]，也就是要成为多态类型。

含有虚函数的对象通常带有[[虚函数表指针]]，指向该动态类型的[[虚函数表]]。虚表附近还会保存或关联类型信息结构，可以理解为 `Type Information`。运行时通过对象的虚表关联到类型信息，从而判断对象的动态类型。

因此 RTTI 与[[虚机制]]关系密切。

> [!warning] 易错点
> RTTI 不是免费的。它需要在运行时查找类型信息，效率低于普通静态类型检查。能用虚函数多态解决的问题，通常不要先写成大量类型判断。

## typeid 操作符

`typeid` 是操作符，不是普通函数，地位类似 `sizeof`。

它可以返回表达式对应的类型信息。

```cpp
#include <typeinfo>

typeid(expr)
```

设：

```cpp
class A {
public:
    virtual ~A() = default;
};

class B : public A {};
class C : public A {};

A* p1 = new B;
```

## typeid 用在指针上

```cpp
typeid(p1) == typeid(A*)
```

`p1` 本身是一个指针变量，它的静态类型是 `A*`。因此 `typeid(p1)` 得到的是指针变量的静态类型信息，而不是它指向对象的动态类型。

所以：

```cpp
typeid(p1) == typeid(A*)   // true
typeid(p1) == typeid(B*)   // false
```

## typeid 用在对象或引用上

如果要判断 `p1` 指向对象的动态类型，应对对象本身使用 `typeid`：

```cpp
typeid(*p1) == typeid(A)   // false
typeid(*p1) == typeid(B)   // true
```

因为 `*p1` 表示 `p1` 指向的对象。`A` 是多态类型时，`typeid(*p1)` 会根据运行时对象返回动态类型信息。

| 写法 | 关注对象 | 常见结果 |
|---|---|---|
| `typeid(p1)` | 指针变量本身 | 静态类型 `A*` |
| `typeid(*p1)` | 指针指向的对象 | 动态类型 `B` |
| `typeid(ref)` | 引用绑定的对象 | 动态类型 |

## dynamic_cast 的用途

`dynamic_cast` 常用于从基类指针或引用向派生类指针或引用转换，也就是安全的[[向下类型转换]]。

```cpp
A* p = new B;
B* pb = dynamic_cast<B*>(p);
```

如果 `p` 真实指向 `B` 对象，或指向 `B` 的派生类对象，转换成功；否则转换失败。

例如：

```cpp
A* p = new C;
B* pb = dynamic_cast<B*>(p);
```

这里 `p` 实际指向 `C`，不是 `B`，也不是 `B` 的派生类，所以 `pb` 得到 `nullptr`。

## 指针形式与引用形式

指针形式转换失败返回空指针：

```cpp
if (B* pb = dynamic_cast<B*>(p)) {
    pb->someBOperation();
}
```

引用形式不能返回空引用，因此转换失败会抛出异常：

```cpp
try {
    B& rb = dynamic_cast<B&>(*p);
    rb.someBOperation();
} catch (...) {
    // 转换失败
}
```

课程中强调，实际代码里更常用指针形式，因为失败时判断 `nullptr` 比处理异常更直接。

## 本节小结

| 主题 | 结论 |
|---|---|
| RTTI | 运行时获取对象真实类型信息 |
| 适用类 | 通常需要含虚函数的多态类型 |
| 实现基础 | 借助虚机制和类型信息结构 |
| `typeid(p)` | `p` 是指针时，得到指针静态类型 |
| `typeid(*p)` | `*p` 是多态对象时，可得到动态类型 |
| `dynamic_cast<T*>(p)` | 失败返回 `nullptr` |
| `dynamic_cast<T&>(r)` | 失败抛异常 |

> [!summary] 考前速记
> `typeid(p)` 看指针变量，通常是静态类型；`typeid(*p)` 才看指针指向的对象。`dynamic_cast` 用 RTTI 做安全向下转换，指针失败返回 `nullptr`，引用失败抛异常。
