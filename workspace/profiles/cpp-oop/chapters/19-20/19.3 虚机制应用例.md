# 19.3 虚机制应用例

本节用三个例子说明[[虚机制]]如何通过[[子类型化]]适应变化：水果能量、软件开发流程、虚拟拷贝构造函数。

## 例一：老鼠吃水果

设有一个 `Fruit` 基类，派生出 `Apple`、`Strawberry` 等不同水果。

老鼠吃水果后体重增加，增加量取决于水果能量：

```cpp
class Fruit {
public:
    virtual ~Fruit() = default;
    virtual double energy() const = 0;
};

class Mouse {
public:
    void eat(Fruit* fruit) {
        weight += fruit->energy() * 0.1;
    }
private:
    double weight{};
};
```

不同水果的能量计算方式不同：

```cpp
class Apple : public Fruit {
public:
    double energy() const override;
};

class Strawberry : public Fruit {
public:
    double energy() const override;
};
```

`Mouse::eat` 不关心具体水果类型，只向 `Fruit` 接口发送 `energy()` 消息。以后新增 `Banana`，只需要新增一个 `Fruit` 子类并实现 `energy()`，`Mouse` 类不需要修改。

这体现了：变化在水果种类和能量计算中，稳定的是“水果能提供能量”这个接口。

## 例二：软件开发流程

软件可以分为手机 App、Web 应用、游戏等多种类型。不同软件的设计、编码、测试、维护细节不同，但大的开发流程相似。

可以在基类中固定流程框架，把细节留给派生类实现：

```cpp
class Software {
public:
    virtual ~Software() = default;

    void develop() {
        design();
        code();
        test();
        maintain();
    }

protected:
    virtual void design() = 0;
    virtual void code() = 0;
    virtual void test() = 0;
    virtual void maintain() = 0;
};
```

派生类实现具体步骤：

```cpp
class MobileApp : public Software {
protected:
    void design() override;
    void code() override;
    void test() override;
    void maintain() override;
};

class WebApp : public Software {
protected:
    void design() override;
    void code() override;
    void test() override;
    void maintain() override;
};
```

这里基类提供“骨架”，派生类提供“步骤细节”。新增一种软件时，开发流程本身不变，只增加新的派生类。

这种思想接近[[模板方法]]：父类定义算法流程，子类重写其中可变步骤。

## 例三：复制一个真实动态类型对象

设有继承树：

```cpp
class A {};
class B : public A {};
class C : public A {};
```

现在有一个 `A& object`，它实际可能是 `A`、`B`、`C` 或未来新增的派生类。希望复制出一个与它动态类型相同的新对象。

朴素做法可能是：

```cpp
A* copy(const A& object) {
    if (typeid(object) == typeid(B)) {
        return new B(static_cast<const B&>(object));
    } else if (typeid(object) == typeid(C)) {
        return new C(static_cast<const C&>(object));
    }
    return new A(object);
}
```

这种写法有两个问题：

- 依赖 [[RTTI]]，效率和可维护性都差；
- 每新增一个派生类，就要修改 `copy` 函数，容易引入错误。

## 虚拟拷贝构造函数

构造函数和拷贝构造函数不能是虚函数。因此不能直接写“虚拷贝构造函数”。

解决方法是定义一个普通虚函数 `clone()`，由它内部调用本类的拷贝构造函数：

```cpp
class A {
public:
    virtual ~A() = default;

    virtual A* clone() const {
        return new A(*this);
    }
};

class B : public A {
public:
    B* clone() const override {
        return new B(*this);
    }
};

class C : public A {
public:
    C* clone() const override {
        return new C(*this);
    }
};
```

使用时：

```cpp
A* p = getObject();
A* copy = p->clone();
```

如果 `p` 实际指向 `B`，动态绑定会调用 `B::clone()`，返回 `new B(*this)`；如果实际指向 `C`，就调用 `C::clone()`。

这种 `clone()` 称为[[虚拟拷贝构造函数]]：它不是构造函数，但以虚函数形式实现了“按动态类型复制对象”的效果。

## 本节小结

| 应用 | 稳定部分 | 变化部分 |
|---|---|---|
| 老鼠吃水果 | `Mouse::eat(Fruit*)` | 不同水果的 `energy()` |
| 软件开发流程 | `Software::develop()` 框架 | 各软件的设计、编码、测试、维护 |
| 对象克隆 | `clone()` 接口 | 各类调用自己的拷贝构造函数 |

> [!summary] 考前速记
> 虚机制适合把“稳定接口”留在基类，把“变化实现”放进派生类。典型题是 `clone()`：构造函数不能虚，所以用普通虚函数调用本类拷贝构造函数，实现按动态类型复制对象。
