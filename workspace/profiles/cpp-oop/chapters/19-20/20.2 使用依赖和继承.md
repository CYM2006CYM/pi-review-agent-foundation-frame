# 20.2 使用依赖和继承

本节通过老鼠吃水果、怪物战斗、鳄鱼杀怪物三个例子，说明如何组合使用[[依赖]]和[[继承]]。

## 水平关系与垂直关系

类之间关系大体分两类：

| 关系 | 包含内容 | 复用方式 |
|---|---|---|
| [[水平关系]] | [[依赖]]、[[关联]]、[[聚集]]、[[组合]] | [[黑盒复用]] |
| [[垂直关系]] | [[继承]] | [[白盒复用]] |

现代设计中更常用的是：

- 水平关系；
- 必要时使用公有继承。

[[私有继承]]能用组合替代时应优先使用组合；[[保护继承]]较少使用。公有继承通常只在需要[[子类型化]]时使用。

## 设计结构：短小继承树加水平关系

课程强调，好的面向对象结构不应是一棵很深的大继承树，而应更像“一片低矮的灌木”：

- 每棵继承树较短，通常两三层；
- 有很多小继承树；
- 小继承树之间通过依赖、关联等水平关系协作。

如果没有子类型化需求，代码中就不应该随意出现公有继承。

## 依赖和继承的基本形式

设 `A` 依赖于 `Parent`，而 `Parent` 有多个派生类：

```cpp
class Parent {
public:
    virtual ~Parent() = default;
    virtual void f() = 0;
};

class Child1 : public Parent {};
class Child2 : public Parent {};

class A {
public:
    void use(Parent& p) {
        p.f();
    }
};
```

`A::use` 以参数形式依赖 `Parent`，实际传入对象可以是 `Child1`、`Child2` 或未来新增子类。

这种结构很常见，例如：

- 老鼠吃水果；
- 学生做作业；
- 屏幕绘制区域；
- 打印机打印日期。

## 例一：老鼠吃水果

`Mouse` 通过 `eat(Fruit&)` 依赖 `Fruit`：

```cpp
class Fruit {
public:
    virtual ~Fruit() = default;
    virtual double weight() const = 0;
    virtual double edibleRatio() const = 0;
};

class Orange : public Fruit {
public:
    double weight() const override;
    double edibleRatio() const override;
};

class Apple : public Fruit {
public:
    double weight() const override;
    double edibleRatio() const override;
};

class Mouse {
public:
    virtual ~Mouse() = default;

    virtual void eat(Fruit& fruit) {
        weightValue += fruit.weight() * fruit.edibleRatio();
    }

    virtual double weight() const {
        return weightValue;
    }

private:
    double weightValue{};
};
```

这里的变化有两类：

- 水果可以新增种类；
- 不同水果可以有不同重量和可食用比例。

`Mouse` 只依赖 `Fruit` 接口。以后新增 `Strawberry`，只需要从 `Fruit` 派生新类，`Mouse` 不需要改。

## 老鼠本身也可以子类型化

老鼠也可能有不同种类，例如大老鼠、小老鼠。它们吃水果后的体重变化规则不同。

```cpp
class BigMouse : public Mouse {
public:
    void eat(Fruit& fruit) override {
        weightValue += fruit.weight() * fruit.edibleRatio() * coefficient;
    }
private:
    double coefficient{1.5};
};

class SmallMouse : public Mouse {
public:
    void eat(Fruit& fruit) override {
        weightValue += fruit.weight() * fruit.edibleRatio() * 0.2;
    }
};
```

这样水果种类可以扩展，老鼠种类也可以扩展。两个方向都通过各自继承树适应变化。

## 把系数也抽象成行为

如果“吸收系数”本身还会变化，可以进一步抽象成虚函数：

```cpp
class Mouse {
public:
    virtual ~Mouse() = default;

    virtual void eat(Fruit& fruit) {
        weightValue += fruit.weight() * fruit.edibleRatio() * coefficient();
    }

protected:
    virtual double coefficient() const = 0;

private:
    double weightValue{};
};

class BigMouse : public Mouse {
protected:
    double coefficient() const override {
        return value;
    }
private:
    double value{1.5};
};

class SmallMouse : public Mouse {
protected:
    double coefficient() const override {
        return 0.2;
    }
};
```

两个版本功能可能相同，但抽象出 `coefficient()` 后，扩展弹性更好。设计优劣常常不是在当前结果中体现，而是在未来变化时体现。

## 双向依赖：把 this 传给对方

有时 `Fruit::edibleRatio()` 也需要根据吃它的老鼠来计算。

```cpp
class Fruit {
public:
    virtual double edibleRatio(Mouse& mouse) const = 0;
};

class Mouse {
public:
    virtual void eat(Fruit& fruit) {
        weightValue += fruit.weight() * fruit.edibleRatio(*this);
    }
};
```

这里 `Mouse` 依赖 `Fruit`，同时 `Fruit` 的实现又依赖 `Mouse`，形成[[双向依赖]]。

`*this` 的静态类型是 `Mouse&`，但动态类型可以是 `BigMouse&`、`SmallMouse&` 等。传给 `Fruit` 后，`Fruit` 可以通过虚函数获得实际老鼠对象的信息。

## 例二：怪物战斗的自依赖

设 `Monster` 有 `fight(Monster&)` 行为，一个怪物与另一个怪物一对一战斗，双方轮流攻击，直到一方生命值为零。

```cpp
class Monster {
public:
    virtual ~Monster() = default;

    bool fight(Monster& other) {
        while (true) {
            if (attack(other)) {
                return true;
            }
            if (other.attack(*this)) {
                return false;
            }
        }
    }

protected:
    virtual bool attack(Monster& other) = 0;
};
```

`fight` 的流程是稳定的，放在基类中；不同怪物的攻击效果不同，放在虚函数 `attack` 中。

```cpp
class Dog : public Monster {
protected:
    bool attack(Monster& other) override;
};

class Cat : public Monster {
protected:
    bool attack(Monster& other) override;
};
```

这里 `Monster` 的 `fight` 参数也是 `Monster&`，所以它是类对自身类型的[[自依赖]]。

## 例三：子类依赖父类

假设怪物继承树中后来新增 `Crocodile`。鳄鱼本身也是怪物，因此继承 `Monster`；但它额外有一个 `kill(Monster&)` 行为，可以杀死各种怪物。

```cpp
class Crocodile : public Monster {
public:
    void kill(Monster& target);

protected:
    bool attack(Monster& other) override;
};
```

这里 `Crocodile` 一方面是 `Monster` 的子类，另一方面它新增的行为又依赖 `Monster`。这体现了“子类依赖父类”这种结构。

## 本节小结

| 场景 | 关系 |
|---|---|
| `Mouse::eat(Fruit&)` | `Mouse` 依赖 `Fruit` |
| `Fruit::edibleRatio(Mouse&)` | `Fruit` 反向依赖 `Mouse` |
| `Monster::fight(Monster&)` | 自依赖 |
| `Crocodile::kill(Monster&)` | 子类依赖父类 |
| 水果、老鼠、怪物扩展 | 通过公有继承和虚函数实现子类型化 |

> [!summary] 考前速记
> 依赖常以函数参数形式出现，继承用于子类型化。设计中应是一堆短小继承树通过依赖、关联协作，而不是一棵很深的继承树。`this` 可以作为参数传给其他对象，形成双向依赖或自依赖。
