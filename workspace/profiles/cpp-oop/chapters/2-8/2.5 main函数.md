# 2.5 main函数

## 本节核心

本节开始详细介绍 C++ 工程中的主要文件和结构，首先讲[[main函数]]。

`main` 是 C/C++ 程序默认的[[入口函数]]。程序执行时，运行环境会从 `main` 开始进入用户代码。

> [!important] 核心认识
> 标准 C++ 程序的 `main` 函数返回值应为 `int`。`return 0;` 通常表示程序正常结束。

## main 函数的基本作用

在 C++ 程序中，`main` 函数是程序执行的起点。

一个最基本的形式是：

```cpp
int main() {
    return 0;
}
```

其中：

- `int` 表示 `main` 的[[返回值]]类型。
- `main` 是函数名。
- `()` 表示没有显式接收命令行参数。
- `{}` 内是程序执行代码。
- `return 0;` 表示程序正常结束。

## 标准写法：返回 int

标准 C++ 中，`main` 函数应返回 `int`。

常见写法有两种：

```cpp
int main() {
    return 0;
}
```

带[[命令行参数]]的写法：

```cpp
int main(int argc, char* argv[]) {
    return 0;
}
```

也可以写成：

```cpp
int main(int argc, char** argv) {
    return 0;
}
```

> [!warning] 易错点
> 有些旧编译器或特定环境可能接受 `void main()`，但这不是标准 C++ 的推荐写法。学习和考试中按 `int main(...)` 记。

## main 返回值有什么用

`main` 的返回值用于向程序的调用者报告程序结束状态。

程序不一定总是由用户双击运行，也可能由：

- 操作系统命令行。
- 另一个程序。
- 脚本。
- 自动化工具。
- 构建或测试系统。

发起程序运行的一方可能需要知道该程序是正常结束还是异常结束。

常见约定：

| 返回值 | 含义 |
|---|---|
| `0` | 正常结束 |
| 非 `0` | 通常表示出现错误或异常状态 |

> [!tip] 初学者理解
> `return 0;` 就像程序结束时交回一张“正常完成”的状态单。

## 命令行参数

[[命令行参数]]是运行程序时跟在程序名后面的参数。

例如在命令行中运行：

```bash
MyApp.exe JLU Computer
```

这里：

- `MyApp.exe` 是程序名。
- `JLU` 是第一个额外参数。
- `Computer` 是第二个额外参数。

程序可以通过 `main` 的参数接收这些内容。

## argc 和 argv

带命令行参数的 `main` 常写作：

```cpp
int main(int argc, char* argv[]) {
    return 0;
}
```

其中：

| 参数 | 含义 |
|---|---|
| [[argc]] | 参数个数 |
| [[argv]] | 参数字符串数组 |

`argc` 是 argument count 的意思，表示参数数量。  
`argv` 是 argument vector 的意思，保存每个参数对应的字符串。

> [!important] 关键点
> 命令行参数默认都按字符串处理。即使输入 `2`，程序接收到的也是字符串 `"2"`，不是整数 `2`。

## argv[0] 通常是程序名

如果只执行：

```bash
MyApp.exe
```

那么通常：

```text
argc == 1
argv[0] == "MyApp.exe"
```

也就是说，即使没有额外参数，`argc` 通常也不是 0，因为程序名本身通常也作为第 0 个参数。

如果执行：

```bash
MyApp.exe JLU Computer
```

那么通常：

```text
argc == 3
argv[0] == "MyApp.exe"
argv[1] == "JLU"
argv[2] == "Computer"
```

> [!warning] 易错点
> `argv[1]` 才是用户在程序名后输入的第一个额外参数；`argv[0]` 通常保存程序名或程序路径。

## 命令行参数都是字符串

如果执行：

```bash
MyApp.exe 2
```

那么 `argv[1]` 是字符串 `"2"`，不是整数 `2`。

如果要把它当整数使用，需要进行转换。例如：

```cpp
#include <cstdlib>

int main(int argc, char* argv[]) {
    if (argc > 1) {
        int value = atoi(argv[1]);
        return value;
    }

    return 0;
}
```

`atoi` 可以把形如数字的字符串转换为整数。后续学习中也可以使用更现代、更安全的转换方式。

## 输出命令行参数示例

下面程序可以打印传入的参数：

```cpp
#include <iostream>

using namespace std;

int main(int argc, char* argv[]) {
    cout << "argc = " << argc << endl;

    for (int i = 0; i < argc; ++i) {
        cout << "argv[" << i << "] = " << argv[i] << endl;
    }

    return 0;
}
```

如果运行：

```bash
MyApp.exe JLU Computer
```

可能输出：

```text
argc = 3
argv[0] = MyApp.exe
argv[1] = JLU
argv[2] = Computer
```

## 命令行参数有什么用

命令行参数可以让同一个程序根据不同输入执行不同任务。

例如：

- 指定输入文件。
- 指定输出文件。
- 指定运行模式。
- 指定配置选项。
- 给自动化脚本传递参数。

这在命令行工具、编译工具、测试工具中很常见。

## 本节和工程结构的关系

C++ 工程中常见源文件主要包括[[实现文件]]和[[头文件]]。`main` 函数通常位于某个实现文件中。

前面学过的编译流程中，包含 `main` 的 `.cpp` 文件也会作为一个[[编译单元]]参与编译，最终在[[链接]]阶段成为程序入口的一部分。

如果工程里没有正确的入口函数，就不能生成普通可执行程序。
