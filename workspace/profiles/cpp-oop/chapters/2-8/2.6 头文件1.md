# 2.6 头文件1

## 本节核心

本节开始系统讲[[头文件]]，重点包括头文件扩展名、标准库头文件与用户自定义头文件、`#include` 的尖括号和双引号写法，以及头文件书写顺序的维护习惯。

> [!important] 核心认识
> 头文件主要提供声明和接口信息；真正的函数代码通常在实现文件或库文件中。`#include` 让当前编译单元“看见声明”，不等于把所有函数实现都写进来了。

## 头文件常见扩展名

常见头文件形式有三类：

| 形式 | 常见含义 |
|---|---|
| `.h` | C 和 C++ 中都常见，用户自定义头文件最常用 |
| `.hpp` | 常用于明确表示 C++ 头文件 |
| 无扩展名 | C++ 标准库头文件常见形式，如 `<iostream>`、`<cmath>` |

在实际 C++ 工程中，用户自定义头文件既可以使用 `.h`，也可以使用 `.hpp`。受历史习惯影响，`.h` 仍然非常常见。

> [!tip] 初学者理解
> 看到 `.h` 不一定就是 C 语言；很多 C++ 项目也大量使用 `.h` 作为头文件扩展名。

## 标准库头文件与用户自定义头文件

头文件大体可以分成两类：

| 类型 | 例子 | 来源 |
|---|---|---|
| [[标准库头文件]] / 系统头文件 | `<iostream>`、`<cmath>`、`<cstdlib>`、`<stdlib.h>` | 标准库、操作系统或编译器附带 |
| [[用户自定义头文件]] | `"MyFile.h"`、`"Student.hpp"` | 程序员自己在工程中编写 |

标准库头文件属于 C++ 标准库或系统/编译器提供的接口。用户自定义头文件属于当前工程的一部分。

## 头文件不是函数代码本身

初学者容易产生误解：包含头文件后就“拥有了函数代码”。

更准确的理解是：

- 头文件通常提供[[声明]]。
- 函数真正的[[定义]]可能在 `.cpp` 实现文件中。
- 标准库函数的实现可能在系统库或编译器附带库中。
- 最终通过[[链接]]把需要的实现与当前程序组合起来。

例如：

```cpp
#include <cstdlib>
```

这让当前代码能看到 `atoi` 等函数的声明。但 `atoi` 的实现代码不是你自己写在当前文件里的，而是来自标准库。

## 标准库代码通常在哪里

标准库或系统库的可执行代码可能来自：

- 操作系统附带的库。
- 编译器附带的库。
- 开发环境安装目录中的库文件。

在 Windows 中，部分系统库可能与系统目录、动态链接库有关；编译器也可能附带自己的运行库和标准库实现。

> [!important] 关键理解
> `#include` 主要解决“声明可见性”问题；库文件和链接过程解决“实现在哪里”问题。

## #include 的尖括号与双引号

`#include` 常见两种写法：

```cpp
#include <iostream>
#include "MyFile.h"
```

它们的搜索习惯不同：

| 写法 | 常用场景 | 搜索倾向 |
|---|---|---|
| `#include <...>` | 标准库头文件、系统头文件 | 在系统/编译器配置的 include 路径中查找 |
| `#include "..."` | 用户自定义头文件 | 先在当前工程或当前文件相关目录查找，找不到再按系统路径查找 |

不同编译器细节可能略有差异，但这个理解足够应付本课程和常规工程阅读。

> [!warning] 易错点
> 理论上有时全用双引号也能找到文件，但不建议这样写。系统头文件用尖括号，自己写的头文件用双引号，更清晰、更利于维护。

## 相对路径和绝对路径

包含头文件时，文件名可以带路径。

例如：

```cpp
#include "utils/MyFile.h"
```

这表示包含当前工程某个相对路径下的头文件。

一般不建议在代码中写绝对路径，例如：

```cpp
#include "C:/some/path/MyFile.h"
```

因为绝对路径依赖具体机器和目录结构，换一台电脑或换一个工程位置就可能失效。

## C 风格头文件与 C++ 风格头文件

有些头文件带 `.h`，例如：

```cpp
#include <stdlib.h>
```

有些 C++ 标准库头文件不带 `.h`，例如：

```cpp
#include <cstdlib>
#include <cmath>
#include <iostream>
```

可以粗略理解为：

- `<stdlib.h>` 是 C 风格头文件。
- `<cstdlib>` 是 C++ 风格头文件。
- C++ 风格标准库通常会把相关名字放进[[名字空间]] `std` 中。

> [!tip] 记忆
> C++ 标准库常见形式是不带 `.h`，例如 `<iostream>`、`<cmath>`；使用其中名字时经常需要 `std::` 或 `using namespace std;`。

## include 的书写顺序

课程建议：标准库或系统头文件放前面，用户自定义头文件放后面。

例如：

```cpp
#include <cstdlib>
#include <cmath>

#include "MyFile.h"
```

这样做的好处是：

- 系统头文件通常比较稳定，不太可能由用户写错。
- 用户自定义头文件更容易出错，放在后面更便于定位。
- 代码结构更清晰，可维护性更好。

一些开发环境会自动生成特殊头文件，例如 Visual Studio 早期项目中常见的 `stdafx.h`。这类文件虽然写法像用户头文件，但它往往由工具生成或用于预编译头，不能简单等同于普通用户自定义头文件。

| 可能题型 | 可能问法 | 答题要点 |

## 本节速记

> [!summary] 速记
> `.h/.hpp` 常用于用户头文件，无扩展名常见于 C++ 标准库头文件。系统头文件用 `<...>`，用户头文件用 `"..."`。头文件主要给声明，真正实现通常在实现文件或库中，最后通过链接组合。
