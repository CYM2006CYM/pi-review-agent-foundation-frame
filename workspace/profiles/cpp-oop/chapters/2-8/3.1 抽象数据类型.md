# 3.1 抽象数据类型

## 本节核心

本节进入第 3 章，开始讲[[抽象数据类型]]。课程要把“类型”从 `int`、`char`、`double` 这种具体语言类型，提升为更一般的、与语言无关的抽象概念。

> [!important] 核心认识
> 抽象数据类型 = 数据集合 + 数据之间的关系/不变式 + 可施加的操作集合。

## 为什么要重新理解“类型”

在 C 语言或基础编程中，我们常把类型理解为：

- 整型。
- 字符型。
- 浮点型。
- 数组。
- 结构体。

但在面向对象程序设计中，类型不只是语言预先给出的几种基本类型。我们可以根据问题需要，自己抽象出新的类型。

例如：

- 学生。
- 账户。
- 向量。
- 日期。
- 订单。
- 图形。

这些都可以被建模为某种类型。

## 抽象数据类型的定义

[[抽象数据类型]]可以理解为一种数学模型。

它通常包含三部分：

| 组成 | 含义 |
|---|---|
| [[数据集合]] | 这个类型可能包含哪些数据或状态 |
| [[不变式]] / 数据关系 | 这些数据之间必须满足什么关系 |
| [[操作集合]] | 外部可以对这个类型执行哪些操作 |

例如，一个“日期”类型可能包含：

- 数据：年、月、日。
- 关系：月份必须在 1 到 12 之间，日期必须符合该月天数。
- 操作：创建日期、比较日期、计算下一天、格式化输出。

这就是一种抽象数据类型。

## 与具体表示无关

抽象数据类型强调“概念本身”，不先关心具体怎样表示。

例如日期可以表示为：

- 三个整数：`year/month/day`。
- 一个字符串：`"2026-06-02"`。
- 一个从某个基准日开始的天数。

这些表示方式不同，但可以对应同一个抽象数据类型。

> [!tip] 初学者理解
> 抽象数据类型先回答“它是什么、能做什么”，再考虑“用什么变量和结构存”。

## 与现实世界是否存在无关

抽象数据类型不要求现实世界中一定存在对应实体。

只要在问题中需要，并且能定义清楚它的数据、关系和操作，就可以抽象出一个类型。

这意味着：

- 可以抽象现实对象。
- 可以抽象数学对象。
- 可以抽象程序运行中的虚拟对象。
- 可以抽象为了算法方便而设计出的对象。

类型设计服务于问题求解，而不是简单照搬现实世界。

## 任意性和无穷性

因为抽象数据类型与具体表示、现实存在都无关，所以它具有很强的任意性和无穷性。

也就是说，理论上可以根据需要定义无数种类型。

但这不意味着类型可以随便乱定义。好的类型抽象应符合[[问题域]]，能帮助解决问题，并且让程序结构更清晰。

## 抽象数据类型的一般表示

可以把一个抽象数据类型写成：

```text
类型名
数据集合
数据关系 / 不变式
操作集合
```

例如：

```text
类型名：日期
数据：年、月、日
不变式：月份 1-12；日期符合月份和闰年规则
操作：创建、比较、下一天、输出
```

这不是 C++ 代码，而是建模层面的描述。

## 不变式

[[不变式]]是一个类型内部数据必须始终满足的约束。

例如：

- 年龄不能为负。
- 分数应在 0 到 100 之间。
- 栈顶位置不能超过数组容量。
- 日期必须是合法日期。

不变式体现了“这个类型什么时候是有效的”。

> [!important] 关键理解
> 类型不只是数据的集合，还包括这些数据必须满足的规则。

## 操作集合

[[操作集合]]描述外部可以对该类型做什么。

例如“账户”类型可能有：

- 存款。
- 取款。
- 查询余额。
- 冻结账户。

这些操作共同定义了这个类型对外表现出的行为。

在面向对象程序设计中，这些操作将逐渐对应到类的成员函数。

## 例子：用 ADT 理解整数

我们平时说整数，可能想到：

```text
..., -2, -1, 0, 1, 2, ...
```

但这种枚举式理解并不严谨，因为整数无限多，无法真正列完。

从抽象数据类型角度，可以把整数理解为：

- `0` 是整数。
- 任意整数的[[后继]]是整数。
- 任意整数的[[前驱]]是整数。
- 操作包括求前驱、求后继等。

这种定义比单纯列举更抽象，也更接近数学模型。

## 同一事物可以抽象成不同类型

对同一个问题对象，可以根据需要抽象出不同类型。

例如“人”这个概念：

如果关注生命过程，可以抽象为：

- 数据：年龄。
- 关系：年龄不为负。
- 操作：出生、成长、死亡。

如果关注学习过程，可以抽象为：

- 数据：知识量、疲劳度。
- 关系：知识量和疲劳度不为负。
- 操作：学习、休息、娱乐。

同样是“人”，抽象出的数据和操作不同，是因为要解决的问题不同。

> [!summary] 抽象原则
> 抽象不是越全越好，而是根据问题需要选取关键数据、关键关系和关键操作。

## ADT 与面向对象的关系

抽象数据类型为后续学习[[类和对象]]做铺垫。

可以先建立对应关系：

| ADT 视角 | C++/OOP 中的对应 |
|---|---|
| 类型名 | 类名 |
| 数据集合 | 数据成员 |
| 不变式 | 类内部约束 |
| 操作集合 | 成员函数 |
| 具体表示 | 类的实现细节 |

后续学习类时，本质上就是在 C++ 中实现某种抽象数据类型。

## 本节速记

> [!summary] 速记
> 抽象数据类型不是具体代码，而是类型的数学化描述：它包含数据集合、数据关系/不变式和操作集合。ADT 与具体表示无关，也不要求现实世界一定存在。面向对象中的类，可以看作对抽象数据类型的一种程序实现。
