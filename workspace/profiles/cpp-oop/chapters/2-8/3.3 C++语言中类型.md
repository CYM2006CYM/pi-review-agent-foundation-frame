# 3.3 C++语言中类型

## 本节核心

本节把前面“面向对象语言中的类型”具体落到 C++。C++ 类型体系大体包括[[内置类型]]、扩展或修饰后的基本类型、[[自定义类型]]、[[导出类型]]、[[泛型编程|泛型]]相关类型等。

> [!important] 核心认识
> C++ 的类型体系很丰富。本课程后续重点是类等自定义类型，但内置类型、`bool`、`auto`、`decltype`、`typedef`、指针、引用等基础类型机制必须能读懂。

## C++类型的大体分类

| 类型类别 | 例子 | 说明 |
|---|---|---|
| [[内置类型]] | `char`、`int`、`float`、`double`、`void`、`bool` | 语言直接提供 |
| 修饰后的基本类型 | `unsigned int`、`long int`、`long double` | 在基本类型上加修饰 |
| [[自定义类型]] | `typedef`、`enum`、`struct`、`union`、`class` | 用户或程序定义 |
| [[导出类型]] | 数组、[[指针]]、[[引用]] | 由已有类型导出形成 |
| 泛型相关类型 | 模板、类模板、函数模板 | 以类型作为参数 |

## 内置基本类型

C++ 中常见[[内置类型]]包括：

- `char`
- `wchar_t`
- `int`
- `float`
- `double`
- `void`
- `bool`

其中 `wchar_t` 是宽字符类型，常用于表示需要多个字节的字符集合。课程中只要求了解其存在，不深入展开编码细节。

`void` 表示“无类型”或“无返回值”，常见于函数返回类型：

```cpp
void printMessage();
```

## 类型修饰

基本类型可以加[[类型修饰符]]形成扩展类型。

常见修饰符包括：

- `short`
- `long`
- `signed`
- `unsigned`

例如：

```cpp
short int
long int
unsigned int
unsigned long int
long double
long long
```

这些修饰符改变数据范围、符号含义或精度。

## bool 类型

C++ 增加了真正的[[bool]]类型。

`bool` 只有两个值：

```cpp
true
false
```

其他类型转换为 `bool` 时，常见规则是：

| 值 | 转换为 bool |
|---|---|
| `0` | `false` |
| 非 `0` | `true` |

这里的 `0` 可以理解为底层表示为零值的状态。指针为空、整数为 0 等都可以转换为 `false`。

> [!warning] 易错点
> `bool` 不是只能由 `0` 和 `1` 表示。它的逻辑值只有 `true/false`，但非零数值转换为 `bool` 时通常都是真。

## 类型推导：auto

C++11 以后，`auto` 可以让编译器根据初始化表达式自动推导变量类型。

例如：

```cpp
int a = 10;
auto b = a + 99;
```

因为 `a + 99` 的结果是 `int`，所以 `b` 被推导为 `int`。

`auto` 的关键点：

- 必须能从初始化表达式推导出类型。
- 类型在编译期确定。
- 不是“运行时动态类型”。

## 类型推导：decltype

[[decltype]]可以根据表达式推导出类型。

例如：

```cpp
int a = 10;
decltype(a + 99) b = a + 99;
```

这里 `decltype(a + 99)` 推导出表达式 `a + 99` 的类型。

再如：

```cpp
#include <cmath>

int a = 10;
decltype(std::sqrt(a)) x = std::sqrt(a);
```

`std::sqrt(a)` 的返回类型会决定 `x` 的类型。

在模板中，`decltype` 常用于表达“返回类型由参数表达式决定”。

## typedef：定义类型别名

[[typedef]]用于给已有类型起一个新名字。

基本格式：

```cpp
typedef 已有类型 新类型名;
```

例如：

```cpp
typedef unsigned char UChar;
typedef unsigned long DWord;
typedef unsigned int UInt;
```

使用时：

```cpp
UChar ch = 'A';   // 等价于 unsigned char ch = 'A';
DWord n = 99;    // 等价于 unsigned long n = 99;
UInt count = 10; // 等价于 unsigned int count = 10;
```

> [!important] 关键点
> `typedef` 本质上没有创造一个全新的、不同的类型，只是给已有类型起了别名，方便书写和阅读。

## typedef 定义函数类型

`typedef` 也可以给函数类型起别名。

例如有两个函数：

```cpp
int f(int a, int b) {
    return a + b;
}

int g(int a, int b) {
    return a * b;
}
```

可以定义函数类型别名：

```cpp
typedef int MyFuncType(int, int);
```

它表示一种函数类型：返回 `int`，参数是两个 `int`。

函数作为参数传递时，常配合[[函数指针]]使用：

```cpp
int apply(MyFuncType* func) {
    return func(88, 99);
}

int x = apply(f);
int y = apply(g);
```

这类写法在 C 语言、兼容 C 的库、底层接口和开源代码中比较常见。现代 C++ 中还会经常使用函数对象、lambda、`std::function` 等更高层机制。

## 自定义类型

C++ 中的[[自定义类型]]包括：

- [[类型别名]]：`typedef` 或 `using`。
- [[枚举]]：`enum`。
- [[结构体]]：`struct`。
- [[联合]]：`union`。
- [[类和对象|类]]：`class`。

其中，本课程面向对象主线最关注的是 `class` 和 `struct` 形成的类型。

## 导出类型

[[导出类型]]是由已有类型进一步构造出来的类型。

常见有：

- [[数组]]
- [[指针]]
- [[引用]]

例如：

```cpp
int a[10];   // int 数组
int* p;      // 指向 int 的指针
int& r = x;  // int 引用
```

引用是 C++ 相对 C 语言新增的重要机制，后续课程会进一步展开。

## 泛型和模板

C++ 中的[[泛型编程]]主要通过[[模板]]实现。

常见有：

- [[类模板]]
- [[函数模板]]

模板允许类型作为参数，例如：

```cpp
template <typename T>
T add(T a, T b) {
    return a + b;
}
```

模板属于后续重点内容，本节只要求知道它属于 C++ 类型体系的一部分。

## 本节速记

> [!summary] 速记
> C++ 类型体系包括内置类型、自定义类型、导出类型和泛型相关类型。`bool` 表示真假，0 假非 0 真。`auto/decltype` 是编译期类型推导。`typedef` 是类型别名，不是本质新增类型。数组、指针、引用属于导出类型。
