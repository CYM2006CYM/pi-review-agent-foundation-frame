# 3.4 C++语言中类型-枚举

## 本节核心

本节主要讲 C++ 中的[[枚举]]，并补充类、结构、联合、数组、指针、引用等类型形式。

枚举的目的，是用有意义的名字代替代码中的裸数字，提高可读性和可维护性。

> [!important] 核心认识
> 传统 `enum` 使用方便，但容易与整数混用、容易发生枚举值命名冲突；C++11 的 `enum class` 更安全，但使用时需要作用域限定和显式转换。

## 为什么需要枚举

如果程序中直接写：

```cpp
int status = 1;
int direction = 2;
```

阅读代码的人很难知道 `1` 和 `2` 分别代表什么。

使用枚举可以写成：

```cpp
enum Weekday {
    Monday,
    Tuesday,
    Wednesday,
    Thursday,
    Friday,
    Saturday,
    Sunday
};
```

这样代码中的名字具有明确语义。

## 传统枚举 enum

传统 C++ 枚举格式：

```cpp
enum 枚举类型名 {
    枚举值1,
    枚举值2,
    枚举值3
};
```

例如：

```cpp
enum Weekday {
    Monday,
    Tuesday,
    Wednesday,
    Thursday,
    Friday,
    Saturday,
    Sunday
};
```

`Weekday` 是枚举类型名，`Monday` 等是[[枚举值]]。

## 枚举值的默认编号

传统枚举的枚举值默认从 `0` 开始，后续值依次加 `1`。

```cpp
enum Weekday {
    Monday,    // 0
    Tuesday,   // 1
    Wednesday  // 2
};
```

也可以显式指定：

```cpp
enum Weekday {
    Monday = 1,
    Tuesday,   // 2
    Wednesday, // 3
    Sunday = 0
};
```

如果某个枚举值没有显式赋值，它默认等于前一个枚举值加 `1`。

> [!warning] 易错点
> 枚举值可以重复取同一个整数值；编译器通常不会因为值相同就报错。

## 传统枚举的底层表示

传统枚举的底层通常按整数处理。

因此传统枚举可以比较方便地输出或参与整数运算：

```cpp
Weekday day = Tuesday;
cout << day << endl;
```

但这种方便也带来问题：如果把星期几拿去随意加 `100`，语法上可能能过，语义上却很奇怪。

```cpp
int x = day + 100; // 可能能编译，但语义不自然
```

## 传统枚举的问题

传统枚举主要有两个问题：

1. 容易和整数混用，类型约束不够强。
2. 枚举值容易污染外层作用域，导致命名冲突。

例如：

```cpp
enum Direction {
    Left,
    Right
};

enum Result {
    Right, // 冲突：Right 已经出现过
    Fail
};
```

这里两个枚举都想使用 `Right`，传统枚举中容易发生名字冲突。

## 强类型枚举 enum class

C++11 引入[[强类型枚举]]，写作：

```cpp
enum class Direction {
    Left,
    Right
};
```

也可以指定底层整数类型：

```cpp
enum class Code : unsigned int {
    A = 65,
    F = 120
};
```

如果不指定底层类型，通常默认为 `int`。

## enum class 的访问方式

强类型枚举的枚举值需要通过类型名限定：

```cpp
Direction d = Direction::Right;
```

这里的 `::` 是[[作用域限定符]]。

如果要输出底层整数值，需要显式转换：

```cpp
cout << static_cast<int>(Direction::Right) << endl;
```

> [!important] 关键区别
> `enum class` 不会把 `Right` 直接暴露到外层作用域，必须写 `Direction::Right`。

## 传统 enum 与 enum class 对比

| 对比项 | 传统 `enum` | `enum class` |
|---|---|---|
| 使用方便性 | 更方便，可与整数混用 | 更严格，需要限定和转换 |
| 类型安全 | 较弱 | 较强 |
| 命名冲突 | 容易冲突 | 不容易冲突 |
| 访问枚举值 | `Right` | `Direction::Right` |
| 转整数 | 常可隐式转换 | 通常需要显式转换 |

一般现代 C++ 更推荐优先考虑 `enum class`。

## 类、结构和联合

本节还补充了三类自定义类型：

- `class`
- `struct`
- `union`

其中 `union` 主要用于多个成员共享同一段内存，在现代 C++ 面向对象课程中不是重点。

`class` 和 `struct` 在 C++ 中能力非常接近，主要区别是默认[[访问控制]]：

| 类型 | 默认访问权限 |
|---|---|
| `class` | [[private]] |
| `struct` | [[public]] |

> [!warning] 易错点
> C++ 中 `struct` 不只是 C 语言结构体，它也可以有成员函数等面向对象特性。

## 类定义格式

类的大体格式：

```cpp
class 类名 {
public:
    // 对外接口

private:
    // 内部数据和实现细节
};
```

注意类定义最后需要分号：

```cpp
class Student {
};
```

少写这个分号是常见语法错误。

## 导出类型：数组、指针、引用

本节最后回到[[导出类型]]。

常见导出类型包括：

```cpp
int a[10]; // 数组
int* p;    // 指针
int& r = x; // 引用
```

## 指针声明怎么读

指针声明可以有不同书写风格：

```cpp
int* p;
int *p;
int * p;
```

三种都能表示 `p` 是指向 `int` 的指针。

课程强调：理解时要以 `*` 为界。

```text
T * p
```

可以读作：

> `p` 是一个指针，它指向的对象类型是 `T`。

当 `T` 变成复杂类型时，这种读法更重要。

## 引用

[[引用]]可以理解为给已有变量起别名。

```cpp
int a = 100;
int& b = a;
```

此后 `a` 和 `b` 指向同一个对象：

- 修改 `a`，`b` 也变。
- 修改 `b`，`a` 也变。

引用是 C++ 中非常重要的机制，后续课程会继续讲。

## 本节速记

> [!summary] 速记
> 枚举用名字表示有限取值。传统 `enum` 默认从 0 开始、值依次加 1，方便但不够安全；`enum class` 强类型、更少冲突，但要写 `类型名::枚举值`，转整数需显式转换。`class` 默认 private，`struct` 默认 public。引用是变量别名。
