# 3.5 声明和定义

## 本节核心

本节讲 C++ 中非常基础但容易混淆的两个概念：[[声明]]和[[定义]]。

> [!important] 核心认识
> 声明告诉编译器“这个名字是什么”；定义在声明的基础上给出实体本身，例如分配存储空间、给出函数体、给出类的完整结构。

## 什么是声明

[[声明]]的作用是说明一个名字的含义和属性。

程序中的名字可能表示：

- 变量。
- 常量。
- 函数。
- 类。
- 结构体。
- 枚举。
- 类型别名。
- 名字空间成员。

编译器读到声明后，就能知道这个[[标识符]]代表什么。

例如：

```cpp
extern int a;
```

这告诉编译器：

- `a` 是一个名字。
- `a` 表示 `int` 类型变量。
- 它具有[[外部连接]]。
- 它的定义可能在别处。

## 声明的例子

```cpp
extern int a;          // 变量声明
int f(int);            // 函数声明
class Student;         // 类的前置声明
typedef unsigned int UInt; // 类型别名声明
```

这些语句的共同点是：它们让编译器认识某个名字。

其中有些声明同时也是定义，后面会区分。

## 什么是定义

[[定义]]首先也是一种声明，但它比普通声明更进一步。

定义会给出实体的确定内容，例如：

- 给变量分配[[存储空间]]。
- 给函数提供[[函数体]]。
- 给类或结构体提供完整成员列表。
- 给枚举列出枚举值。
- 给名字空间定义成员。

例如：

```cpp
int a;
```

这是变量定义，因为它不仅说明 `a` 是 `int`，还会为 `a` 分配存储空间。

```cpp
int f(int x) {
    return x + 1;
}
```

这是函数定义，因为它给出了函数体。

## 声明和定义的直观区别

| 问题 | 声明 | 定义 |
|---|---|---|
| 是否告诉编译器名字含义？ | 是 | 是 |
| 是否给出实体内容？ | 不一定 | 是 |
| 是否分配变量存储空间？ | 通常不 | 通常是 |
| 是否给出函数体？ | 不给 | 给 |
| 是否给出类完整成员？ | 前置声明不给 | 类定义给 |

> [!summary] 简化理解
> 声明偏“告诉你有这个东西”；定义偏“这个东西就在这里，内容也给你了”。

## 变量声明与变量定义

变量声明：

```cpp
extern int a;
```

含义是：存在一个外部的 `int` 变量 `a`，具体定义可能在某个 `.cpp` 文件中。

变量定义：

```cpp
int a;
```

含义是：定义一个 `int` 变量 `a`，并为它分配存储空间。

带初始化的变量定义：

```cpp
int c = 1;
```

这不仅分配空间，还给出初始值。

> [!warning] 易错点
> `extern int a;` 通常只是声明；`int a;` 通常是定义。少了 `extern`，含义就不同。

## 函数声明与函数定义

函数声明也称函数原型：

```cpp
int myFunc(int);
```

它告诉编译器：

- `myFunc` 是函数。
- 返回值是 `int`。
- 参数是一个 `int`。

但它没有函数体。

函数定义：

```cpp
int myFunc(int x) {
    return x + 1;
}
```

它给出了函数体。编译时需要为函数体生成代码，链接时其他调用才能找到这个函数实现。

## 类声明与类定义

类的前置声明：

```cpp
class Student;
```

它只告诉编译器 `Student` 是一个类名，不说明类内部有什么。

类定义：

```cpp
class Student {
public:
    void study();

private:
    int age;
};
```

它给出了类的完整结构，包括成员函数声明和数据成员。

这就是[[前置声明]]和完整定义的区别。

## typedef 是什么

```cpp
typedef unsigned int UInt;
```

它声明了一个[[类型别名]]：`UInt` 表示 `unsigned int`。

课程中把它放在声明例子中理解即可。它让编译器知道 `UInt` 这个名字代表某种类型。

## 为什么要区分声明和定义

区分声明和定义，是理解后续内容的基础：

- [[头文件]]中通常放声明。
- [[实现文件]]中通常放定义。
- 同一个函数可以声明多次，但定义通常只能有一个。
- 同一个全局变量如果在多个 `.cpp` 中定义，会导致链接问题。
- 后续的[[单一定义规则]]建立在这个区分上。

> [!important] 工程规则
> 让多个文件都“知道一个名字”靠声明；让程序里真正“有这个实体”靠定义。

## 图示化判断：常见声明与定义

复习时可以把常见写法按“只说明名字”与“给出实体”来判断：

| 写法 | 类型 | 关键理由 |
|---|---|---|
| `extern int a;` | 变量声明 | 说明外部有 `a`，这里不分配空间 |
| `int a;` | 变量定义 | 为 `a` 分配存储空间 |
| `extern const int c;` | 常量声明 | 声明外部常量名 |
| `extern const int c = 1;` | 常量定义 | 有初始化，给出实体 |
| `int f(int);` | 函数声明 | 只有函数原型，没有函数体 |
| `int f(int x) { return x; }` | 函数定义 | 有函数体 |
| `struct S;` | 类型声明 | 只说明有这个结构名 |
| `struct S { int x; };` | 类型定义 | 给出完整成员结构 |
| `typedef int Int;` | 类型别名声明 | 引入一个类型名字 |
| `namespace N { int a; }` | 名字空间成员定义 | 在名字空间中给出实体 |

> [!warning] 易错点
> 类定义中的成员函数声明不等于成员函数定义；但类定义本身已经给出了类的完整结构，所以 `class A { ... };` 是类定义。

## extern 与外部连接

`extern` 表示外部连接声明。

```cpp
extern int a;
```

可以理解为：

> 编译器，这里有一个 `int` 变量 `a`，但它的存储空间不在这里分配；真正定义会在工程中的某处出现。

对应定义通常写在某个 `.cpp` 文件中：

```cpp
int a = 0;
```

这也是为什么全局变量一般不应该直接定义在头文件中。

## 本节考点整理

| 可能题型 | 可能问法 | 答题要点 |
|---|---|---|
| 名词解释 | 什么是声明？ | 说明名字的含义和属性 |
| 名词解释 | 什么是定义？ | 声明并给出实体内容，如存储空间、函数体、类完整结构 |
| 判断题 | 所有定义都是声明。 | 对 |
| 判断题 | 所有声明都是定义。 | 错 |
| 选择题 | `extern int a;` 通常是什么？ | 变量声明 |
| 选择题 | `int a;` 通常是什么？ | 变量定义 |
| 区分题 | 函数声明和函数定义的区别是什么？ | 是否有函数体 |
| 区分题 | 类前置声明和类定义的区别是什么？ | 是否给出完整成员结构 |

## 本节速记

> [!summary] 速记
> 声明让编译器认识名字；定义给出实体本身。`extern int a;` 是声明，`int a;` 是定义；`int f(int);` 是函数声明，带 `{}` 函数体的是定义；`class A;` 是类声明，带完整成员列表的是类定义。
