# 3.6 变量的使用原则

## 本节核心

本节讲变量使用的几个基本原则，重点是：

- [[就近原则]]：变量尽量在需要使用的位置附近定义。
- [[先声明后使用]]：任何名字在使用前必须让编译器见过声明。
- [[单一定义规则]]：后续单独展开。

> [!important] 核心认识
> 就近定义是现代 C++ 的代码风格原则；先声明后使用是编译器能否识别名字的基本要求。

## 就近原则

[[就近原则]]指：变量应该尽量在第一次需要使用的位置附近定义，而不是全部堆在函数开头。

旧式 C 代码中常见写法：

```cpp
int main() {
    int i;
    int j;
    int k;

    // 很多代码之后才使用 i、j、k
}
```

现代 C++ 更建议：

```cpp
int main() {
    for (int i = 0; i < 10; ++i) {
        // i 只在循环中使用
    }

    int value = 0;
    cin >> value;

    if (value > 0) {
        int temp = value * 2;
        cout << temp << endl;
    }
}
```

变量在靠近使用位置的地方定义，读代码时更容易看出它的用途和类型。

## 就近定义的好处

就近定义有几个好处：

- 变量含义更清晰。
- 不需要反复回到函数开头查类型。
- 变量[[作用域]]更小。
- 减少误用变量的机会。
- 同名临时变量可以在不同小作用域中复用。

例如：

```cpp
for (int i = 0; i < 10; ++i) {
    // i 的作用域只在这个 for 循环中
}

for (int i = 0; i < 5; ++i) {
    // 这里可以重新定义另一个 i
}
```

如果把 `i` 提前定义在函数开头，它的作用域可能贯穿整个函数，反而更容易产生混乱。

## 作用域

[[作用域]]决定一个名字在哪一段代码中有效。

例如：

```cpp
{
    int x = 1;
    cout << x << endl;
}

// 这里不能再使用 x
```

`x` 只在它所在的大括号内部有效。离开这个大括号后，`x` 就不再可见。

> [!tip] 初学者理解
> 变量定义在哪个 `{}` 里，通常就主要在哪个 `{}` 里有效。

## 先声明后使用

[[先声明后使用]]指：一个名字在被使用之前，编译器必须已经见过它的[[声明]]。

例如：

```cpp
int count = 0;

void f() {
    count = 10; // 可以使用，因为前面已经有 count 的定义
}
```

如果在当前编译单元中直接写：

```cpp
void f() {
    count = 10; // 错误：编译器还不认识 count
}
```

编译器会报错，因为它不知道 `count` 是什么。

## 跨文件使用变量：extern

假设 `a.cpp` 中定义了一个全局变量：

```cpp
// a.cpp
int count = 0;
```

如果 `b.cpp` 想使用它，不能直接写：

```cpp
// b.cpp
count = 10; // 错误：b.cpp 独立编译时不认识 count
```

需要先声明：

```cpp
// b.cpp
extern int count;

void f() {
    count = 10;
}
```

`extern int count;` 告诉编译器：`count` 是一个外部的 `int` 变量，真正定义在别处。

> [!warning] 易错点
> 如果在 `b.cpp` 中写 `int count;`，那不是声明外部变量，而是在 `b.cpp` 中又定义了一个 `count`，可能导致重复定义。

## 先声明不等于先定义

本节强调“先声明后使用”，不是“先定义后使用”。

例如函数可以先声明，后面再定义：

```cpp
int add(int, int); // 声明

int main() {
    return add(1, 2); // 可以使用
}

int add(int a, int b) { // 定义
    return a + b;
}
```

编译器在 `main` 中看到 `add(1, 2)` 时，已经通过声明知道 `add` 是什么，因此可以继续编译。

## 类的前置声明也是先声明后使用

类名也必须先声明后使用。

例如：

```cpp
class Bone;

class Dog {
public:
    void eat(Bone* bone);
};
```

这里 `Bone` 在作为参数类型出现前，先通过 `class Bone;` 做了[[前置声明]]。

如果没有前置声明，编译器遇到 `Bone*` 时就不知道 `Bone` 是什么。

## 与编译单元的关系

每个 `.cpp` 文件独立编译。因此“先声明后使用”是对每个[[编译单元]]分别成立的。

`a.cpp` 中定义了 `count`，不代表 `b.cpp` 编译时自动知道 `count`。  
`b.cpp` 要使用，就必须通过头文件或 `extern` 声明让自己先知道这个名字。

这也是为什么工程中需要头文件：

- 头文件放声明。
- 多个 `.cpp` 包含头文件。
- 每个 `.cpp` 独立编译时都能识别相关名字。

## 本节速记

> [!summary] 速记
> 变量尽量就近定义，作用域越小越清晰。任何名字必须先声明后使用，编译器才知道它是什么。跨文件使用变量时用 `extern` 声明，不能把声明误写成新的定义。
