# 3.8 变量的存储与作用域

## 本节核心

本节把[[变量]]放到内存模型中理解：一个变量不仅有类型和值，还涉及它存放在哪里、名字在哪里可见、对象能活多久。

- 普通局部变量通常存放在[[栈区]]，离开作用域后自动销毁。
- [[静态局部变量]]的名字只在局部作用域可见，但对象通常存放在[[全局数据区]]，只初始化一次。
- 程序代码存放在[[代码区]]，代码通过变量地址访问不同位置的数据。
- `new` 创建的对象存放在[[堆区]]，指针变量本身仍可能是栈上的局部变量。
- `delete p;` 释放的是 `p` 指向的堆对象，不是销毁指针变量 `p` 本身。

> [!important] 核心认识
> 作用域说的是“名字能在哪里用”，存储位置说的是“对象放在内存哪里”，生命周期说的是“对象什么时候创建和销毁”。这三件事相关，但不是一回事。

## 静态局部变量

函数内部可以定义[[静态局部变量]]：

```cpp
void f() {
    static int n = 0;
    ++n;
}
```

它有两个特点：

- `n` 的[[作用域]]仍然只在函数 `f` 内部。
- `n` 的存储位置通常在[[全局数据区]]，生命周期持续到程序结束。

普通局部变量每次进入函数都会重新创建：

```cpp
void g() {
    int n = 0;
    ++n;
}
```

而静态局部变量只在第一次执行到定义处时初始化一次，之后再次进入函数会跳过初始化，继续使用原来的值。

## 作用域与生命周期不要混淆

静态局部变量很适合说明“作用域”和“生命周期”的区别。

```cpp
void f() {
    static int n = 0;
    ++n;
}
```

`n` 的名字只在 `f` 的大括号内能用，这叫作用域。

但 `n` 对应的对象不会在 `f` 返回时销毁，它会一直存在到程序结束，这叫生命周期。

> [!warning] 易错点
> 不能因为一个变量“只在函数里能访问”，就认为它一定存放在栈上，也不能认为函数返回后它一定消失。`static` 局部变量就是反例。

## 代码区和数据区

程序运行时，代码和数据是两类不同的东西。

```cpp
int main() {
    int a = 10;
    static int b = 20;
    return a + b;
}
```

可以这样理解：

- 函数指令本身放在[[代码区]]。
- 普通局部变量 `a` 通常放在[[栈区]]。
- 静态局部变量 `b` 通常放在[[全局数据区]]。
- 字面值 `20` 可以理解为写在代码中的常量数据。

代码是固定的指令，真正变化的是它访问的数据。执行 `a + b` 时，代码会分别从 `a` 和 `b` 所在的地址取值。

## 堆区中的对象与栈上的指针

动态分配示例：

```cpp
int* p = new int(55);
```

这行代码包含两件事：

- `new int(55)` 在[[堆区]]分配一个 `int` 对象，并把它初始化为 `55`。
- `p` 是一个局部[[指针]]变量，通常存放在[[栈区]]。

也就是说：

```text
栈区：p 这个指针变量，里面保存一个地址
堆区：地址指向的 int 对象，值为 55
```

`p` 本身不是堆对象，它只是保存堆对象地址的变量。

## delete 释放的是什么

释放动态对象：

```cpp
delete p;
```

`delete p;` 的含义是：释放 `p` 里面保存的地址所指向的堆对象。

它不是把栈上的指针变量 `p` 本身删除掉。

```cpp
int* p = new int(55);
delete p; // 释放堆上的 int

// p 这个局部变量本身仍然存在，但里面的地址已经不能再当作有效对象使用
```

> [!warning] 易错点
> `delete` 之后，指针变量还在，但它指向的对象已经被释放。继续解引用这个指针会造成未定义行为。

## new/delete 与 malloc/free

课程提醒：C++ 中进行动态内存管理时，主要使用 `new` 和 `delete`，不推荐混用 C 风格的 `malloc/free`。

原因可以先记为：

- `new` 不只是分配内存，还会进行对象初始化。
- `delete` 不只是释放内存，还会配合对象析构。
- C++ 面向对象程序中，直接使用 `malloc/free` 容易绕开对象构造和析构机制。

后续学习[[动态内存管理]]时会进一步展开。

## 本节速记

> [!summary] 速记
> 普通局部变量多在栈上，静态局部变量名字局部可见但对象长期存在，动态分配对象在堆上。指针变量和它指向的对象不是一个东西：`int* p = new int(55);` 中，`p` 通常在栈上，`55` 对应的对象在堆上；`delete p;` 释放的是堆对象。
