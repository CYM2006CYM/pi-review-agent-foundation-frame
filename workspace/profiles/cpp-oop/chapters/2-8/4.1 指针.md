# 4.1 指针

## 本节核心

本节回顾 C++ 中[[指针]]的基本语法和语义，并把它与数组的底层访问联系起来。

- 指针变量保存的是地址。
- 指针类型要理解为“指向某种类型对象的指针”。
- 现代 C++ 中空指针优先使用 [[nullptr]]，不要继续依赖 `NULL` 或数值 `0`。
- `*p` 表示[[解引用]]，即访问 `p` 指向的对象。
- [[指针运算]]按指向类型的大小移动，不是简单按字节加一。
- 数组下标访问本质上可理解为“首地址 + 偏移 + 解引用”。

> [!important] 核心认识
> 指针的关键不是星号写在哪里，而是想清楚：这个变量保存一个地址，这个地址指向什么类型的对象。

## 指针变量的基本定义

指针变量的基本形式：

```cpp
T* p;
```

含义是：`p` 是一个指针变量，它指向的对象类型是 `T`。

例如：

```cpp
int value = 10;
int* p = &value;
```

这里：

- `value` 是一个 `int` 变量。
- `&value` 取得 `value` 的地址。
- `p` 保存这个地址。
- `p` 的类型是 `int*`。

## 指针要尽量初始化

指针变量如果没有初始化，里面可能保存不可预期的地址。

不推荐：

```cpp
int* p; // 未初始化，危险
```

推荐：

```cpp
int* p = nullptr;
```

或者定义时直接指向有效对象：

```cpp
int value = 10;
int* p = &value;
```

> [!warning] 易错点
> 未初始化指针不是“空指针”，而是值不确定的指针。直接解引用未初始化指针会导致严重错误。

## nullptr 与 NULL

早期 C/C++ 中常用 `NULL` 表示空指针：

```cpp
int* p = NULL;
```

但 `NULL` 在很多实现中本质上是宏定义的 `0`，没有明确的指针类型，在重载、模板等场景中容易引发类型问题。

现代 C++ 推荐使用：

```cpp
int* p = nullptr;
```

[[nullptr]] 有明确的空指针语义，类型更安全。

## 指针的指针

指针本身也是变量，也有地址。因此可以定义“指向指针的指针”：

```cpp
int value = 10;
int* p = &value;
int** pp = &p;
```

含义是：

- `p` 指向 `int` 对象 `value`。
- `pp` 指向 `int*` 指针变量 `p`。

可以按层次理解：

```text
pp -> p -> value
```

## 解引用

[[解引用]]指通过指针访问它指向的对象。

```cpp
int a = 10;
int b = 20;

int* p = &a;
*p = b;
```

`*p = b;` 的意思不是修改 `p` 这个指针变量，而是修改 `p` 指向的对象。因为 `p` 指向 `a`，所以这句执行后 `a` 变成 `20`。

> [!tip] 初学者理解
> `p` 是地址，`*p` 是地址处的对象。

## 指针运算

指针可以做加减运算：

```cpp
int a[3]{1, 2, 3};
int* p = &a[0];

int* q = p + 1;
```

这里 `p + 1` 的含义不是“地址数值加 1 个字节”，而是移动到下一个 `int` 元素的位置。

一般规律：

```cpp
p + n
```

表示从 `p` 当前指向的位置开始，向后移动 `n` 个 `T` 类型元素。如果 `p` 是 `T*`，底层地址偏移量大致是：

```text
n * sizeof(T)
```

所以：

```cpp
int* p;     // p + 1 移动 sizeof(int) 个字节
double* q;  // q + 1 移动 sizeof(double) 个字节
```

## 指针运算与数组下标

数组元素在内存中连续存放。正因为连续，指针加法才能用于访问下一个元素。

```cpp
int a[3]{1, 2, 3};
```

数组下标访问：

```cpp
a[2]
```

可以理解为：

```cpp
*(a + 2)
```

含义是：

1. 取得数组首元素地址。
2. 按元素大小偏移 2 个位置。
3. 解引用得到该位置的元素。

> [!important] 高频考点
> 数组下标访问依赖数组元素连续存储。`a[k]` 可以理解为 `*(a + k)`。
