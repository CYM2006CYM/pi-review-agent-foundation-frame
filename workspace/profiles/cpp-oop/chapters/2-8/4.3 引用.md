# 4.3 引用

## 本节核心

[[引用]]是 C++ 引入的重要概念，可以把它理解为“给已有对象起别名”。

- 引用定义时必须绑定到一个真实对象。
- 引用一旦绑定，就在自己的作用域内作为该对象的别名使用。
- 通过引用修改值，等同于修改被引用的原对象。
- 引用常用于[[函数参数]]，让函数可以直接操作实参，同时调用形式保持普通变量传参的样子。
- 引用和指针都能间接操作对象，但使用方式、初始化要求不同。

> [!important] 核心认识
> 引用不是“另一个独立变量”，而是已有对象的别名。`int& b = a;` 之后，在 `b` 的作用域里，操作 `b` 就是在操作 `a`。

## 引用的基本语法

引用定义形式：

```cpp
T& ref = object;
```

例如：

```cpp
int a = 10;
int& b = a;
```

这里 `b` 是 `a` 的引用，也就是 `a` 的别名。

```cpp
a = 6;
cout << b << endl; // 6

b = 5;
cout << a << endl; // 5
```

对 `a` 的修改会从 `b` 看见，对 `b` 的修改也会反映到 `a` 上，因为它们代表同一个对象。

## 引用必须初始化

引用定义时必须绑定到一个已有对象：

```cpp
int a = 10;
int& b = a; // 正确
```

不能只声明一个没有绑定对象的引用：

```cpp
int& b; // 错误：引用必须初始化
```

原因是引用的含义是“别名”。没有真实对象，就不能凭空起别名。

> [!warning] 易错点
> 指针可以先设为空指针，例如 `int* p = nullptr;`；引用没有“空引用”这种常规安全状态，定义时必须绑定有效对象。

## 引用和指针访问对象的区别

指针需要通过 `*p` [[解引用]]访问对象：

```cpp
int a = 10;
int* p = &a;
*p = 20;
```

引用可以直接像原变量一样使用：

```cpp
int a = 10;
int& r = a;
r = 20;
```

两段代码都会把 `a` 改成 `20`，但引用的使用形式更接近普通变量。

## 引用作为函数参数

引用最常见的用途之一是函数参数。

如果按值传参：

```cpp
void f(int a, int b) {
    a = 1;
    b = 2;
}

int x = 10;
int y = 20;
f(x, y);
```

函数内部修改的是形参副本，不会修改 `x` 和 `y`。

如果用指针传参：

```cpp
void f(int* a, int* b) {
    *a = 1;
    *b = 2;
}

f(&x, &y);
```

可以修改实参，但调用方必须显式传地址。

引用传参写法：

```cpp
void f(int& a, int& b) {
    a = 1;
    b = 2;
}

f(x, y);
```

调用方仍然写 `f(x, y)`，函数内部却可以直接修改实参。

## 引用传参的意义

引用传参的价值在于：

- 调用形式和普通传参一致。
- 函数实现可以决定是否修改实参。
- 不需要调用方写 `&x`、`&y`。
- 函数内部也不需要写 `*p` 进行解引用。

> [!warning] 易错点
> 引用参数可能修改实参。看到函数声明 `void f(int& a);` 时，要意识到调用 `f(x)` 后 `x` 可能被改变。

## 引用和指针对比

| 对比项 | 指针 | 引用 |
|---|---|---|
| 定义形式 | `int* p = &a;` | `int& r = a;` |
| 是否必须初始化 | 可以不初始化，但不推荐 | 必须初始化 |
| 是否可以为空 | 可以是 `nullptr` | 通常没有空引用 |
| 访问对象 | `*p` | 直接用 `r` |
| 主要用途 | 地址操作、动态内存、可空关系 | 别名、函数参数、避免拷贝 |
| 调用形式 | `f(&a)` | `f(a)` |

## 本节速记

> [!summary] 速记
> 引用就是别名，必须一开始就绑定到真实对象。`int& b = a;` 后，`a` 和 `b` 代表同一个对象。引用传参让函数可以修改实参，但调用方仍写普通的 `f(a, b)`，不用显式传地址。
