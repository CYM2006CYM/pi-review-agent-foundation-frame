# 4.4 define和宏

## 本节核心

本节从“代码中不要到处裸写数字”引出[[宏定义]]，再说明宏的本质和常见用途。

- 直接写在代码里的 `1`、`2.5`、`3.14` 等属于[[字面常量]]。
- 有明确含义的字面常量不宜到处裸写，否则代码难理解、难维护。
- `#define` 可以定义宏，但宏本质是[[宏替换]]，没有类型检查。
- C++ 中表示有类型的常量，更推荐后续要讲的[[命名常量]]。
- 宏仍有重要用途，例如[[条件编译]]、字符串化、符号连接、调试信息等。

> [!important] 核心认识
> 宏不是变量，也不是普通常量；宏发生在预处理阶段，本质是文本替换。

## 不要滥用裸字面常量

代码中常见这样的写法：

```cpp
double area = 3.14 * 2.5 * 2.5;
double volume = 3.14 * 2.5 * 2.5 * 2.5 * 3 / 4;
```

从数学知识看，`3.14` 可能代表圆周率，`2.5` 可能代表半径，但代码本身没有表达这些含义。

这种写法有两个问题：

- 可读性差：读代码的人需要猜数字含义。
- 可维护性差：如果圆周率精度要改，所有 `3.14` 都要找出来修改，而且容易误改不相关的数字。

更好的做法是用有名字的常量表达含义。

## 字面常量

[[字面常量]]是直接写在源代码中的常量值，例如：

```cpp
1
2.5
3.14
'a'
"hello"
true
false
```

字面常量不是不能出现，而是有业务含义或需要反复使用的数字不应该裸写。

> [!tip] 初学者判断
> `0`、`1` 作为循环下标或布尔判断有时很自然；但 `3.14`、`86400`、`1024` 这类有特定含义的值，通常应该给名字。

## 使用 define 定义宏

可以用 `#define` 给某段文本起名字：

```cpp
#define PI 3.1415926
```

之后源文件中出现 `PI` 的地方，预处理阶段会被替换为 `3.1415926`。

这就是[[宏替换]]。

```cpp
double area = PI * r * r;
```

预处理后近似变成：

```cpp
double area = 3.1415926 * r * r;
```

## 宏没有类型检查

宏是预处理文本替换，不是 C++ 的类型系统对象。

```cpp
#define PI 3.1415926
```

`PI` 不是一个 `double` 变量，也不是有作用域的常量对象。预处理器只是把文本替换进去。

因此宏有缺点：

- 没有类型检查。
- 不遵守普通变量那样的作用域规则。
- 替换结果可能在编译阶段才暴露错误。
- 调试时不如有类型的常量清晰。

所以 C++ 中如果只是定义常量，后面更推荐使用[[命名常量]]，例如 `const` 或现代 C++ 的 `constexpr`。

## 宏仍然有用：条件编译

宏的一个重要用途是[[条件编译]]。

```cpp
#define DEBUG

#ifdef DEBUG
cout << "debug mode" << endl;
#else
cout << "release mode" << endl;
#endif
```

如果 `DEBUG` 已定义，预处理后保留第一段输出；如果没有定义，则保留 `#else` 后面的部分。

注意：条件编译发生在预处理阶段，被跳过的代码在进入真正编译前就已经被删掉了。

这和运行时 `if` 不同：

```cpp
if (debug) {
    cout << "debug mode" << endl;
}
```

运行时 `if` 的两条分支代码通常都参与编译，只是在运行时决定走哪条。

## # 字符串化

在函数式宏中，`#` 可以把宏参数转成字符串，这叫[[字符串化]]。

```cpp
#define STR(x) #x

cout << STR(abc) << endl; // 输出 "abc"
cout << STR(123) << endl; // 输出 "123"
```

`STR(abc)` 不是访问变量 `abc`，而是把参数文本变成字符串 `"abc"`。

## ## 符号连接

`##` 可以把两个记号连接成一个新记号，这叫[[记号连接]]。

```cpp
#define VAR(x) n##x

int n1 = 2;
int n2 = 4;

cout << VAR(2) << endl; // 等价于 cout << n2 << endl;
```

`VAR(2)` 展开时，`n` 和 `2` 被连接成 `n2`。

## 内置宏

C/C++ 提供一些常见[[内置宏]]，用于获得当前代码位置相关信息：

```cpp
__LINE__   // 当前行号
__FILE__   // 当前文件名
__func__   // 当前函数名，C++ 中常用
```

它们常用于错误信息、日志输出、调试辅助。

例如：

```cpp
cout << __FILE__ << ":" << __LINE__ << endl;
```

编译器报错时能指出“哪个文件、哪一行”，背后也和这类位置信息有关。

## 本节速记

> [!summary] 速记
> 有含义的数字不要裸写。`#define` 可以起名，但宏只是预处理文本替换，没有类型检查；单纯表示常量时，更推荐后续的命名常量。宏的重要价值在条件编译、字符串化 `#`、记号连接 `##` 和调试定位信息。
