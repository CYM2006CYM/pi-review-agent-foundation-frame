# 4.5 命名常量

## 本节核心

[[命名常量]]是给常量值起一个有意义的名字，避免代码中到处出现难以理解的[[字面常量]]。

- 最常见写法是使用 [[const]]。
- 常量名通常使用全大写，多个单词用下划线连接。
- [[枚举]]和[[强类型枚举]]也可以用于给一组相关常量起名。
- `const` 命名常量可以放在头文件中供多个 `.cpp` 使用。
- 编译器可通过[[常量折叠]]避免普通变量那样的重复定义问题。

> [!important] 核心认识
> 命名常量的价值是“表达含义”和“集中维护”。`CARD_COUNT` 比裸写 `54` 更清楚。

## const 命名常量

常见写法：

```cpp
const int CARD_COUNT = 54;
```

含义是：定义一个整型常量 `CARD_COUNT`，值为 `54`，之后不能再修改。

不允许：

```cpp
CARD_COUNT = 60; // 错误
```

命名习惯上，常量常写成全大写：

```cpp
const double PI = 3.1415926;
const int MAX_BUFFER_SIZE = 1024;
```

## 枚举也能表达命名常量

枚举可以给一组相关常量起名：

```cpp
enum Color {
    RED = 0xff0000,
    GREEN = 0x00ff00,
    BLUE = 0x0000ff
};
```

现代 C++ 更推荐[[强类型枚举]]：

```cpp
enum class Color {
    Red,
    Green,
    Blue
};
```

枚举适合表示一组有限、相关的取值；`const` 更适合表示单个有意义的常量值。

## 命名常量放在哪里

如果命名常量只在一个 `.cpp` 中使用，可以放在该 `.cpp` 中。

如果多个 `.cpp` 都要使用，例如一个扑克牌程序中多个模块都需要 `CARD_COUNT`，就希望把它放到公共头文件中：

```cpp
// card_config.h
const int CARD_COUNT = 54;
```

多个 `.cpp` 包含这个头文件后都能使用 `CARD_COUNT`。

## 为什么放头文件通常不冲突

前面讲过[[头文件只放声明]]，普通全局变量定义放头文件会导致[[重复定义]]。

那为什么 `const int CARD_COUNT = 54;` 可以放头文件？

原因是：这种命名常量通常会在编译期被处理成[[常量折叠]]。

例如：

```cpp
int cards[CARD_COUNT];
```

编译器可以直接把 `CARD_COUNT` 替换或折叠为 `54` 来使用，不一定为它分配普通全局变量那样的存储空间。因此多个编译单元包含同一个 `const` 常量定义，通常不会像普通全局变量那样在链接时冲突。

> [!warning] 易错点
> 这里说的是适合编译期使用的 `const` 命名常量。普通可变全局变量仍然不能直接定义在公共头文件中。
