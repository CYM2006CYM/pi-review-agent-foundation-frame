# 4.6 const和指针

## 本节核心

[[const]] 和[[指针]]结合时，重点看 `const` 限制的是“指针本身”还是“指针指向的对象”。

- `const int* p` 或 `int const* p`：[[指向常量的指针]]，不能通过 `p` 修改对象，但 `p` 可以改指向。
- `int* const p`：[[常量指针]]，`p` 不能改指向，但可以通过 `p` 修改对象。
- `const int* const p`：[[指向常量的常量指针]]，既不能改指向，也不能通过 `p` 修改对象。
- 字符串字面量通常应看作常量数据，指针写法应使用 `const char*`。

> [!important] 读法
> 看到 `*` 时分两边看：`*` 左边修饰“指向的东西”，`*` 右边修饰“指针变量本身”。

## 指向常量的指针

两种写法等价：

```cpp
const int* p;
int const* p;
```

读作：`p` 是一个指针，指向的 `int` 通过 `p` 看是常量。

示例：

```cpp
int v1 = 100;
int v2 = 200;

const int* p = &v1;

p = &v2;   // 可以：p 本身可以改指向
*p = 300;  // 错误：不能通过 p 修改指向的对象
```

注意：对象本身不一定真是常量。

```cpp
int v = 100;
const int* p = &v;

v = 99;     // 可以，通过原变量改
// *p = 99; // 不可以，通过 p 改
```

`const int* p` 的限制是：不能通过 `p` 修改对象。

## 不能丢掉 const 保护

如果有：

```cpp
const int* p1 = &v;
int* p2 = p1; // 错误
```

原因是：`p1` 承诺不能通过它修改对象；如果允许赋给普通 `int*`，就可以通过 `p2` 修改同一个对象，破坏了 `const` 保护。

## 常量指针

写法：

```cpp
int* const p = &v;
```

读作：`p` 是一个常量指针，它保存的地址不能再改变。

示例：

```cpp
int v1 = 100;
int v2 = 200;

int* const p = &v1;

*p = 300; // 可以：指向的对象可改
p = &v2;  // 错误：p 本身不能改指向
```

因为指针本身是常量，所以定义时必须初始化。

## 指向常量的常量指针

写法：

```cpp
const int* const p = &v;
```

读作：`p` 是常量指针，指向的对象也不能通过 `p` 修改。

```cpp
const int* const p = &v;

*p = 1;   // 错误
p = &v2;  // 错误
```

这是最严格的组合：不能改指向，也不能通过它改值。

## 三种写法对比

| 写法 | 指针能否改指向 | 能否通过指针改对象 | 读法 |
|---|---|---|---|
| `int* p` | 可以 | 可以 | 普通指针 |
| `const int* p` | 可以 | 不可以 | 指向常量的指针 |
| `int* const p` | 不可以 | 可以 | 常量指针 |
| `const int* const p` | 不可以 | 不可以 | 指向常量的常量指针 |

## 字符串字面量和 const char*

字符串字面量通常存放在常量数据区，不应通过指针修改。

推荐写法：

```cpp
const char* str = "Hello";
```

不应通过 `str` 修改字符串内容：

```cpp
str[0] = 'h'; // 错误或未定义行为风险
```

如果确实希望修改字符内容，应使用字符数组：

```cpp
char str[] = "Hello";
str[0] = 'h'; // 可以
```

区别是：

- `const char* str = "Hello";`：指向常量字符串数据。
- `char str[] = "Hello";`：把字符串内容复制到数组中，数组元素可修改。

## string 的提醒

C++ 标准库中有 `std::string`，使用方便，但内部实现涉及类、模板、动态存储等机制。课程此处提醒：如果当前目标是理解底层存储和指针关系，可以先用 `char*`、`const char*`、字符数组理解基础。

## 本节考点整理

| 可能题型 | 可能问法 | 答题要点 |
|---|---|---|
| 代码题 | `const int* p` 限制什么？ | 不能通过 `p` 修改指向对象，`p` 可改指向 |
| 代码题 | `int* const p` 限制什么？ | `p` 不能改指向，可通过 `p` 改对象 |
| 代码题 | `const int* const p` 限制什么？ | 指向和值都受限制 |
| 判断题 | `const int* p` 和 `int const* p` 等价。 | 对 |
| 判断题 | `const int* p` 指向的普通变量不能通过原变量修改。 | 错，不能通过 `p` 修改，但原变量可改 |
| 判断题 | `const int*` 可以赋给 `int*`。 | 错，会丢掉 const 保护 |
| 选择题 | 可修改字符串内容应使用哪种形式？ | `char str[] = "..."` |
| 判断题 | 字符串字面量可以随便通过 `char*` 修改。 | 错 |

## 本节速记

> [!summary] 速记
> `const` 在 `*` 左边，限制指向的对象；`const` 在 `*` 右边，限制指针本身。`const int* p` 可换指向但不能通过它改值；`int* const p` 不能换指向但可改值；`const int* const p` 两者都不能。字符串字面量用 `const char*`，要改内容用字符数组。
