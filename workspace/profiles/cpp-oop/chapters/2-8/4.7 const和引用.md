# 4.7 const和引用

## 本节核心

[[const引用]]表示“不能通过这个引用修改被引用对象”。

- 普通引用 `int& r = v;` 可以通过 `r` 修改 `v`。
- 常量引用 `const int& r = v;` 不能通过 `r` 修改 `v`。
- 被引用对象本身如果是普通变量，仍可通过原变量名修改。
- 常量引用可以绑定到字面值或临时结果。
- 函数参数中，普通引用不能绑定常量或字面值，常量引用可以。

> [!important] 核心认识
> `const int& r = v;` 的意思不是 `v` 永远不能变，而是不能通过 `r` 这个别名去改 `v`。

## 普通引用与常量引用

普通引用：

```cpp
int value = 100;
int& r1 = value;

r1 = 300; // 可以
```

常量引用：

```cpp
int value = 100;
const int& r2 = value;

r2 = 300; // 错误
```

`r2` 是 `value` 的别名，但这个别名带有 `const` 限制，不能通过它修改对象。

但可以通过原变量修改：

```cpp
value = 300; // 可以
cout << r2 << endl; // 300
```

## 常量引用可以绑定字面值

普通非常量引用不能绑定字面值：

```cpp
int& r = 1; // 错误
```

原因是普通引用要求能修改对象，而字面值 `1` 不是可修改变量。

常量引用可以绑定字面值：

```cpp
const int& r = 1; // 可以
```

编译器会为这个字面值创建一个临时对象，并让常量引用绑定到它。由于引用是 `const`，不能通过引用修改这个临时对象，因此是安全的。

## 函数参数中的匹配规则

假设函数：

```cpp
void f(int a1, int& a2, const int& a3);
```

三个参数分别是：

- `a1`：按值传递。
- `a2`：普通引用传递。
- `a3`：常量引用传递。

调用时：

```cpp
int v = 100;
const int cv = 200;
```

匹配规则：

| 实参 | `int a1` | `int& a2` | `const int& a3` |
|---|---|---|---|
| 字面值 `1` | 可以 | 不可以 | 可以 |
| 普通变量 `v` | 可以 | 可以 | 可以 |
| 常量 `cv` | 可以 | 不可以 | 可以 |

解释：

- 按值传递会复制值，所以字面值、变量、常量都可以。
- 普通引用可能修改实参，因此只能绑定可修改变量。
- 常量引用承诺不通过引用修改对象，所以可以绑定普通变量、常量和字面值。

## 常量引用的实际用途

真实 C++ 程序中，常量引用最常用于函数参数：

```cpp
void print(const string& text);
```

好处是：

- 避免复制大对象。
- 承诺函数不会通过该参数修改对象。
- 可以接收普通对象、常量对象和临时对象。

在当前课程阶段，只需先理解参数匹配规则；后续学类对象和拷贝时，常量引用的价值会更明显。

## 本节考点整理

| 可能题型 | 可能问法 | 答题要点 |
|---|---|---|
| 判断题 | `const int& r = v;` 表示 `v` 永远不能修改。 | 错，只是不能通过 `r` 修改 |
| 代码题 | `const int& r = 1;` 是否合法？ | 合法，绑定到临时对象 |
| 代码题 | `int& r = 1;` 是否合法？ | 不合法 |
| 选择题 | 普通引用参数能否接收字面值？ | 不能 |
| 选择题 | 常量引用参数能否接收字面值？ | 能 |
| 判断题 | 常量对象可以绑定到普通非常量引用。 | 错 |
| 简答题 | 常量引用作为函数参数有什么好处？ | 避免拷贝，且不修改实参 |

## 本节速记

> [!summary] 速记
> 普通引用能改对象，所以只能绑定可修改变量；常量引用不能通过引用改对象，所以可以绑定普通变量、常量和字面值。函数参数里，`const T&` 是“只读引用”，既安全又灵活。
