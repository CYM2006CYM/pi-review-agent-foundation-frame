# 5.1 函数声明

## 本节核心

本节讲[[函数声明]]，也就是告诉编译器一个函数“叫什么、怎么调用、参数是什么、返回什么”。

- 函数声明也常称为[[函数原型]]或[[函数签名]]。
- 函数声明主要包含函数名、返回类型、参数列表、调用约定、异常说明等信息。
- 函数形参名在声明中可以省略，关键是参数类型、个数和顺序。
- [[返回类型]]不能作为区分两个函数的标志。
- [[默认参数]]也不能作为区分函数的标志。
- 带默认参数的函数声明应集中放在头文件中，避免不同编译单元使用不同默认值。

> [!important] 高频考点
> 判断函数是否“同一个函数”，主要看函数名和参数列表，不看返回类型，也不看默认参数值。

## 函数声明的作用

函数声明告诉编译器：

- 函数名是什么。
- 返回类型是什么。
- 参数个数是多少。
- 参数类型和顺序是什么。
- 调用约定是什么。
- 是否有异常说明。

例如：

```cpp
int max(int a, int b);
```

这句声明告诉编译器：有一个函数叫 `max`，接收两个 `int` 参数，返回 `int`。

## 形参名可以省略

函数声明中，形参名不是必须的。

```cpp
int max(int a, int b);
int max(int, int);
```

两者都可以作为函数声明。形参名有助于阅读，但真正决定函数参数结构的是类型、个数和顺序。

## 函数声明的一般组成

一个函数声明可能包含：

```text
连接说明 调用约定 返回类型 函数名(参数列表) const 异常说明;
```

初学时最常见的形式是：

```cpp
返回类型 函数名(参数列表);
```

其中，`const` 修饰主要用于后续成员函数；异常说明用于说明函数是否可能抛出异常。

## 异常说明与 noexcept

如果函数声明不写异常说明，通常表示它可能抛出异常。

C++11 以后常用 [[noexcept]] 表示函数不抛出异常：

```cpp
void swap(int& a, int& b) noexcept;
```

旧式写法中可能出现：

```cpp
void f() throw();
```

现代 C++ 更常见 `noexcept`。

## 返回类型不能区分函数

下面两个函数不能仅靠返回类型区分：

```cpp
int func();
void func();
```

如果允许它们同时存在，调用时会产生歧义：

```cpp
func();
```

这句没有使用返回值，编译器无法仅靠返回类型判断应该调用哪个函数。因此 C++ 不允许仅用返回类型区分函数。

## 默认参数不能区分函数

默认参数是在调用处缺少实参时由编译器补上的值。

```cpp
void func(int n = 6);
func(); // 编译期可补成 func(6)
```

如果另一个文件里又看到：

```cpp
void func(int n = 8);
```

同样的 `func();` 可能在不同编译单元中被补成 `func(6)` 或 `func(8)`，导致源代码看起来一样，实际行为不一致。

因此默认参数应集中写在一个公共声明中，通常放在头文件里。

## 本节速记

> [!summary] 速记
> 函数声明告诉编译器“这个函数怎么用”。形参名可省，参数类型、个数和顺序最关键。返回类型不能区分函数，默认参数也不能区分函数。默认参数在编译期补入，应集中放在头文件中保持一致。
