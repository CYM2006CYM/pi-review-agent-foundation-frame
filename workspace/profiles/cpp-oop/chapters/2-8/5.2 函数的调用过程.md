# 5.2 函数的调用过程

## 本节核心

本节从底层角度理解[[函数调用过程]]：一次函数调用不是一句简单语句，而是包含压参数、调用、建立局部环境、返回和清理现场等步骤。

- 函数调用时，参数通常会按调用约定压入[[栈区]]。
- 标准 C 调用中，常见参数压栈顺序是从右到左。
- 被调用函数执行前要记录调用现场，执行后要恢复调用现场。
- [[清栈]]通常只是移动栈顶指针，不是把内存内容逐个清零。
- 谁负责清栈由[[调用约定]]决定。
- [[可变参数函数]]通常需要调用方负责清栈。

> [!important] 核心认识
> 函数调用的本质是“跳到另一段代码执行，并在执行完后回到原来的下一条位置”。因此必须保存返回位置和必要现场。

## 一个函数调用的大致过程

假设有函数：

```cpp
void func(int a, int b, int c) {
    int v1 = a;
    int v2 = b;
    int v3 = c;
}

int main() {
    func(1, 2, 3);
}
```

一次调用大致包括：

1. 调用方准备参数。
2. 按调用约定把参数放入寄存器或栈中。
3. 记录返回地址，跳转到被调用函数。
4. 被调用函数建立自己的局部环境。
5. 函数体执行。
6. 返回调用点。
7. 清理参数和调用现场。

## 参数压栈顺序

在课程展示的标准 C 调用场景中，`func(1, 2, 3)` 的参数常见压栈顺序是：

```text
先压 3
再压 2
再压 1
```

也就是从右到左。压栈顺序属于调用约定的一部分。

## 调用现场与返回地址

程序跳到被调用函数执行后，还要回到调用语句后面继续执行。因此调用时必须保存：

- 从哪里跳过去的。
- 执行完后应该回到哪里。
- 必要的寄存器和栈状态。

这些合起来可以理解为[[调用现场]]。

## 图示化理解：栈上的一次调用

以 `func(1, 2, 3)` 为例，在典型标准 C 调用模型中，可以把调用过程理解成：

```text
高地址
┌──────────────┐
│ 调用方原有栈帧 │
├──────────────┤
│ 参数 3        │  先压入
│ 参数 2        │
│ 参数 1        │  后压入，靠近栈顶
├──────────────┤
│ 返回地址      │  call 指令保存
├──────────────┤
│ 被调用函数局部变量 │ var1, var2, var3
└──────────────┘
低地址
```

调用结束时，并不是把 `1`、`2`、`3` 和局部变量逐个擦除，而是恢复栈顶位置。以后这片内存会被新的调用覆盖。

> [!tip] 复习抓手
> 看到函数调用过程题，先写四个词：压参、返回地址、局部变量、清栈。再根据调用约定说明谁清栈。

## 清栈

函数调用结束后，需要把参数占用的栈空间清理掉。

课程强调：清栈通常不是把内存内容逐个清零，而是移动[[栈顶指针]]。

```text
清栈 = 调整栈顶指针
不是 = 把每个字节都擦掉
```

因此清栈操作通常很快。

## 谁负责清栈

谁负责清栈由调用约定决定：

| 方式 | 清栈者 | 特点 |
|---|---|---|
| 调用方清栈 | 调用函数的一方 | 可支持可变参数 |
| 被调用方清栈 | 被调用函数 | 适合固定参数 |

例如 `printf` 这类[[可变参数函数]]，参数个数不固定，通常由调用方负责清栈。

## 函数名与底层地址

程序员写代码时使用函数名：

```cpp
func(1, 2, 3);
```

但真正的机器代码中，执行的是地址跳转。函数名主要是给程序员和编译链接过程使用的符号，最终会解析到代码地址。

## 本节考点整理

| 可能题型 | 可能问法 | 答题要点 |
|---|---|---|
| 简答题 | 函数调用大致包括哪些步骤？ | 准备参数、保存现场、跳转执行、建立局部环境、返回、清理 |
| 选择题 | 标准 C 调用中参数常见压栈顺序是什么？ | 从右到左 |

| 简答题 | 为什么可变参数函数通常由调用方清栈？ | 调用方知道实际传了几个参数 |
| 判断题 | 机器代码中直接按函数名执行。 | 不准确，底层是地址跳转 |

## 本节速记

> [!summary] 速记
> 函数调用会准备参数、保存返回地址、跳到函数体、建立局部变量、执行后恢复现场。标准 C 调用常从右到左压参数。清栈通常只是移动栈顶指针，谁清栈由调用约定决定；可变参数一般需要调用方清栈。
