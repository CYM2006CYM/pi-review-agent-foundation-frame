# 5.4 函数重载

## 本节核心

[[函数重载]]允许多个函数使用同一个名字，但它们的参数列表等必须能被编译器区分。

- C++ 允许同名函数共存，C 语言通常不允许。
- 重载主要靠参数个数、参数类型、参数顺序等区分。
- [[返回类型]]不能作为重载区分标志。
- [[默认参数]]不能作为重载区分标志。
- 按值参数顶层 `const` 不能作为重载区分标志。
- 指针/引用指向对象是否 `const`，在某些情况下可以影响重载匹配。
- 重载底层依靠[[名字重整]]实现。

> [!important] 高频考点
> `int f();` 和 `void f();` 不能构成重载；`void f(int);` 和 `void f(const int);` 也不能仅靠按值参数的 `const` 构成重载。

## 什么是函数重载

多个函数同名，但参数列表不同：

```cpp
void print(int);
void print(double);
void print(int, int);
```

调用时，编译器根据实参类型、个数和顺序选择最合适的函数。

```cpp
print(1);      // 调用 print(int)
print(1.5);    // 调用 print(double)
print(1, 2);   // 调用 print(int, int)
```

## 可以作为重载区分标志的内容

通常可以区分：

- 参数个数不同。
- 参数类型不同。
- 参数顺序不同。
- 引用或指针是否允许修改实参的差异。
- 成员函数中的 `const` 修饰差异。

示例：

```cpp
void f(int);
void f(double);
void f(int, int);
```

这些是合法重载。

## 不能作为重载区分标志

### 返回类型不能区分

```cpp
int f();
void f(); // 错误
```

原因是调用 `f();` 时可以丢弃返回值，编译器无法仅凭返回类型判断调用哪个。

### 默认参数不能区分

```cpp
void f(int);
void f(int = 5); // 错误或冲突
```

默认参数只是调用时缺少实参的补充规则，不改变函数本身的参数类型结构。

### 按值参数的顶层 const 不能区分

```cpp
void f(int);
void f(const int); // 不能构成重载
```

因为按值传参会创建形参副本，`const` 只限制函数内部是否修改这份副本，对调用者看不到差异。

## 指针和引用中的 const

下面这类可以区分：

```cpp
void f(int*);
void f(const int*);
```

区别是：第一个允许通过指针修改指向对象，第二个不允许通过指针修改指向对象。

引用也类似：

```cpp
void f(int&);
void f(const int&);
```

普通变量可能更匹配 `int&`，常量或字面值只能绑定到 `const int&`。

## 二义性调用

有些函数声明本身可以共存，但某个调用可能让编译器无法决定选哪一个。

例如：

```cpp
void f(int*);
void f(int*, int = 6);
```

调用：

```cpp
f(p);
```

两个函数都能匹配，编译器可能报[[二义性]]错误。

这不是函数不能重载，而是调用点无法唯一选择。

## 名字重整

函数重载的底层实现依靠[[名字重整]]。

程序员写：

```cpp
void f(int);
void f(double);
```

编译器会把函数名和参数类型等信息组合成不同的内部符号名，让链接器能区分它们。

不同编译器的名字重整规则可能不同，因此跨编译器、跨语言调用时要特别注意。

## extern "C"

如果希望 C++ 函数按 C 风格导出，避免 C++ 名字重整，可以使用：

```cpp
extern "C" {
    void f(int);
}
```

这常用于 C 和 C++ 互相调用、动态链接库导出接口等场景。

> [!warning] 易错点
> `extern "C"` 会让函数按 C 链接方式处理，因此同一作用域下不能再依靠 C++ 函数重载导出多个同名函数。

## 本节速记

> [!summary] 速记
> 重载看参数，不看返回值和默认参数。按值参数的顶层 `const` 不区分；指针/引用影响能否修改实参时可以区分。重载底层靠名字重整；跨语言或导出动态库时常用 `extern "C"` 避免 C++ 风格重整。
