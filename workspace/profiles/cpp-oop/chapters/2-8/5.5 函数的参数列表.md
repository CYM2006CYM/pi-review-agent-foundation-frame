# 5.5 函数的参数列表

## 本节核心

本节集中讲函数参数列表的声明、定义、默认参数、实参与形参匹配，以及常见参数传递方式。

- 声明中的参数名可以写，也可以省略。
- 定义中的参数可以无名，但无名参数在函数体内无法访问。
- 默认参数应写在函数原型中，通常放头文件；没有原型时才写在定义处。
- 参数压栈顺序由[[调用约定]]决定，但参数[[求值顺序]]不一定由压栈顺序决定。
- 不要在同一次函数调用的多个实参中依赖副作用顺序。
- 参数传递方式主要有[[按值传递]]、[[指针传递]]、[[引用传递]]、数组参数退化为指针等。

> [!important] 高频考点
> 参数“压栈顺序”和参数“求值顺序”不是一回事。`f(++a, ++a, ++a)` 这类写法不要依赖具体结果。

## 声明与定义中的参数列表

函数声明：

```cpp
void f(int, int&);
```

函数定义：

```cpp
void f(int a, int& b) {
    // 可以使用 a 和 b
}
```

声明中的参数名可以省略；定义中的参数名如果省略，函数体内就无法访问该参数。

```cpp
void f(int, int b) {
    // 第一个参数存在并参与传递，但函数体里没有名字可用
}
```

## 默认参数写在哪里

默认参数应优先写在函数原型中：

```cpp
// header.h
void f(int a, int b = 10);
```

如果没有单独函数原型，才在函数定义处给出默认参数：

```cpp
void f(int a, int b = 10) {
}
```

工程中更推荐把带默认参数的声明放在头文件中，所有 `.cpp` 都包含同一份声明，避免默认值不一致。

## 默认参数的右侧规则

从第一个带默认值的参数开始，右侧所有参数都必须有默认值。

合法：

```cpp
void f(int a, int b = 1, int c = 2);
```

不合法：

```cpp
void f(int a, int b = 1, int c);
```

原因是 C++ 不支持命名参数，调用实参按顺序匹配。中间缺一个、后面再给一个这种写法无法表达。

## 参数求值顺序

调用约定可以规定参数如何压栈，但并不等于规定每个实参表达式先算哪个。

例如：

```cpp
int a = 10;
f(++a, ++a, ++a);
```

不同编译器或不同标准语境下，实参表达式的求值顺序可能不同，结果不应依赖它。

更好的写法是拆开：

```cpp
int a = 10;
int x = ++a;
int y = ++a;
int z = ++a;
f(x, y, z);
```

> [!warning] 易错点
> 不要在同一个函数调用的多个实参中对同一个变量做多次自增、自减或修改。

## 参数类型转换

实参和形参类型不完全一致时，编译器可能尝试类型转换。

```cpp
void f(int);
void f(double);

f(2.5f);
```

`2.5f` 是 `float`，可能被转换为 `double` 来匹配。

课程强调：参数匹配中的类型转换不能无限绕路，多次转换会导致复杂和歧义，编译器只会按规则选择有限的转换路径。

## 按值传递

[[按值传递]]会把实参复制成形参。

```cpp
void f(int x) {
    x = 10;
}
```

函数内部修改 `x` 不会修改外部实参。

对象也可以按值传递，但会产生对象复制，复杂对象可能成本较高。

```cpp
void f(MyObject obj);
```

对于复杂对象，除非确实需要副本，否则通常更倾向于引用或常量引用传递。

## 指针传递

[[指针传递]]传的是地址。

```cpp
void f(int* p) {
    *p = 10;
}
```

如果参数是 `const int*`，表示不能通过该指针修改指向对象：

```cpp
void f(const int* p) {
    // *p = 10; // 错误
}
```

指针参数中的 `const` 会影响函数重载和参数匹配。

## 数组参数

数组作为函数参数时，底层通常退化为指针。

下面几种写法在函数参数中常可等价理解：

```cpp
void f(int a[]);
void f(int a[3]);
void f(int* a);
```

它们本质上都接收指向数组首元素的指针。形参中的数组大小往往不能真正限制传入数组长度。

> [!warning] 易错点
> `void f(int a[3])` 不表示函数一定只接收长度为 3 的数组；在参数位置它通常仍按指针处理。

## 引用传递

[[引用传递]]让形参成为实参的别名：

```cpp
void f(int& x) {
    x = 10;
}
```

普通引用参数只能绑定可修改变量；[[const引用]]可以绑定普通变量、常量和字面值。

```cpp
void g(const int& x);
g(1); // 可以
```

## 本节考点整理

| 可能题型 | 可能问法 | 答题要点 |
|---|---|---|
| 判断题 | 声明中的形参名必须和定义中的形参名相同。 | 错 |
| 判断题 | 定义中的无名参数在函数体内可以直接访问。 | 错 |
| 简答题 | 默认参数应该写在哪里？ | 优先写在函数原型/头文件中 |
| 判断题 | 第一个默认参数右侧可以有无默认值参数。 | 错 |
| 判断题 | 参数压栈顺序等于参数求值顺序。 | 错 |
| 代码题 | `f(++a, ++a, ++a)` 是否推荐？ | 不推荐，求值顺序不应依赖 |
| 名词解释 | 什么是按值传递？ | 复制实参为形参 |
| 判断题 | 数组形参 `int a[3]` 一定限制传入长度为 3。 | 错，参数中通常退化为指针 |
| 选择题 | 常量或字面值能绑定到哪类引用参数？ | `const T&` |

## 本节速记

> [!summary] 速记
> 默认参数写头文件声明中，且从第一个默认参数开始右侧都要有默认值。参数压栈顺序由调用约定定，但实参求值顺序不要依赖。按值传递会复制，指针传递传地址，数组参数通常退化为指针，引用传递是别名，常量引用最灵活。
