# 5.6 函数的返回

## 本节核心

函数返回值不只是“返回一个结果”，还关系到结果是否可修改、结果是否仍然有效、调用者能否把函数调用表达式放在赋值号左边。

本节主要讨论三类返回方式：

- [[按值返回]]：返回一个独立的值。
- [[返回指针]]：返回一个地址值。
- [[返回引用]]：返回某个已经存在对象的别名。

其中最容易出错、也最常考的是 [[返回引用]]。

## 按值返回

[[按值返回]] 是最常见的返回方式：

```cpp
int f();
double g();
Point makePoint();
```

按值返回的含义是：函数调用表达式得到一个结果值，这个结果通常不是原对象本身，而是一个独立的返回结果。

```cpp
int f() {
    return 10;
}

int x = f();
```

对内置类型来说，返回结果本身一般不能作为可修改对象使用：

```cpp
f() = 20; // 错误：按值返回的 int 结果不是可修改左值
```

因此，对于内置类型的按值返回：

```cpp
int f();
const int f();
```

在很多使用场景下差别不大，因为这个返回出来的纯数值本身不能直接被修改。

> [!note] 注意
> 对类类型按值返回时，`const` 是否修饰返回值会影响后续调用和对象语义，本课程后面讲类和对象时还会继续展开。现在先记住：内置类型的按值返回不必刻意写成 `const int`。

## `void` 返回

如果函数不需要返回结果，应写成 [[void]]：

```cpp
void printMessage() {
    // 只完成动作，不返回结果
}
```

有些语言会把这种“只做事、不返回值”的函数称为“过程”。在 C++ 中通常仍称为函数，只是返回类型为 `void`。

## 返回指针

[[返回指针]] 返回的是一个地址值：

```cpp
int* find();
```

指针本身也是一个值。因此：

```cpp
int* f();
int* const f();
```

这里的 `const` 如果修饰的是“返回出来的指针本身”，通常意义有限，因为返回出来的地址值本身就是临时结果。

真正有区别的是：

```cpp
const int* f();
```

它表示：函数返回一个指针，通过这个指针不能修改所指向的 `int` 对象。

```cpp
const int* p = f();
*p = 10; // 错误：p 指向 const int
```

所以要区分：

- `int* f()`：返回指向可修改 `int` 的指针。
- `const int* f()`：返回指向不可通过该指针修改的 `int` 的指针。
- `int* const f()`：试图把返回出来的指针值本身设为 const，通常没有实际必要。

## 返回引用

[[返回引用]] 的语义是：函数调用表达式成为某个已有对象的别名。

```cpp
int& ref();
const int& cref();
```

这和 [[按值返回]] 的本质区别是：

- 按值返回：得到一个结果值。
- 返回引用：得到某个对象的别名。

因此，返回引用必须满足一个前提：返回时被引用的对象仍然真实存在。

## 不能返回局部变量的引用

局部变量通常存放在函数调用的 [[栈区]] 中。函数执行结束后，局部变量生命周期结束，栈空间被清理。

```cpp
int& bad() {
    int x = 88;
    return x; // 错误：x 在函数结束后失效
}
```

这里返回的是 `x` 的别名，但 `x` 离开函数后已经不存在，于是产生悬空引用。

> [!danger] 考试高频错误
> 函数不能返回普通局部变量的引用或指针。因为函数结束后，普通局部变量已经销毁，返回出去的引用或指针会指向无效对象。

## 可以返回静态局部变量和全局变量的引用

[[静态局部变量]] 的生命周期不是函数调用期间，而是整个程序运行期间：

```cpp
int& ok1() {
    static int x = 0;
    return x; // 可以：x 函数结束后仍然存在
}
```

[[全局变量]] 同理：

```cpp
int global = 0;

int& ok2() {
    return global; // 可以：global 仍然存在
}
```

但要注意，返回引用会把内部对象暴露给调用者修改：

```cpp
ok1() = 5; // 合法：ok1() 是 x 的引用
```

这正是返回非常量引用的威力，也是风险。

## 返回引用参数

如果形参本身是引用，那么它在调用前就已经绑定到某个真实对象：

```cpp
int& choose(int& a, int& b) {
    return a > b ? a : b;
}
```

调用时：

```cpp
int x = 3;
int y = 4;

choose(x, y) = 5; // 修改的是 y
```

因为 `choose(x, y)` 返回的是 `x` 或 `y` 的引用，所以它可以出现在赋值号左侧。

如果形参是按值传递，就不能返回它的引用：

```cpp
int& bad(int a) {
    return a; // 错误：a 是函数内部的形参副本，函数结束后失效
}
```

## 普通引用返回与 const 引用返回

非常量引用返回：

```cpp
int& maxRef(int& a, int& b) {
    return a > b ? a : b;
}
```

调用者可以修改返回对象：

```cpp
int a = 3;
int b = 4;

maxRef(a, b) = 5; // b 被修改为 5
```

[[const引用]] 返回：

```cpp
const int& maxConstRef(const int& a, const int& b) {
    return a > b ? a : b;
}
```

调用者只能读取，不能修改：

```cpp
maxConstRef(a, b) = 6; // 错误：返回的是 const int&
```

> [!tip] 设计判断
> 如果希望调用者能通过函数结果修改原对象，返回 `T&`；如果只希望避免复制并允许读取，返回 `const T&`。

## 不能把右值绑定到非常量引用参数

非常量引用参数要求传入可修改对象：

```cpp
int& maxRef(int& a, int& b);

int x = 3;
maxRef(x, 10); // 错误：10 是右值，不能绑定到 int&
```

如果要允许字面量、临时对象等右值参与，应使用 `const T&` 或按值传递：

```cpp
const int& maxConstRef(const int& a, const int& b);

maxConstRef(x, 10); // 可以
```

## 从指针参数返回引用要谨慎

如果函数接收指针参数，再返回 `*p` 的引用：

```cpp
int& risky(int* p) {
    return *p;
}
```

这在 `p` 指向有效对象时看起来没问题，但如果 `p == nullptr`，解引用就会出错。

引用参数比指针参数更强：引用参数天然要求有对象绑定；指针参数可能为空。

```cpp
int& safer(int& x) {
    return x;
}
```

## 本节考点整理

- [[返回类型]] 不是函数重载的区分依据。
- [[按值返回]] 得到的是结果值，不是原对象别名。
- [[返回引用]] 得到的是已有对象的别名。
- 返回引用时，被引用对象必须在函数返回后仍然存在。
- 不能返回普通局部变量、按值形参、临时表达式结果的引用。
- 可以返回全局变量、静态局部变量、引用形参的引用。
- 返回 `T&` 的函数调用表达式可以作为左值。
- 返回 `const T&` 可以读取但不能修改。
- 非常量引用参数不能绑定右值。

## 本节速记

> 按值返回看结果，返回引用看对象；  
> 局部变量不能回，静态全局可以回；  
> 返回 `T&` 可修改，返回 `const T&` 只可读。
