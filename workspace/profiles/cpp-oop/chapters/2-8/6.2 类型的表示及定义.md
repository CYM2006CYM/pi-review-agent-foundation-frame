# 6.2 类型的表示及定义

## 本节核心

上一节讨论的是 [[类型的抽象]]，这一节讨论如何在 C++ 中表示这种抽象。

在 C++ 中，通常用 [[class]] 定义一个 [[类]]，用类来表示抽象出来的类型。

一个类通常包括：

- [[类名]]：对应抽象数据类型的名字。
- [[成员函数]]：表示对象的行为。
- [[数据成员]]：表示对象内部保存的数据。

## 从抽象数据类型到 class

抽象数据类型的结构可以映射到 C++ 类：

| 抽象层面的内容 | C++ 表示 |
|---|---|
| 类型名字 | 类名 |
| 行为或操作 | 成员函数 |
| 数据及数据关系 | 数据成员 |

示意：

```cpp
class Book {
public:
    const char* title() const;
    void turnTo(int page);
    void addBookmark(int page);

private:
    const char* title_;
    int totalPages_;
    int currentPage_;
};
```

这里的 `Book` 是类名，`title`、`turnTo`、`addBookmark` 是行为，`title_`、`totalPages_`、`currentPage_` 是数据。

## 行为与操作

在抽象阶段，最应该关注的是 [[行为]]，不是具体有几个函数。

一个行为可能由一个操作完成，也可能由多个操作共同完成。

例如“阅读一本书”这个行为，在程序里可能拆成：

- 打开书
- 翻到某页
- 读取若干页
- 更新当前页
- 记录阅读进度

但在分析设计时，先关注“阅读”这个行为，再考虑它在代码中如何拆分为成员函数。

> [!important] 设计顺序
> 先问“对象有什么行为”，再问“这个行为用几个成员函数实现”。

## 成员函数与数据成员

类内部大体有两类成员：

- [[成员函数]]：描述对象能做什么。
- [[数据成员]]：描述对象保存什么。

```cpp
class Cup {
public:
    void addWater(int ml);
    void drink(int ml);

private:
    int capacity_;
    int water_;
};
```

`addWater`、`drink` 是水杯的行为；`capacity_`、`water_` 是水杯内部状态。

## 类成员的书写顺序

C++ 语法并不强制要求先写函数再写数据，也不强制要求先写 `public` 再写 `private`。

但 C++ 中常见习惯是：

1. 把对外可见的 `public` 行为放在前面。
2. 把内部使用的 `private` 数据放在后面。

原因是读代码的人通常更关心一个类对外能做什么，而不是它内部怎么存。

```cpp
class Dog {
public:
    void bark();
    void eat();

private:
    int age_;
    int weight_;
};
```

这和 Java 等语言的常见风格可能不同。风格没有绝对标准，但要理解 C++ 代码里这种写法的意图：突出对象对外行为。

## 类定义通常放在头文件中

[[类定义]] 通常放在 [[头文件]] 中，因为其他 `.cpp` 文件要使用这个类，就需要知道类的声明和定义结构。

```cpp
// Dog.h
#ifndef DOG_H
#define DOG_H

class Dog {
public:
    void bark();
};

#endif
```

类定义放在头文件时，必须注意 [[包含警戒]]，否则同一个翻译单元中多次包含同一个头文件，可能导致类重复定义，违反 [[单一定义规则]]。

## 为什么类定义需要包含警戒

如果没有包含警戒：

```cpp
// Dog.h
class Dog {
public:
    void bark();
};
```

某个 `.cpp` 可能通过不同路径把 `Dog.h` 展开两次：

```cpp
#include "Dog.h"
#include "Cat.h" // Cat.h 里也 include "Dog.h"
```

预处理后，`class Dog { ... };` 可能在同一个 `.cpp` 中出现两次，于是产生重复定义错误。

包含警戒的作用就是保证头文件内容在同一个翻译单元中只展开一次。

也可以使用：

```cpp
#pragma once
```

但无论采用哪种形式，都要有防止重复包含的机制。

## 前置声明

[[前置声明]] 用来提前告诉编译器：某个名字是一个类。

```cpp
class Cat;

class Dog {
public:
    void fight(Cat& cat);
};
```

这里 `class Cat;` 并没有给出 `Cat` 的完整定义，只告诉编译器 `Cat` 是一个类名。这样编译器在看到 `Cat&` 时就能识别它。

## 前置声明解决循环定义

如果 `Dog.h` 包含 `Cat.h`，同时 `Cat.h` 又包含 `Dog.h`，就会形成循环包含。

错误倾向：

```cpp
// Dog.h
#include "Cat.h"

class Dog {
public:
    void fight(Cat& cat);
};
```

```cpp
// Cat.h
#include "Dog.h"

class Cat {
public:
    void fight(Dog& dog);
};
```

更好的方式：

```cpp
// Dog.h
class Cat;

class Dog {
public:
    void fight(Cat& cat);
};
```

```cpp
// Cat.h
class Dog;

class Cat {
public:
    void fight(Dog& dog);
};
```

这样两个头文件可以独立存在，避免相互 `#include` 导致循环定义。

## 前置声明降低编译期依赖

前置声明还有一个重要作用：降低文件之间的 [[编译期依赖]]。

如果 `Dog.h` 直接包含 `Bone.h`：

```cpp
#include "Bone.h"

class Dog {
public:
    void eat(Bone& bone);
};
```

那么任何包含 `Dog.h` 的 `.cpp` 文件，都会间接依赖 `Bone.h`。只要 `Bone.h` 改动，相关 `.cpp` 就可能需要重新编译。

如果改成前置声明：

```cpp
class Bone;

class Dog {
public:
    void eat(Bone& bone);
};
```

那么 `Dog.h` 不再依赖 `Bone.h` 的完整定义。`Bone.h` 的普通改动不会无谓触发大量重新编译。

> [!tip] 工程习惯
> 头文件中能用前置声明解决的地方，就尽量不要直接 `#include`。需要完整定义时，再在 `.cpp` 中包含对应头文件。

## 前置声明的使用边界

前置声明只能在“不需要知道完整类定义”的场景使用，例如：

- 声明指针：`Cat*`
- 声明引用：`Cat&`
- 声明函数参数或返回类型

如果要按值保存对象，就需要完整定义，因为编译器必须知道对象大小：

```cpp
class Cat;

class Dog {
private:
    Cat cat_; // 通常不行：编译器不知道 Cat 的大小
};
```

这时就需要包含 `Cat.h`。

## 本节考点整理

- C++ 用 [[class]] 表示抽象出来的类型。
- 类中通常包含 [[成员函数]] 和 [[数据成员]]。
- 成员函数对应对象行为，数据成员对应对象状态。
- C++ 常见习惯是 `public` 行为在前，`private` 数据在后。
- [[类定义]] 通常放在 [[头文件]] 中。
- 类定义放头文件必须配合 [[包含警戒]]，避免违反 [[单一定义规则]]。
- [[前置声明]] 可以解决循环定义问题。
- 前置声明可以降低 [[编译期依赖]]，提升大型工程编译效率。
- 只用指针或引用时通常可用前置声明；按值包含对象通常需要完整定义。

## 本节速记

> 抽象到 C++，行为变成员函数，数据变数据成员；  
> 类定义放头文件，头文件要防重复；  
> 能前置声明就少 include，减少循环和依赖。
