# 6.3 类的成员

## 本节核心

[[类]] 的成员大体分为两类：

- [[数据成员]]：表示对象内部保存的数据。
- [[成员函数]]：表示对象对外表现出的行为。

面向对象设计中，数据成员比较容易理解，但真正重要、真正体现抽象能力的是成员函数所表示的 [[行为]]。

## 数据成员可以有哪些类型

类中的数据成员可以使用很多类型：

- 内置类型：`int`、`char`、`double`
- [[枚举]] 类型
- [[指针]]
- [[引用]]
- [[数组]]
- 其他类类型对象
- `const` 成员
- [[静态数据成员]]

例如：

```cpp
class Dog {
private:
    int age_;
    char color_;
    int legs_[4];
};
```

这些都是普通的数据成员。

## 引用成员表示必须绑定对象

如果一个类中有引用成员，表示这个成员必须绑定到一个真实对象。

```cpp
class Person {
};

class Dog {
private:
    Person& owner_;
};
```

`owner_` 是引用成员，意味着每个 `Dog` 对象都必须有一个主人对象与之对应。引用不能为 `nullptr`，所以语义上比指针更强。

> [!tip] 语义区别
> 用指针表示“可能有，也可能没有”；用引用表示“必须有”。

## 类不能按值包含自身类型成员

类中不能直接定义一个同类型的对象成员：

```cpp
class Dog {
private:
    Dog part_; // 错误
};
```

原因是对象大小会无限递归：

- 一个 `Dog` 中包含一个 `Dog part_`
- `part_` 中又包含一个 `Dog part_`
- 如此无限下去，编译器无法确定 `Dog` 对象大小

这种情况属于递归式的对象包含，不能成立。

## 可以包含自身类型的指针或引用

虽然不能按值包含自身对象，但可以包含自身类型的指针或引用：

```cpp
class Dog {
private:
    Dog* next_;
    Dog& friendDog_;
};
```

指针成员本身只占一个指针大小，不需要把另一个完整 `Dog` 对象嵌入进来，所以不会导致无限递归。

引用成员在底层实现上通常也按类似指针处理，因此也不会导致对象大小无限递归。

> [!important] 考点
> `Dog dog_;` 不行，`Dog* dog_;` 可以，`Dog& dog_;` 可以。

## 不同类之间也要避免递归对象包含

如果两个类互相按值包含，也会出问题：

```cpp
class Cat {
private:
    Dog dog_;
};

class Dog {
private:
    Cat cat_;
};
```

这同样会导致无限递归：`Dog` 里有 `Cat`，`Cat` 里又有 `Dog`。

解决思路通常是改用指针或引用，并结合 [[前置声明]]：

```cpp
class Cat;

class Dog {
private:
    Cat* cat_;
};
```

## const 成员与静态成员的初始化

现代 C++ 允许在类内对某些数据成员给出默认初始值：

```cpp
class Dog {
private:
    int age_ = 3;
    double weight_ = 12.5;
};
```

对于 `static` 成员，早期 C++ 中只有“静态常量整型成员”比较适合在类内初始化：

```cpp
class Dog {
private:
    static const int legs_ = 4;
};
```

普通静态数据成员通常需要在类外定义：

```cpp
class Dog {
private:
    static int count_;
};

int Dog::count_ = 0;
```

> [!note] 复习理解
> 静态数据成员属于类本身，不属于某一个具体对象；所以它的定义、存储位置和普通数据成员不同。

## 成员函数表示行为

[[成员函数]] 用来表示抽象数据类型的行为。

例如双向整型链表可以有这些行为：

- 从后面添加整数
- 从前面添加整数
- 取得某个位置的整数
- 设置某个位置的整数

可以表示为：

```cpp
class IntList {
public:
    void pushBack(int value);
    void pushFront(int value);
    int get(int index) const;
    void set(int index, int value);
};
```

这里先关注“链表能做什么”，暂时不关心底层到底用数组、链表节点还是其他结构实现。

## 成员函数隐含当前对象

成员函数看起来只写了显式参数：

```cpp
void pushBack(int value);
```

但它实际是对“当前对象”执行操作。

```cpp
list.pushBack(10);
```

这句话的意思不是“往某个不明链表添加 10”，而是“往 `list` 这个当前对象中添加 10”。

在 C++ 内部，这个“当前对象”通过 [[this指针]] 隐含传入。

## 行为抽象不要过早陷入实现

以赛车游戏中的汽车为例，它的行为可以抽象为：

- 启动
- 加速
- 减速
- 换挡
- 刹车
- 碰撞

可以写成：

```cpp
class Car {
public:
    void start();
    void speedUp();
    void slowDown();
    void shift(int gear);
    void brake();
    void hit(Car& other);
};
```

设计这些成员函数时，先不要关心“现实汽车如何点火”“游戏里如何计算速度”。这些属于实现细节。抽象阶段只需要把问题中需要的行为表达出来。

## 瓶子例子：行为是设计核心

分油问题中有 10 斤、7 斤、3 斤三个瓶子。对“瓶子”这个类来说，真正需要的行为可能只有：

- 从当前瓶子向另一个瓶子倒
- 取得当前瓶子中已有多少油

```cpp
class Bottle {
public:
    Bottle(int capacity, int current);
    void pourTo(Bottle& other);
    int current() const;

private:
    int capacity_;
    int current_;
};
```

这里 `capacity_`、`current_` 是数据；`pourTo`、`current` 是行为。

面向对象设计更关注行为部分，因为用户真正使用对象时，是通过行为和对象交互，而不是直接关心对象内部怎么存。

## 本节考点整理

- 类成员分为 [[数据成员]] 和 [[成员函数]]。
- 数据成员可以是内置类型、枚举、数组、指针、引用、其他类对象等。
- 类不能按值包含自身类型成员。
- 类可以包含自身类型指针或引用成员。
- 两个类也不能互相按值递归包含。
- [[成员函数]] 表示对象行为，行为是面向对象分析设计的重点。
- 成员函数有隐含的当前对象参数，即 [[this指针]]。
- 设计类时先抽象行为，再考虑数据和实现。

## 本节速记

> 数据成员存状态，成员函数表行为；  
> 自身对象不能嵌，自身指针引用可；  
> 类设计先看行为，不要先陷进实现。
