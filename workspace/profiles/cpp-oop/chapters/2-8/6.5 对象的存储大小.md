# 6.5 对象的存储大小

## 本节核心

对象实例化后会占据存储空间。对象大小主要由 [[非静态数据成员]] 决定，而不是由成员函数数量决定。

对象大小通常与这些因素有关：

- 非静态数据成员的个数和类型
- 是否有 [[虚函数]]
- [[字节对齐]] 方式

通常与这些因素无关：

- [[成员函数]] 个数
- [[静态数据成员]] 个数和类型
- `public` / `private` 等 [[访问控制]]

## 成员函数不计入每个对象大小

成员函数属于类的行为描述，不会在每个对象中各存一份函数代码。

```cpp
class A {
public:
    void f();
    void g();

private:
    int x_;
};
```

`A` 对象的大小主要看 `x_`，不会因为有 `f` 和 `g` 两个成员函数而给每个对象额外复制两份函数。

> [!note] 理解
> 对象保存状态，函数代码属于类的公共行为实现；每个对象不需要重复保存同样的函数代码。

## 非静态数据成员决定对象基本大小

例如：

```cpp
class A {
private:
    int v1_;
    int v2_;
    char c_;
    int arr_[2];
    Person person_;
    Car* car_;
    Person& owner_;
};
```

对象大小要考虑：

- `int v1_`
- `int v2_`
- `char c_`
- `int arr_[2]`
- `Person person_`
- `Car* car_`
- `Person& owner_`

其中：

- 对象型成员 `Person person_` 会嵌入一个完整 `Person` 对象。
- 指针成员大小就是指针大小。
- 引用成员在实现上通常类似指针，也会占一定空间。

## 静态数据成员不属于单个对象

[[静态数据成员]] 属于类，不属于某一个对象。

```cpp
class A {
private:
    int x_;
    static int count_;
};
```

`count_` 不会被每个 `A` 对象各存一份。因此它不计入普通对象本身的大小。

```cpp
A a1;
A a2;
```

`a1` 和 `a2` 各自有自己的 `x_`，但共享同一个 `A::count_`。

## 访问控制不影响对象大小

`public`、`private`、`protected` 只是控制访问权限，不改变数据成员是否属于对象。

```cpp
class A {
public:
    int x_;

private:
    int y_;
};
```

`x_` 和 `y_` 都是非静态数据成员，都会影响对象大小。`private` 不会让成员“不占空间”。

## 空类对象大小不为 0

即使类中没有任何数据成员，对象大小也不能是 0：

```cpp
class Empty {
};

Empty e;
```

通常 `sizeof(Empty)` 至少为 1。

原因是：C++ 需要通过对象地址区分不同对象。

```cpp
Empty a[2];
```

如果每个 `Empty` 对象大小为 0，则 `a[0]` 和 `a[1]` 可能拥有相同地址，这会破坏“不同对象应有不同地址”的基本要求。

> [!important] 考点
> 空类对象也占空间，常见结果是 1 字节，但具体大小受编译器和实现影响。

## 虚函数会影响对象大小

如果类中有 [[虚函数]]，对象中通常会额外保存一个 [[虚函数表指针]]，即 `vptr`。

```cpp
class A {
public:
    virtual void f();
};
```

当类含有虚函数时，对象大小通常会增加一个指针大小，用来支持运行时多态。

本节只先记住这个现象，虚函数机制后续会详细讲。

## 字节对齐会影响对象大小

对象大小不仅是成员大小简单相加，还可能受 [[字节对齐]] 影响。

例如成员顺序中出现：

```cpp
char c_;
int arr_[2];
```

`char` 只占 1 字节，但后面的 `int` 数组可能要求按 4 字节边界存放。编译器可能在 `char` 后面填充若干无用字节，让后面的成员对齐。

这种填充称为 padding。

字节对齐的目的通常是提高读取速度，但会增加对象大小。

## 对象大小的判断顺序

复习时可以按下面顺序判断：

1. 找出所有 [[非静态数据成员]]。
2. 排除成员函数和静态数据成员。
3. 计算成员类型本身需要的空间。
4. 考虑指针、引用、对象型成员。
5. 考虑是否有虚函数导致 `vptr`。
6. 考虑字节对齐带来的填充。
7. 如果是空类，大小也至少非零。

## 图示化判断：哪些成员进入对象

以下类可以用来复习对象大小：

```cpp
class A {
public:
    void f();
    int v1;

private:
    int g();
    int v2;
    char c;
    int arr[2];
    Person person;
    Car* pCar;
    Person& obj;
    static int num;
};
```

判断时可以做标记：

| 成员 | 是否影响每个 `A` 对象大小 | 原因 |
|---|---:|---|
| `void f()` | 否 | 成员函数代码不复制进每个对象 |
| `int g()` | 否 | `private` 成员函数也不进对象 |
| `int v1`, `int v2` | 是 | 非静态数据成员 |
| `char c` | 是 | 非静态数据成员，还可能引发对齐填充 |
| `int arr[2]` | 是 | 数组作为成员整体嵌入对象 |
| `Person person` | 是 | 成员对象完整嵌入 |
| `Car* pCar` | 是 | 指针本身是数据成员 |
| `Person& obj` | 通常是 | 引用成员需要实现上的存储支持 |
| `static int num` | 否 | 静态数据成员属于类，不属于单个对象 |

> [!important] 结论
> 对象大小不是“把类里所有东西相加”，而是看每个对象必须保存哪些状态。函数、访问控制、静态数据成员都不是每个对象的独立状态。

## 本节考点整理

- 对象大小主要由 [[非静态数据成员]] 决定。
- [[成员函数]] 不计入每个对象大小。
- [[静态数据成员]] 不属于单个对象，不计入普通对象大小。
- [[访问控制]] 不影响对象大小。
- 指针成员按指针大小计算。
- 引用成员底层通常类似指针，也会影响对象大小。
- 对象型成员会嵌入完整成员对象。
- 空类对象大小不为 0。
- 有 [[虚函数]] 时通常会增加 [[虚函数表指针]]。
- [[字节对齐]] 可能导致填充字节，使对象大小大于成员大小之和。

## 本节速记

> 对象大小看数据，不看函数；  
> 静态成员不进对象；  
> 空类也非零，虚函数加指针，对齐会填充。
