# 7.2 this 指针

## 本节核心

[[this指针]] 是普通成员函数中的隐含参数，永远指向当前对象。

它回答了成员函数实现中的核心问题：

> 同一个成员函数被多个对象调用时，函数体里访问的是哪个对象的数据？

答案是：`this` 指向哪个对象，就访问哪个对象的数据。

## this 是关键字

`this` 是 C++ 关键字，也是保留字，不能拿来当普通变量名。

```cpp
// 错误：不能定义名为 this 的变量
int this = 0;
```

在非静态成员函数中，编译器会自动提供 `this`。

## this 只存在于非静态成员函数中

`this` 指向当前对象，因此只有存在“当前对象”的成员函数才有 `this`。

普通成员函数有 `this`：

```cpp
class Student {
public:
    void study(int hours);
};
```

静态成员函数没有 `this`：

```cpp
class Student {
public:
    static int count();
};
```

因为 [[静态数据成员]] 和静态成员函数属于类，不属于某一个对象，所以静态成员函数中没有“当前对象”。

## this 的类型

在普通成员函数中，`this` 可以理解为一个指向当前类对象的常量指针：

```cpp
T* const this
```

其中 `T` 是当前类类型。

例如在 `Student` 的成员函数中：

```cpp
Student* const this
```

这里的 `const` 修饰的是指针本身，表示 `this` 不能改指向：

```cpp
this = other; // 错误：this 不能改指向
```

但这不代表当前对象一定不可修改。普通成员函数中仍可以通过 `this` 修改当前对象的数据成员：

```cpp
this->score_ += 1;
```

## 成员函数调用的底层理解

调用：

```cpp
s1.study(4);
```

可以理解为：

```cpp
Student::study(&s1, 4);
```

这里 `&s1` 是隐含传入的当前对象地址，对应成员函数内部的 `this`。

因此在函数体中：

```cpp
score_ += 4;
```

实际作用于：

```cpp
this->score_ += 4;
```

如果调用的是：

```cpp
s2.study(4);
```

那么 `this` 指向 `s2`，修改的就是 `s2` 的 `score_`。

## 图示化理解：普通成员函数多了一个隐含参数

成员函数表面上只写显式参数：

```cpp
s1.study(4);
s2.study(2);
```

可以理解为编译器在内部补上当前对象地址：

```cpp
Student::study(&s1, 4);
Student::study(&s2, 2);
```

于是函数体中的成员访问：

```cpp
score += hours;
```

等价于：

```cpp
this->score += hours;
```

这解释了两个问题：

- 同一份 `Student::study` 代码可以服务多个对象；
- 函数体里访问哪个对象的数据，由调用时传入的 `this` 决定。

> [!warning] 易错点
> `this` 是 `T* const`，表示指针本身不能改指向；不是说当前对象一定是常对象。只有常成员函数中，`this` 才进一步表现为指向常对象的指针。

## this 的作用域和生存期

`this` 只在当前非静态成员函数的函数体内有效。

```cpp
void Student::study(int hours) {
    this->score_ += hours; // 有效
}
```

离开这个函数体后，这次调用对应的 `this` 就不再存在。下次调用成员函数时，编译器会根据新的调用对象重新提供一个 `this`。

## this 可显式写，也可省略

下面两种写法等价：

```cpp
happiness_ += 5;
```

```cpp
this->happiness_ += 5;
```

一般情况下可以省略 `this->`。当形参或局部变量与成员同名时，显式写 `this->` 可以消除歧义：

```cpp
class Student {
public:
    void setScore(int score) {
        this->score = score;
    }

private:
    int score;
};
```

## 当前对象与其他对象共同参与行为

成员函数的参数列表中通常只写“除当前对象以外的参与者”。

例如：

```cpp
class Hero {
public:
    void fight(Monster& monster);
};
```

`fight` 行为由两个对象参与：

- 当前 `Hero` 对象：由 `this` 表示
- 一个 `Monster` 对象：由参数 `monster` 表示

调用：

```cpp
hero.fight(monster);
```

可以理解为：

```cpp
Hero::fight(&hero, monster);
```

当前对象不需要写进显式参数列表，因为它由 `this` 隐含提供。

## return *this

如果成员函数希望返回当前对象本身，可以返回 `*this`。

```cpp
class Hero {
public:
    Hero& fight(Monster& monster) {
        happiness_ += 5;
        return *this;
    }

private:
    int happiness_ = 0;
};
```

`this` 是指针，`*this` 就是当前对象本身。

因为当前对象在函数结束后仍然存在，所以返回 `*this` 的引用是合法的。

## 返回引用支持链式调用

如果 `fight` 返回 `Hero&`：

```cpp
hero.fight(m1).fight(m1).fight(m2);
```

含义是：

1. `hero` 打一次 `m1`
2. 返回 `hero` 本身的引用
3. 继续让同一个 `hero` 再打一次 `m1`
4. 再返回同一个 `hero`
5. 继续打 `m2`

这叫 [[链式调用]]。

## 按值返回与返回引用的区别

如果写成按值返回：

```cpp
Hero fight(Monster& monster) {
    happiness_ += 5;
    return *this;
}
```

语法上也可能通过，但返回的是当前对象的副本。

```cpp
hero.fight(m1).fight(m1).fight(m2);
```

这时：

- 第一次 `fight(m1)` 修改原来的 `hero`
- 返回一个新的临时 `Hero` 副本
- 后续 `fight(m1)`、`fight(m2)` 操作的是副本，不再是原来的 `hero`

因此，如果希望连续调用都作用在同一个原对象上，应返回引用：

```cpp
Hero& fight(Monster& monster);
```

> [!important] 考点
> `return *this` 如果用于链式调用，通常返回 `T&`，不是 `T`。返回 `T` 会产生副本，行为可能完全不同。

## 本节考点整理

- `this` 是 C++ 关键字。
- `this` 只存在于非静态成员函数中。
- `this` 指向当前对象。
- `this` 可理解为 `T* const this`。
- `this` 本身不能改指向。
- 普通成员函数中可以省略 `this->`。
- 成员函数调用可理解为隐含传入当前对象地址。
- 参数列表中通常只写当前对象之外的参与对象。
- `*this` 表示当前对象本身。
- 返回 `*this` 的引用可以支持 [[链式调用]]。
- 按值返回 `*this` 会返回副本，后续调用不再作用于原对象。

## 本节速记

> `this` 指当前对象；  
> `this` 是指针，`*this` 是对象；  
> 链式调用要返回 `*this` 的引用。
