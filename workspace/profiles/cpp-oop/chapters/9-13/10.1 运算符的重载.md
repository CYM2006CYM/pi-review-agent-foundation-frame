# 10.1 运算符的重载

## 本节核心

[[运算符重载]] 让自定义类型也可以使用接近内置类型的运算写法。

例如一个二维向量类：

```cpp
class TVector {
public:
    int GetX() const { return x; }
    int GetY() const { return y; }

private:
    int x;
    int y;
};
```

两个向量相加时，如果只写普通函数，可以这样：

```cpp
TVector result = AddVector(v1, v2);
```

但是更符合数学表达习惯的写法是：

```cpp
TVector result = v1 + v2;
```

运算符重载的目标就是让 `+`、`-`、`[]`、`()` 等运算符在自定义类型上也有合理含义。

> [!important] 高频考点
> 运算符重载不是创造新运算符，而是为已有运算符在自定义类型上的使用提供函数定义。

## 为什么需要运算符重载

普通函数可以完成同样功能：

```cpp
TVector AddVector(const TVector& v1, const TVector& v2);
```

但普通函数名存在两个问题：

- 函数名由程序员自行命名，不同代码可能写成 `AddVector`、`add`、`plus` 等，阅读成本高。
- 对数学含义很强的操作，符号表达更自然。

运算符重载把“操作含义”放回运算符本身。

```cpp
v1 + v2
```

读代码的人不用先查 `AddVector` 名字，就能大致知道它表示向量加法。

## 运算符重载的基本形式

运算符重载函数名由 `operator` 加运算符组成：

```cpp
operator+
operator=
operator[]
operator()
```

例如：

```cpp
TVector operator+(const TVector& v1, const TVector& v2);
```

这里 `operator+` 可以理解成一个特殊函数名。

## 哪些运算符可以重载

大多数 C++ 运算符可以重载，例如：

```cpp
+  -   *   /   %   ^   &   |   ~
!   =   <   >   +=  -=  *=  /=  %=
^=  &=  |=  <<  >>  >>= <<= ==  !=
<=  >=  &&  ||  ++  --  ->* ,   ->
[]  ()  new delete
```

也有一些运算符不能重载：

```cpp
.    .*    ::    ?:
sizeof
typeid
static_cast
dynamic_cast
const_cast
reinterpret_cast
```

> [!warning] 易错点
> 不能重载 `.`、`.*`、`::`、`?:`。这些通常是选择题和判断题高频点。

## 运算符重载的限制

运算符重载不是任意改造 C++ 语法，它有严格边界。

第一，不能改变运算符的操作数个数。

`+` 原来是二元运算符时，重载后仍然应作为二元运算符使用。

第二，不能改变运算符优先级和结合性。

```cpp
a + b * c
```

重载后仍然先算 `*`，再算 `+`。

第三，不能发明新的运算符。

不能自己造出 `**`、`<>` 这样的新运算符再重载。

第四，至少有一个操作数必须是自定义类型。

不能试图改变内置类型之间原有运算：

```cpp
int operator+(int a, int b); // 错误：不能重载纯内置类型运算
```

第五，不要破坏常见语义。

虽然 C++ 允许你把 `+` 写成“减法”行为，但这会严重破坏代码可读性。

## 运算符重载的本质

运算符表达式最终会被编译器转换成函数调用。

例如：

```cpp
a + b
```

可以转换为类似：

```cpp
operator+(a, b);
```

或者如果 `+` 被实现为成员函数：

```cpp
a.operator+(b);
```

因此，运算符重载既有“符号写法”的外观，本质上仍然是函数调用。

## 本节考点整理

- [[运算符重载]] 让自定义类型能使用已有运算符。
- 运算符重载函数名写作 `operator` 加运算符。
- 运算符重载不能创造新运算符。
- 运算符重载不能改变操作数个数、优先级、结合性。
- 至少一个操作数必须是自定义类型。
- `.`、`.*`、`::`、`?:`、`sizeof`、类型转换关键字等不能重载。
- 设计重载运算符时，应保持与内置类型相近的使用习惯。
