# 10.3 成对重载

## 本节核心

[[成对重载]] 指一些语义相关的运算符最好一起提供。

最典型的例子是：

```cpp
operator+
operator+=
```

如果一个类支持：

```cpp
a = a + b;
```

通常也应支持：

```cpp
a += b;
```

这样能适应用户的不同使用习惯，也让类型更像内置类型。

## += 的语义

`+=` 表示“加并赋值”，会修改左侧对象。

```cpp
a += b;
```

含义是：

```cpp
a = a + b;
```

但 `+=` 更直接地表达“在原对象上修改”。

成员函数写法通常是：

```cpp
class A {
public:
    A& operator+=(const A& rhs) {
        x += rhs.x;
        y += rhs.y;
        return *this;
    }

private:
    int x;
    int y;
};
```

## += 为什么返回引用

`+=` 修改的是左侧当前对象，返回的也应是修改后的当前对象。

因此返回类型通常是：

```cpp
A&
```

函数末尾写：

```cpp
return *this;
```

这与 [[赋值函数]] 的返回思路一致：返回当前对象引用，支持继续参与表达式。

> [!warning] 易错点
> `+=` 不应返回局部临时对象引用，也不应随便按值返回。它的语义是修改当前对象并返回当前对象。

## + 的语义

普通 `+` 不修改原对象，而是生成新对象。

```cpp
A operator+(const A& lhs, const A& rhs) {
    return A(lhs.x + rhs.x, lhs.y + rhs.y);
}
```

如果写成成员函数，也通常不修改当前对象：

```cpp
A operator+(const A& rhs) const {
    return A(x + rhs.x, y + rhs.y);
}
```

所以 `+` 常按值返回。

## 用 += 实现 +

一种常见写法是先写好 `+=`，再用它实现 `+`。

```cpp
A operator+(A lhs, const A& rhs) {
    lhs += rhs;
    return lhs;
}
```

这里 `lhs` 故意按值传入，得到左操作数的一个副本。然后在副本上执行 `+=`，最后返回副本。

这样做的好处：

- `+=` 中维护真正的“加并赋值”逻辑。
- `+` 复用 `+=`，减少重复代码。
- 改成员数据结构时，只需重点维护 `+=`。

## 为什么建议成对提供

成对提供 `+` 和 `+=` 有三个好处：

1. 适应不同使用习惯。
2. 保持类型行为接近内置类型。
3. 便于复用实现，降低维护成本。

对于其他运算符也有类似关系：

| 普通运算 | 复合赋值运算 |
|---|---|
| `+` | `+=` |
| `-` | `-=` |
| `*` | `*=` |
| `/` | `/=` |
| `%` | `%=` |

## 本节考点整理

- [[成对重载]] 常见于 `+` 与 `+=`、`-` 与 `-=` 等。
- `+=` 会修改左侧对象，通常写成成员函数。
- `+=` 通常返回 `A&`，并 `return *this;`。
- `+` 通常不修改操作数，返回一个新对象。
- 可以先实现 `+=`，再用 `+=` 实现 `+`。
- 成对重载能保持自定义类型接近内置类型的使用习惯。
