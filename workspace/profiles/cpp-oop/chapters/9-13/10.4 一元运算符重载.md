# 10.4 一元运算符重载

## 本节核心

[[一元运算符重载]] 处理只有一个操作数的运算符。

本节重点用 `++` 说明：

- [[前置自增]]：`++a`
- [[后置自增]]：`a++`

两者都让对象增加，但返回语义不同。

## 内置类型中的前置 ++

```cpp
int a = 10;
++a;
```

前置 `++` 的语义是：

1. 先让 `a` 加 1。
2. 返回修改后的 `a`。

等价理解：

```cpp
a += 1;
return a;
```

## 内置类型中的后置 ++

```cpp
int a = 10;
a++;
```

后置 `++` 的语义是：

1. 先保存旧值。
2. 再让 `a` 加 1。
3. 返回修改前的旧值。

等价理解：

```cpp
int temp = a;
a += 1;
return temp;
```

自定义类型重载 `++` 时，也应保持这个语义差异。

## 前置 ++ 的重载

成员函数形式：

```cpp
class A {
public:
    A& operator++() {
        ++num;
        ++den;
        return *this;
    }

private:
    int num;
    int den;
};
```

表达式：

```cpp
++a;
```

会被理解为：

```cpp
a.operator++();
```

前置 `++` 修改当前对象后返回当前对象引用，所以返回类型常写为 `A&`。

## 后置 ++ 的重载

后置 `++` 的函数名同样是 `operator++`。为了和前置版本区分，C++ 规定后置 `++` 多一个无实际使用意义的 `int` 形参：

```cpp
A operator++(int) {
    A temp(*this);
    ++num;
    ++den;
    return temp;
}
```

表达式：

```cpp
a++;
```

会被理解为：

```cpp
a.operator++(0);
```

这个 `int` 参数只是语法标记，用来区分前置和后置，不是为了传递业务数据。

> [!warning] 易错点
> 前置 `++` 是 `operator++()`，后置 `++` 是 `operator++(int)`。后置版本里的 `int` 形参通常不用。

## 返回类型差异

前置 `++` 返回修改后的当前对象：

```cpp
A& operator++();
```

后置 `++` 返回修改前的旧值：

```cpp
A operator++(int);
```

因此后置 `++` 通常按值返回，因为旧值需要保存在临时副本中。

## 示例：分数对象自增

假设 `A` 表示分数，有分子 `num` 和分母 `den`：

```cpp
class A {
public:
    A(int a = 0, int b = 1) : num(a), den(b) {}

    A& operator++() {
        num += den;
        return *this;
    }

    A operator++(int) {
        A temp(*this);
        num += den;
        return temp;
    }

private:
    int num;
    int den;
};
```

如果 `b` 原来表示 `1/3`：

```cpp
b = ++a;
```

`b` 得到的是自增后的值。

```cpp
b = a++;
```

`b` 得到的是自增前的旧值，`a` 自身仍然完成自增。

## 本节考点整理

- 一元运算符只有一个操作数。
- 前置 `++a`：先加，再返回修改后的对象。
- 后置 `a++`：先保存旧值，再加，返回旧值。
- 前置 `++` 重载为 `operator++()`。
- 后置 `++` 重载为 `operator++(int)`。
- 后置版本中的 `int` 参数用于区分语法，不代表实际业务参数。
- 前置 `++` 通常返回 `A&`。
- 后置 `++` 通常返回 `A`。
