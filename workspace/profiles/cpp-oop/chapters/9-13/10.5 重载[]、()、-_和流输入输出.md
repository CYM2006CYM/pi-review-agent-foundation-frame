# 10.5 重载[]、()、->和流输入输出

## 本节核心

本节讨论几个特殊运算符的重载：

- [[下标运算符]]：`[]`
- [[函数调用运算符]]：`()`
- [[箭头运算符]]：`->`
- 流输出 `<<` 与流输入 `>>`

这些运算符有固定使用习惯，重载时更要贴近内置或标准库类型的语义。

## 重载 []

`[]` 常用于让对象像数组一样访问内部元素。

例如类中有一个数组：

```cpp
class A {
public:
    int operator[](int index) const {
        return nums[index];
    }

    int& operator[](int index) {
        return nums[index];
    }

private:
    int nums[100];
};
```

使用方式：

```cpp
A a1;
cout << a1[0] << endl;
a1[0] = 10;
```

## [] 为什么常写两个版本

第一个版本用于常对象：

```cpp
int operator[](int index) const;
```

它末尾有 `const`，表示不修改当前对象。返回值按值返回，适合读取。

第二个版本用于普通对象：

```cpp
int& operator[](int index);
```

它返回引用，使下标表达式可以出现在赋值号左边：

```cpp
a1[0] = 10;
```

如果只返回 `int`，`a1[0]` 得到的是一个值，不能作为可修改位置使用。

> [!warning] 易错点
> `[]` 的非 const 版本常返回引用，是为了支持 `a[i] = value`。const 版本不能返回普通可修改引用，否则会破坏常对象语义。

## 下标范围检查

课程示例为了突出运算符重载语法，可能省略下标范围检查。

实际工程中，`operator[]` 通常不做或少做检查以保持数组习惯；如果希望强检查，可以提供类似 `at()` 的成员函数。

复习时要区分：

- 语法重点：`operator[](int index)`。
- 设计重点：是否允许越界、是否返回引用、是否区分 const 对象。

## 重载 ()

`()` 是函数调用运算符。重载后，对象可以像函数一样被调用。

```cpp
class A {
public:
    int operator()() const {
        return 999;
    }

    int operator()(int a, int b) const {
        return a + b;
    }
};
```

使用方式：

```cpp
A a;
cout << a() << endl;
cout << a(10, 30) << endl;
```

重载了 `operator()` 的对象也称为 [[函数对象]]。

## () 的特点

`operator()` 的参数列表可以有多个形参：

```cpp
int operator()(int a, int b, int c);
```

这和大多数运算符不同。它常用于：

- 函数对象。
- 策略对象。
- 比较器。
- 模拟函数调用。
- 后续标准库算法和模板编程场景。

## 重载 ->

`->` 用于让对象表现得像指针。

例如：

```cpp
class B {
public:
    A* operator->() {
        return pa;
    }

private:
    A* pa;
};
```

使用：

```cpp
B b;
b->f();
```

会被理解为先调用：

```cpp
b.operator->()
```

然后对返回结果继续执行 `->f()`。

## -> 的返回要求

`operator->` 的返回值必须满足后续还能使用 `->`：

- 返回原始指针。
- 或返回另一个也重载了 `operator->` 的对象。

因此它通常用于智能指针、代理对象等场景。

> [!warning] 易错点
> `operator->` 不能随便返回普通值。返回结果必须能继续进行 `->` 操作。

## 重载流输出 <<

流输出通常写成非成员函数：

```cpp
ostream& operator<<(ostream& os, const A& obj) {
    os << obj.ToString();
    return os;
}
```

使用：

```cpp
cout << obj << endl;
```

为什么通常不用成员函数？

因为左操作数是 `cout`，类型是 `ostream`，不是我们自己的类。

如果写成成员函数，就要求左操作数是当前类对象：

```cpp
obj.operator<<(cout);
```

这不符合 `cout << obj` 的使用习惯。

## 重载流输入 >>

流输入也通常写成非成员函数：

```cpp
istream& operator>>(istream& is, A& obj) {
    is >> obj.x;
    return is;
}
```

注意第二个参数一般不能是 `const A&`，因为输入操作要修改对象。

为了支持连续输入输出，`<<` 和 `>>` 通常返回流对象引用：

```cpp
cout << a << b;
cin >> a >> b;
```

## 本节考点整理

- `[]` 常用于让对象像数组一样访问内部元素。
- `[]` 常写 const 与非 const 两个版本。
- 非 const 的 `operator[]` 常返回引用，以支持 `a[i] = value`。
- `()` 可让对象像函数一样调用，重载后对象称为 [[函数对象]]。
- `operator()` 可以有多个参数。
- `->` 的返回结果必须能继续使用 `->`。
- `<<`、`>>` 通常写成非成员函数，因为左操作数是流对象。
- `<<` 返回 `ostream&`，`>>` 返回 `istream&`，用于支持连续输入输出。
