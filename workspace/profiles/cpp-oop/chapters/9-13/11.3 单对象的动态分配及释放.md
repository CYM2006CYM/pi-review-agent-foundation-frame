# 11.3 单对象的动态分配及释放

## 本节核心

单个对象或变量的动态分配使用 [[new运算符]]，释放使用 [[delete运算符]]。

基本形式：

```cpp
T* p = new T(参数列表);
delete p;
```

`new` 在 [[自由存储区]] 中分配空间并构造对象；`delete` 先析构对象，再释放空间。

> [!important] 高频考点
> `new/delete` 不只是 `malloc/free` 的替代品。对类对象来说，`new` 会调用构造函数，`delete` 会调用析构函数。

## new 的基本用法

动态创建一个整数：

```cpp
int* p1 = new int;
```

动态创建并初始化：

```cpp
int* p2 = new int(5);
```

动态创建类对象：

```cpp
A* p3 = new A;
A* p4 = new A(100, 200);
```

这里的参数列表会传给对应构造函数。

## 动态对象没有名字

```cpp
A* p = new A(10);
```

真正创建出来的 `A` 对象位于堆区，它本身没有普通变量名。

程序通过返回的地址访问它：

```cpp
p->Func();
```

因此，指针变量 `p` 是访问堆对象的入口。如果丢失了这个地址，就再也无法正常释放该对象，会造成 [[内存泄漏]]。

## delete 的基本用法

对于单个对象，用普通 `delete`：

```cpp
delete p;
```

释放过程包括：

1. 如果 `p` 非空，先调用对象析构函数。
2. 再释放对象占用的堆内存。

释放后，指针变量本身还存在，但它保存的地址已经无效。为了减少误用，常见习惯是：

```cpp
delete p;
p = nullptr;
```

## new 与 delete 必须匹配

单个对象：

```cpp
A* p = new A;
delete p;
```

数组对象：

```cpp
A* arr = new A[50];
delete[] arr;
```

本节重点是单个对象，数组形式会在后续章节展开。

> [!warning] 易错点
> `new` 对应 `delete`，`new[]` 对应 `delete[]`。两者混用会导致未定义行为。

## new 的底层过程

创建单个对象：

```cpp
T* p = new T(1, 2);
```

大致过程可以理解为：

1. 调用 `operator new(sizeof(T))` 申请足够存放 `T` 的原始内存。
2. 如果申请失败，按规则处理失败，可能抛出异常或调用 [[new_handler]]。
3. 在这块内存上调用 `T` 的构造函数。
4. 返回对象地址给指针 `p`。

因此，`new` 是“分配空间 + 构造对象”的组合动作。

## delete 的底层过程

释放单个对象：

```cpp
delete p;
```

大致过程可以理解为：

1. 如果 `p` 为空指针，直接返回。
2. 调用 `T` 的析构函数。
3. 调用 `operator delete(p)` 释放原始内存。

因此，`delete` 是“析构对象 + 释放空间”的组合动作。

## new/delete 与 malloc/free

可以粗略类比：

| C++ | C |
|---|---|
| `new` | `malloc` |
| `delete` | `free` |

但两组并不等价。

| 对比点 | `new/delete` | `malloc/free` |
|---|---|---|
| 是否是 C++ 语法 | 是 | 来自 C 标准库 |
| 是否知道类型 | 知道 | 只按字节数申请 |
| 构造/析构 | 会调用构造和析构 | 不会 |
| 返回类型 | 对应类型指针 | `void*` |

在 C++ 对象管理中，通常优先使用 `new/delete`；更现代的写法则优先使用 [[智能指针]]，减少手动释放风险。

## 本节考点整理

- 单对象动态分配格式：`T* p = new T(参数列表);`
- 单对象释放格式：`delete p;`
- `new` 在堆区申请空间，并调用构造函数。
- `delete` 调用析构函数，并释放堆区空间。
- 动态对象本身没有普通变量名，通常通过指针访问。
- 丢失堆对象地址会导致 [[内存泄漏]]。
- `new/delete` 与 `malloc/free` 类似但不等价。
- `new` 与 `delete` 要匹配，`new[]` 与 `delete[]` 要匹配。
