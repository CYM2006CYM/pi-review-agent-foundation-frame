# 11.4 重载operator new和operator delete

## 本节核心

`new` 和 `delete` 是完整表达式；其中真正可由用户定制的是底层的 [[operator new]] 和 [[operator delete]]。

单对象 `new` 可以理解为三步：

1. 调用 `operator new(sizeof(T))` 分配原始内存。
2. 在这块内存上调用构造函数。
3. 把对象地址返回给指针。

单对象 `delete` 也可以理解为三步：

1. 判断指针是否为空。
2. 调用析构函数。
3. 调用 `operator delete(p)` 释放原始内存。

> [!important] 高频考点
> 重载 `operator new/operator delete` 定制的是“原始内存分配和释放”，不是替换构造函数和析构函数。

## operator new 的基本形式

类中可以重载 `operator new`：

```cpp
class A {
public:
    static void* operator new(size_t size);
};
```

即使不写 `static`，它本质上也是静态成员函数。

原因是：调用 `operator new` 时，对象还没有构造完成，不存在可用的 `this` 指针。

返回类型通常是：

```cpp
void*
```

因为此时只是申请原始内存，还没有真正形成 `A*` 对象指针。

## size_t 参数

`operator new` 的第一个参数通常是：

```cpp
size_t size
```

它表示需要申请多少字节。

```cpp
A* p = new A(9);
```

会把 `sizeof(A)` 作为大小传给 `A::operator new`。

`size_t` 是与平台相关的无符号整数类型。32 位和 64 位平台上，它的大小可能不同，所以比直接写 `unsigned int` 更合适。

## operator delete 的基本形式

类中可以重载 `operator delete`：

```cpp
class A {
public:
    static void operator delete(void* p);
};
```

也可以带大小参数：

```cpp
static void operator delete(void* p, size_t size);
```

它没有返回值，因为释放操作只负责回收内存。

`p` 是要释放的原始内存地址。

## 带额外参数的 operator new

`operator new` 可以重载多个版本。

例如：

```cpp
class A {
public:
    static void* operator new(size_t size);
    static void* operator new(size_t size, const string& tag);
};
```

调用时可以写：

```cpp
A* p1 = new A(9);
A* p2 = new ("debug") A(8);
```

其中 `"debug"` 会匹配第二个 `operator new` 的额外参数。

这种形式常被称为 placement 风格的扩展写法，用于调试、内存池、带标签分配等场景。

## 类专属 operator new/delete

如果类中定义了自己的 `operator new`，创建该类对象时会优先使用类中的版本：

```cpp
A* p = new A;
```

如果类没有定义，就使用全局版本。

这可以用于：

- 统计对象创建次数。
- 使用内存池。
- 打印分配日志。
- 定制调试信息。

## 计数示例

类可以用静态数据成员记录当前动态对象数量：

```cpp
class A {
public:
    static void* operator new(size_t size) {
        ++newCount;
        return ::operator new(size);
    }

    static void operator delete(void* p) {
        --newCount;
        ::operator delete(p);
    }

private:
    static int newCount;
};
```

这里的 `::operator new` 表示调用全局版本，避免在类版本里递归调用自己。

> [!warning] 易错点
> 在类的 `operator new` 里，如果直接写 `operator new(size)`，可能再次调用自己，形成递归。要调用全局版本应写 `::operator new(size)`。

## 与构造析构的关系

即使类重载了 `operator new`：

```cpp
A* p = new A(10);
```

构造函数仍然会被调用。

即使类重载了 `operator delete`：

```cpp
delete p;
```

析构函数仍然会先被调用。

所以不要把它们混淆：

| 机制 | 负责内容 |
|---|---|
| `operator new` | 申请原始内存 |
| 构造函数 | 初始化对象 |
| 析构函数 | 清理对象内部资源 |
| `operator delete` | 释放原始内存 |

## 本节考点整理

- `new` 的底层第一步是调用 `operator new`。
- `delete` 的底层最后一步是调用 `operator delete`。
- `operator new` 返回 `void*`。
- `operator new` 的第一个参数通常是 `size_t`。
- `operator delete` 通常返回 `void`，参数包含待释放地址。
- 类中的 `operator new/delete` 本质上是静态成员函数，没有 `this` 指针。
- 重载 `operator new/delete` 不会替代构造函数和析构函数。
- 类中版本优先于全局版本。
- 在类版本中调用全局分配函数时，用 `::operator new` 和 `::operator delete`。
