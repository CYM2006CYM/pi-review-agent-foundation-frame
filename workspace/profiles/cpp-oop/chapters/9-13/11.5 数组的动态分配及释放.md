# 11.5 数组的动态分配及释放

## 本节核心

数组的动态分配使用 `new[]`，释放必须使用 `delete[]`。

```cpp
T* p = new T[n];
delete[] p;
```

它解决的是：数组大小在编译期无法确定，需要到运行期间根据变量 `n` 决定。

> [!important] 高频考点
> `new T[n]` 必须配对 `delete[] p`。不能用普通 `delete p` 释放动态数组。

## 为什么需要动态数组

静态数组要求大小通常在编译期确定：

```cpp
const int N = 50;
T objs[N];
```

如果数组大小来自运行期间：

```cpp
int n;
cin >> n;
```

就需要动态分配：

```cpp
T* p = new T[n];
```

动态数组的优点：

- 元素个数可以由运行时数据决定。
- 不必预先写一个“足够大”的固定数组。
- 可以在不需要时释放，减少长期占用。

## new[] 的语义

```cpp
T* p = new T[n];
```

含义是：

1. 在 [[自由存储区]] 中分配能容纳 `n` 个 `T` 对象的连续空间。
2. 对每个元素调用构造函数。
3. 返回第一个元素的地址。

如果 `T` 是类类型，通常调用无参构造函数：

```cpp
A* arr = new A[50]; // 调用 50 次 A::A()
```

## delete[] 的语义

```cpp
delete[] p;
```

含义是：

1. 如果 `p` 为空，直接结束。
2. 对数组中的每个元素调用析构函数。
3. 一次性释放整个数组占用的空间。

如果数组有 50 个元素，就要析构 50 次。

## 为什么 delete[] 不需要写 n

释放时写法是：

```cpp
delete[] p;
```

不需要写：

```cpp
delete[50] p; // 错误写法
```

原因是动态数组分配时，运行时通常会额外记录数组元素个数或相关大小信息。`delete[]` 可以根据这些信息知道应该析构多少个元素、释放多大空间。

这也是 `new[]` 和 `delete[]` 必须配对的重要原因。

## new[] 与 delete[] 必须配对

正确：

```cpp
A* p = new A[50];
delete[] p;
```

错误：

```cpp
A* p = new A[50];
delete p; // 错误
```

如果用普通 `delete` 释放动态数组，可能只把第一个元素当作单对象处理，导致后续元素没有正确析构，产生资源泄漏或未定义行为。

反过来也错误：

```cpp
A* p = new A;
delete[] p; // 错误
```

## operator new[] 和 operator delete[]

动态数组底层会使用数组版本的分配和释放函数：

```cpp
void* operator new[](size_t size);
void operator delete[](void* p);
```

如果用户没有提供类专属版本，就使用全局版本。

注意不同编译器实现细节可能不同，但从语义上复习时应记住：

- 单对象使用 `operator new` / `operator delete`。
- 数组对象使用 `operator new[]` / `operator delete[]`。

## 对象数组示例

一个扑克牌类中可能需要动态保存若干张牌：

```cpp
class Poker {
public:
    Poker(int n) : num(n) {
        cards = new Card[num];
    }

    ~Poker() {
        delete[] cards;
    }

private:
    int num;
    Card* cards;
};
```

这里 `cards` 指向堆区中的 `Card` 数组。

析构函数必须使用：

```cpp
delete[] cards;
```

这样每个 `Card` 元素的析构函数才会正确执行。

## 本节考点整理

- 动态数组格式：`T* p = new T[n];`
- 动态数组释放格式：`delete[] p;`
- `n` 可以是运行时变量。
- 动态数组元素在堆区连续存储。
- `new[]` 会对每个元素调用构造函数。
- `delete[]` 会对每个元素调用析构函数。
- `delete[]` 不需要显式写元素个数。
- `new[]` 必须配对 `delete[]`。
- `new`/`delete` 与 `new[]`/`delete[]` 混用会导致未定义行为。
