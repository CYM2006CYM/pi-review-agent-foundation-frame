# 11.6 智能指针和共享指针

## 本节核心

[[动态内存管理]] 的优点是灵活，缺点是程序员必须记得释放资源。

[[智能指针]] 的思想是：用一个局部对象托管堆区资源，让局部对象析构时自动释放堆对象。

这把“动态分配的灵活性”和“局部对象自动析构的安全性”结合起来。

> [!important] 高频考点
> 智能指针的核心不是“指针变聪明”，而是用对象生命周期管理资源，也就是 [[RAII]] 思想。

## 裸指针的问题

普通写法：

```cpp
T* p = new T;
p->f();
delete p;
```

如果忘记：

```cpp
delete p;
```

就会产生 [[内存泄漏]]。

裸指针的问题在于：创建和释放分散在不同语句中，程序员必须保证每条路径都正确释放。

## auto_ptr 的基本思想

课程用一个简化版 `auto_ptr` 解释智能指针思想：

```cpp
class auto_ptr {
public:
    auto_ptr(T* p) : pt(p) {}

    ~auto_ptr() {
        delete pt;
    }

    T* operator->() {
        return pt;
    }

    T& operator*() {
        return *pt;
    }

private:
    T* pt;
};
```

它内部仍然保存一个普通指针 `pt`，但析构函数负责释放它。

## 使用方式

```cpp
int main() {
    auto_ptr pt(new T);

    pt->f();
    (*pt).f();
}
```

这里没有显式写：

```cpp
delete
```

因为 `pt` 是局部对象，离开作用域时会自动调用析构函数，析构函数中执行 `delete pt;`。

## 为什么能像指针一样使用

智能指针类重载了两个运算符：

```cpp
T* operator->();
T& operator*();
```

因此：

```cpp
pt->f();
```

会通过 `operator->` 访问内部指针指向的对象。

```cpp
(*pt).f();
```

会通过 `operator*` 取得被托管对象本身。

> [!warning] 易错点
> `(*pt).f()` 必须有括号，因为点运算符 `.` 的优先级高于解引用 `*`。

## auto_ptr 的问题

简化版 `auto_ptr` 能说明思想，但真实使用会遇到问题：

- 拷贝构造怎么处理？
- 赋值函数怎么处理？
- 多个智能指针能不能指向同一资源？
- 多线程环境下如何保证安全？
- 谁负责最终释放资源？

这些问题说明：只在析构函数里 `delete` 还不够，还要设计资源所有权模型。

历史上的 `std::auto_ptr` 已经被废弃。现代 C++ 通常使用：

- `std::unique_ptr`
- `std::shared_ptr`
- `std::weak_ptr`

本课程接下来重点用 [[共享指针]] 解释“多个对象共享同一资源时如何释放”的问题。

## shared_ptr 的基本使用

标准库中的共享指针大致写法：

```cpp
#include <memory>

std::shared_ptr<T> pt(new T);
pt->f();
```

`shared_ptr` 允许多个智能指针共享同一个堆对象，并通过 [[引用计数]] 判断什么时候真正释放资源。

## 本节考点整理

- 裸指针需要手动 `delete`，容易忘记释放。
- 智能指针用对象析构自动释放资源。
- 智能指针体现 [[RAII]] 思想。
- 简化版智能指针内部保存裸指针。
- 析构函数中释放托管资源。
- 重载 `operator->` 让智能指针像指针一样访问成员。
- 重载 `operator*` 让智能指针支持解引用。
- `auto_ptr` 思想重要，但现代 C++ 中已废弃。
- [[共享指针]] 通过引用计数支持多个对象共享同一资源。
