# 11.7 共享指针的实现1

## 本节核心

[[共享指针]] 的目标是：多个对象可以共享同一个堆区资源，当最后一个使用者离开时，再释放该资源。

本节先介绍第一种实现思路：把 [[引用计数]] 放到被共享对象内部。

> [!important] 高频考点
> 共享指针不能简单地“多个指针指向同一对象然后都 delete”。必须记录有多少使用者，最后一个使用者释放时才真正删除资源。

## 为什么需要共享

假设 `B` 类内部有一个指向 `A` 对象的指针：

```cpp
class B {
private:
    A* pa;
};
```

如果 `B` 的拷贝构造函数执行深拷贝，就需要复制 `A` 对象。

但 `A` 可能很复杂：

- 拷贝成本高。
- 拷贝构造函数可能不可用。
- `A` 内部还可能有更深层资源。

这时可以考虑让多个 `B` 对象共享同一个 `A` 对象。

## 普通拷贝与共享拷贝

普通深拷贝：

```text
B1 -> A(8)
B2 -> A(8)  // 另一个独立 A 对象
```

共享思想：

```text
B1 ┐
   ├-> A(8)
B2 ┘
```

`B2` 由 `B1` 拷贝构造而来时，不再新建一个 `A(8)`，而是让 `B2.pa` 指向同一个 `A`。

## 引用计数

为了知道一个 `A` 对象被几个 `B` 对象共享，可以在 `A` 中加入计数字段：

```cpp
class A {
public:
    A(int data = 0) : data(data), use(1) {}

    void AddUse() {
        ++use;
    }

    int ReleaseUse() {
        return --use;
    }

private:
    int data;
    int use;
};
```

这里 `use` 表示当前 `A` 对象被使用的次数。

创建一个新 `A` 时，默认引用计数是 1。

## 构造 B 对象

普通构造：

```cpp
B::B(int n) {
    pa = new A(n);
}
```

此时新创建的 `A` 对象只被当前 `B` 使用，所以引用计数为 1。

## 拷贝构造 B 对象

共享指针风格的拷贝构造不复制 `A`，而是共享 `A`：

```cpp
B::B(const B& rhs) {
    pa = rhs.pa;
    pa->AddUse();
}
```

含义：

1. 新对象的 `pa` 指向旧对象的同一个 `A`。
2. 这个 `A` 又多了一个使用者，所以引用计数加 1。

例如：

```cpp
B b1(8);  // A(8), use = 1
B b2(b1); // 共享 A(8), use = 2
```

## 析构 B 对象

析构时不能直接删除 `pa`。

因为可能还有其他 `B` 对象正在共享同一个 `A`。

正确思路：

```cpp
B::~B() {
    if (pa->ReleaseUse() == 0) {
        delete pa;
    }
}
```

含义：

- 当前 `B` 不再使用这个 `A`，所以计数减 1。
- 如果减完后仍大于 0，说明还有别的对象在用，不能删除。
- 如果减到 0，说明没人用了，才能删除 `A`。

## 计数变化示例

```cpp
B b1(8);
B b2(b1);
B b3(10);
```

对应状态：

```text
b1 ┐
   ├-> A(8), use = 2
b2 ┘

b3 -> A(10), use = 1
```

析构顺序如果是 `b3`、`b2`、`b1`：

1. `b3` 析构：`A(10)` 的计数从 1 到 0，删除 `A(10)`。
2. `b2` 析构：`A(8)` 的计数从 2 到 1，不删除。
3. `b1` 析构：`A(8)` 的计数从 1 到 0，删除 `A(8)`。

## 本节方法的局限

把引用计数放进 `A` 内部，能说明共享指针原理，但有明显限制：

- 必须修改被共享类 `A` 的定义。
- 不是所有类都允许加入计数字段。
- 计数字段会污染业务对象。
- 线程安全问题还没有处理。

后续 [[11.8 共享指针的实现2]] 会继续改进实现思路。

## 本节考点整理

- 共享指针让多个对象共享同一个堆区资源。
- 引用计数记录资源当前被多少对象使用。
- 拷贝构造共享资源，并让引用计数加 1。
- 析构时引用计数减 1。
- 计数减到 0 时才真正删除资源。
- 不能多个对象直接对同一裸指针分别 `delete`。
- 把计数放在被共享对象内部简单直观，但会要求修改被共享类。
