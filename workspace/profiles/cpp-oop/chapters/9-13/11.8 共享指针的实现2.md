# 11.8 共享指针的实现2

## 本节核心

上一节 [[11.7 共享指针的实现1]] 把 [[引用计数]] 放进被共享类 `A` 内部。

这种方式直观，但要求能修改 `A` 的源码。现实中，被共享类可能来自库、框架或第三方代码，不能随便加成员。

本节介绍第二种方式：不修改 `A`，而是在管理类 `B` 中维护一个指向引用计数的指针。

> [!important] 高频考点
> 引用计数不能简单放成 `B` 的普通 `int` 成员，也不能放成 `B` 的静态 `int` 成员；更合理的是让共享同一资源的一组对象共享同一个计数对象。

## 为什么不能把计数作为普通成员

假设在 `B` 中直接放一个普通成员：

```cpp
class B {
private:
    A* pa;
    int use;
};
```

问题是：每个 `B` 对象都有自己的 `use`。

当 `B b2(b1);` 发生时，`b1` 和 `b2` 共享同一个 `A`，但它们各自有一份 `use`。如果只改 `b2.use`，`b1.use` 不知道；如果想同时改所有共享者的 `use`，就必须找到所有共享同一 `A` 的 `B` 对象，实现很困难。

普通成员不能自然表达“这一组共享者共同拥有一个计数”。

## 为什么不能把计数作为静态成员

再考虑静态成员：

```cpp
class B {
private:
    A* pa;
    static int use;
};
```

静态成员属于整个类，所有 `B` 对象共享同一个 `use`。

这又太粗了。

如果：

```cpp
B b1(8);
B b2(b1);
B b3(10);
```

`b1` 和 `b2` 共享 `A(8)`，计数应为 2；`b3` 独占 `A(10)`，计数应为 1。

但静态 `use` 只有一份，无法同时表示两组不同资源的计数。

## 用指针指向计数

更合适的结构是：

```cpp
class B {
private:
    A* pa;
    int* puse;
};
```

其中：

- `pa` 指向被共享的 `A` 对象。
- `puse` 指向堆区中的引用计数。

共享同一个 `A` 的 `B` 对象，也共享同一个 `puse`。

不同的 `A` 对象，则对应不同的 `puse`。

## 构造函数

首次创建 `B` 对象时，新建一个 `A`，同时新建一个计数：

```cpp
B::B(int num) {
    pa = new A(num);
    puse = new int(1);
}
```

此时只有当前 `B` 使用该 `A`，所以计数是 1。

## 增加引用计数

可以封装一个辅助函数：

```cpp
void B::AddRef() {
    ++(*puse);
}
```

注意 `puse` 是指针，真正的计数值是 `*puse`。

## 释放引用计数

释放时先减计数：

```cpp
void B::ReleaseRef() {
    if (--(*puse) == 0) {
        delete pa;
        delete puse;
    }
}
```

含义：

- 当前对象不再使用资源，计数减 1。
- 如果减到 0，说明没有任何 `B` 对象使用该 `A`。
- 此时释放 `pa` 指向的 `A`，也释放 `puse` 指向的计数空间。

## 析构函数

析构函数只需要调用释放逻辑：

```cpp
B::~B() {
    ReleaseRef();
}
```

这样每个 `B` 析构时都会减少一次引用计数。

## 拷贝构造函数

拷贝构造时，共享资源和计数：

```cpp
B::B(const B& rhs) {
    pa = rhs.pa;
    puse = rhs.puse;
    AddRef();
}
```

含义：

1. 新对象指向同一个 `A`。
2. 新对象指向同一个计数。
3. 共享者增加一个，计数加 1。

## 赋值函数

赋值比拷贝构造更复杂，因为左侧对象原来可能已经管理着另一份资源。

```cpp
B& B::operator=(const B& rhs) {
    if (this != &rhs) {
        ReleaseRef();
        pa = rhs.pa;
        puse = rhs.puse;
        AddRef();
    }

    return *this;
}
```

步骤：

1. 判断 [[自我赋值]]。
2. 当前对象先放弃原来管理的资源，计数减 1，必要时释放旧资源。
3. 当前对象指向右侧对象的 `pa` 和 `puse`。
4. 新共享的资源计数加 1。
5. 返回 `*this`。

## 两种实现方式对比

| 方式 | 引用计数位置 | 优点 | 缺点 |
|---|---|---|---|
| 方法一 | 被共享类 `A` 内部 | 直观，结构简单 | 必须修改 `A` |
| 方法二 | 管理类 `B` 中的计数指针 | 不需要修改 `A`，通用性更好 | 实现更复杂 |

方法二更接近通用共享指针的思想：引用计数不属于业务对象本身，而属于资源管理机制。

## 本节考点整理

- 把计数放进 `A` 内部要求能修改 `A` 源码。
- 把计数作为 `B` 普通成员，会导致每个 `B` 各有一份计数，无法统一。
- 把计数作为 `B` 静态成员，会导致所有资源共用一个计数，粒度太粗。
- 用 `int* puse` 指向堆区计数，可以让同一组共享者共享同一计数。
- 构造时 `puse = new int(1)`。
- 拷贝构造时共享 `pa` 和 `puse`，并加计数。
- 析构时减计数，计数为 0 时释放资源和计数。
- 赋值函数要先释放旧引用，再共享新引用，并处理自我赋值。
