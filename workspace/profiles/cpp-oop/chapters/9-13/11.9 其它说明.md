# 11.9 其它说明

这一节补充几个动态内存管理中非常容易写错的问题：`delete void*`、释放后的指针状态、[[内存泄漏]]、共享对象的[[写时复制]]，以及在指定内存上构造对象的[[定位分配]]。

## `delete void*` 的问题

普通 `delete p;` 可以理解为三步：

1. 判断指针是否为空；如果为空，什么也不做。
2. 调用指针所指对象类型的[[析构函数]]。
3. 调用[[operator delete]] 回收内存。

如果把对象指针强制转成 `void*` 后再 `delete`，第二步会出问题。因为 `void*` 只表示“某块地址”，不携带对象的静态类型信息，编译器无法知道应该调用哪个析构函数。

```cpp
A* p = new A;
void* q = p;

delete q; // 错误用法：对象析构逻辑不会按 A 来执行
```

关键点是：内存回收和对象析构不是一回事。`delete void*` 即使看起来“释放了内存”，也可能跳过对象的析构逻辑，导致对象内部资源没有正确释放。

> [!warning] 易错点
> 释放动态对象时，`delete` 后面的指针类型必须和真实对象类型匹配。不要把对象指针转成 `void*` 再删除。

## `delete` 不会修改指针本身

执行 `delete p;` 后，被指向的对象已经销毁，内存也被归还，但指针变量 `p` 本身保存的地址值不会自动改变。

```cpp
A* p = new A;
delete p;

// p 仍然保存旧地址，但旧地址已经无效
```

因此，下面这种代码在语法上可能通过，但语义上已经错误：

```cpp
delete p;
p->f(); // 错误：通过已经失效的指针访问对象
```

这种仍保存旧地址、但旧地址已经不再对应有效对象的指针，称为[[悬空指针]]，也常被称为野指针或无效指针。

实际编程中，如果 `delete p;` 不是这个指针最后一次出现，释放后通常应立即写成：

```cpp
delete p;
p = nullptr;
```

这样后续代码至少可以通过空指针判断避免继续使用旧地址。

## 内存泄漏为什么常见

动态分配对象时，`new` 和 `delete` 往往不在相邻位置，甚至可能分散在不同函数、不同文件、不同开发者维护的代码中。只要某条执行路径分配了资源，却没有对应释放，就可能产生[[内存泄漏]]。

```cpp
A* p = new A;

// 中间逻辑很长，可能有多条 return 或异常路径

delete p;
```

当项目规模变大后，单靠人工记忆“哪里 new 过、哪里 delete”并不可靠。内存泄漏检测工具可以帮助定位问题，但也不应把正确性完全寄托在事后检查上。

更稳妥的设计是把资源释放绑定到对象生命周期中，也就是使用[[RAII]] 和[[智能指针]]，让对象销毁时自动释放其拥有的资源。

## 写时复制

[[共享指针]]通过[[引用计数]]让多个指针共享同一个对象。共享本身能减少重复拷贝，但如果其中一个使用者要修改对象，就会产生一个问题：修改会影响所有共享者。

[[写时复制]]的思想是：

- 读操作可以共享同一份对象。
- 写操作前先检查对象是否被多个指针共享。
- 如果只有一个拥有者，可以直接修改。
- 如果有多个拥有者，先复制出一份新对象，再修改新对象。

可以概括为：读时共享，写时复制。

```cpp
void SmartPtr::write_something()
{
    if (use_count() > 1) {
        // 旧对象仍由其他共享者使用
        // 当前指针复制出自己的新对象
        detach();
    }

    // 此时当前指针独占对象，可以安全修改
    ptr->change();
}
```

从资源管理角度看，写时复制通常需要做四件事：

1. 发现引用计数大于 1。
2. 让当前指针退出旧共享关系，旧计数减 1。
3. 拷贝出一个内容相同的新对象。
4. 当前指针持有新对象，新对象计数为 1。

> [!important] 高频考点
> 共享指针解决“多个指针共同管理同一资源”的释放问题；写时复制进一步解决“共享对象被修改时相互影响”的问题。

## 定位分配

普通 `new` 会完成两件事：先申请内存，再在这块内存上构造对象。[[定位分配]]则允许程序员指定一块已经存在的内存，让构造函数直接在这块内存上构造对象。

典型形式是 placement new：

```cpp
char data[sizeof(A) * 3];

A* p1 = new (data) A(1);
A* p2 = new (p1 + 1) A(2);
A* p3 = new (p1 + 2) A(3);
```

这里的 `data` 是一块原始存储空间，`new (data) A(1)` 的含义不是重新向系统申请内存，而是在 `data` 起始位置构造一个 `A` 对象。

定位分配的重点：

- 内存来源由程序员提供，可以来自栈区或堆区缓冲区。
- placement new 只负责在指定地址上构造对象。
- 对象销毁时不能直接依赖普通 `delete`。
- 应显式调用析构函数。

```cpp
p3->~A();
p2->~A();
p1->~A();
```

如果只让底层缓冲区消失，而没有调用对象析构函数，那么对象内部拥有的资源仍可能没有被释放。也就是说，内存空间回收了，不等于对象生命周期被正确结束。

> [!warning] 易错点
> placement new 之后通常要手动调用析构函数；普通 `delete` 适用于普通 `new` 得到的对象，不要混用对象构造方式和释放方式。

## 本节小结

- `delete void*` 会丢失对象类型信息，可能无法正确调用析构函数。
- `delete p;` 释放的是 `p` 指向的对象和内存，不会把 `p` 自动改成空指针。
- 释放后继续使用旧指针会形成[[悬空指针]]问题。
- [[内存泄漏]]常来自 `new` 和 `delete` 分散、路径复杂、异常或提前返回。
- [[写时复制]]让共享对象在读时共用、写时分离。
- [[定位分配]]是在已有内存上构造对象，必须特别注意显式析构。
