# 12.1 转换函数

这一节讨论 C++ 中用户自定义类型参与[[类型转换]]的方式。核心要分清两个方向：其他类型转换成本类，通常靠[[转换构造函数]]；本类转换成其他类型，通常靠[[转换函数]]。

## 类型转换的几类情况

程序中常见的类型转换大致有三类：

| 转换类型 | 例子 | 谁负责 |
|---|---|---|
| 内置类型之间转换 | `int` 转 `float`，`float` 转 `double` | 编译器按内置规则处理 |
| 内置类型与自定义类型转换 | `int` 转 `Fraction`，`Fraction` 转 `float` | 构造函数或转换函数 |
| 自定义类型之间转换 | `A` 转 `B`，`B` 转 `Fraction` | 转换构造函数或转换函数 |

内置类型之间，编译器能根据规则自动转换。例如把 `int` 实参传给 `float` 形参，通常可以完成[[隐式类型转换]]；把 `float` 强制转成 `int`，则属于[[强制类型转换]]。

## 转换构造函数：把别的类型变成本类

如果一个构造函数可以只用一个实参调用，它不仅能构造对象，也可能被编译器当作类型转换路径。

```cpp
class Fraction {
public:
    Fraction(int numerator, int denominator = 1);
    Fraction(const A& a);

private:
    int num;
    int den;
};
```

`Fraction(int numerator, int denominator = 1)` 可以用一个 `int` 构造 `Fraction`，因此也可以理解为“把 `int` 转成 `Fraction`”。

```cpp
void useFraction(const Fraction& fra);

A a;
useFraction(a); // 可通过 Fraction(const A&) 转换
useFraction(2); // 可通过 Fraction(int, int = 1) 转换成 2/1
```

这种构造函数称为[[转换构造函数]]。它写在目标类型中：要把 `A` 转成 `Fraction`，就在 `Fraction` 中提供接收 `A` 的构造函数。

## 转换函数：把本类变成别的类型

另一种方向是：对象已经是本类，想把它转换成其他类型。这时可以在本类中定义转换函数。

转换函数的基本形式：

```cpp
class T {
public:
    operator DestType() const {
        return DestType(...);
    }
};
```

它的特点：

- 函数名由 `operator` 加目标类型组成。
- 不写普通返回类型，因为目标类型已经写在函数名里。
- 函数体中通常必须 `return` 一个目标类型的值。
- 如果不修改当前对象，通常写成[[常成员函数]]。

例如 `Fraction` 可以转换成 `int` 或 `float`：

```cpp
class Fraction {
public:
    operator int() const {
        return num / den;
    }

    explicit operator float() const {
        return static_cast<float>(num) / den;
    }

private:
    int num;
    int den;
};
```

这里 `operator int()` 表示“把 `Fraction` 转成 `int`”；`operator float()` 表示“把 `Fraction` 转成 `float`”。

## `explicit` 对隐式转换的影响

转换函数也可以使用[[explicit关键字]]。一旦写成 `explicit operator float() const`，它就不能用于普通的隐式转换，只能显式调用。

```cpp
Fraction f(3, 2);

int n = f;                  // 可调用 operator int()
float x = static_cast<float>(f); // 可调用 explicit operator float()
```

如果某个函数需要 `float`，而 `operator float()` 被声明为 `explicit`，编译器不能直接隐式调用它，可能退而求其次走 `operator int()`，再把 `int` 转成 `float`。这样 `3/2` 可能先得到 `1`，再变成 `1.0`；而显式转 `float` 才得到 `1.5`。

> [!warning] 易错点
> 转换函数如果太多、太宽松，编译器可能选择出乎意料的转换路径。涉及精度、资源或语义变化时，应优先考虑 `explicit`。

## 两条转换路径会导致二义性

假设要把 `A` 转成 `B`，有两种写法：

```cpp
class B {
public:
    B(const A& a); // 在目标类型 B 中写转换构造函数
};
```

或者：

```cpp
class A {
public:
    operator B() const; // 在源类型 A 中写转换函数
};
```

如果两边都写，编译器可能不知道该走哪一条路径，从而产生[[二义性]]错误。

> [!important] 高频考点
> “源类型写转换函数，目标类型写转换构造函数”。同一方向不要随意提供多条等价转换路径。

## 本节小结

- [[转换构造函数]]：用其他类型构造本类对象，写在目标类型中。
- [[转换函数]]：把本类对象转换成其他类型，写在源类型中。
- 转换函数不写普通返回类型，格式是 `operator 目标类型() const`。
- `explicit` 可以禁止隐式转换，只允许显式转换。
- 多条转换路径可能导致二义性。
