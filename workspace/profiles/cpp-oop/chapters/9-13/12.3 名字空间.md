# 12.3 名字空间

这一节系统整理[[名字空间]]的几种形式：[[标准名字空间]]、[[当前名字空间]]、[[全局名字空间]]、自定义名字空间、嵌套名字空间、名字空间别名，以及[[匿名名字空间]]。

## 标准名字空间 `std`

C++ 标准库中的常用名字大多位于 `std` 中，例如 `cout`、`endl`、标准容器和标准函数等。

```cpp
#include <iostream>

int main()
{
    std::cout << "hello" << std::endl;
}
```

`std` 是 standard 的缩写，本质上也是一个名字空间。自己写代码时不要把 `std` 当作自定义名字空间名称。

## 当前名字空间与全局名字空间

如果一段代码不在任何 `namespace` 块中，它所在的就是[[全局名字空间]]。

```cpp
int abc = 100;

void myFunc()
{
    std::cout << abc << std::endl;
}
```

这里 `abc` 和 `myFunc` 都属于全局名字空间。直接写 `myFunc()` 表示在[[当前名字空间]]中查找；写 `::myFunc()` 表示明确从全局名字空间查找。

```cpp
myFunc();   // 当前名字空间中的 myFunc
::myFunc(); // 全局名字空间中的 myFunc
```

前面的 `::` 是[[作用域解析符]]。当它左边为空时，表示全局名字空间。

## 自定义名字空间

自定义名字空间用 `namespace` 定义：

```cpp
namespace My {
    int abc = 100;

    class T {
    };

    void myFunc();
}
```

使用其中名字时，需要写完整限定名：

```cpp
My::myFunc();
std::cout << My::abc << std::endl;
```

`My::abc` 中的 `My::` 限定了 `abc` 的查找范围。

## 嵌套名字空间

名字空间内部还可以定义名字空间：

```cpp
namespace My {
    namespace My1 {
        int efg = 200;
    }
}
```

访问嵌套名字空间中的名字：

```cpp
std::cout << My::My1::efg << std::endl;
```

这和类中的 `ClassName::member` 形式很像。可以把类名看成一种更特殊的名字归属范围。

## 名字空间可以分段定义

同一个名字空间的内容可以分散在多个文件或多个代码段中：

```cpp
namespace My {
    class T1 {};
}

namespace My {
    class T2 {};
}
```

这两个 `namespace My` 不是重新定义两个名字空间，而是向同一个 `My` 名字空间中继续添加名字。

> [!warning] 易错点
> 名字空间可以分段定义；类不能用同样方式拆成多段重复定义。

## 名字空间别名

名字空间名过长时，可以起别名：

```cpp
namespace me = My;

me::myFunc();
std::cout << me::abc << std::endl;
```

`me` 只是 `My` 的别名，不是新的名字空间。

## 匿名名字空间

[[匿名名字空间]]没有名字：

```cpp
namespace {
    int abc = 100;
}
```

它的作用类似于文件级内部链接：其中的名字只在当前编译单元中有效，不会和其他文件中的同名实体冲突。

现代 C++ 中，匿名名字空间常用于替代文件作用域的 `static` 变量或函数，让“只在本文件内可见”的意图更清晰。

## 本节小结

- `std` 是标准库常用名字所在的[[标准名字空间]]。
- 不在任何名字空间中的名字属于[[全局名字空间]]。
- `::name` 表示从全局名字空间查找。
- 自定义名字空间用 `namespace Name { ... }`。
- 名字空间可以嵌套，也可以分段定义。
- 名字空间别名只是别名，不是新空间。
- [[匿名名字空间]]用于限制名字在当前编译单元内可见。
