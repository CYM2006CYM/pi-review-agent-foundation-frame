# 12.4 名字汇入

这一节讨论如何把名字空间中的名字带入当前作用域。重点区分两种写法：`using namespace` 汇入整个名字空间，`using Name::x` 汇入指定名字。

## 为什么需要名字汇入

如果每次使用名字空间中的实体都写完整限定名，代码会变长：

```cpp
std::cout << std::endl;
My::myFunc();
My::My1::efg;
```

[[名字汇入]]允许把某些名字带入当前作用域，减少书写负担。

## `using namespace`：汇入整个名字空间

```cpp
using namespace std;
```

这句话表示把 `std` 名字空间中的名字汇入当前名字空间。它汇入的是“名字”，不关心这个名字代表变量、函数、类还是子名字空间。

```cpp
using namespace std;

cout << "hello" << endl;
```

这很方便，但也容易带来命名冲突，尤其不建议在头文件或大型项目的全局范围中滥用。

> [!warning] 易错点
> `using namespace first;` 汇入的是 `first` 中所有名字；如果另一个名字空间中也有同名实体，后续查找可能出现冲突。

## 作用域限制可以避免冲突

名字汇入受作用域限制。如果把 `using namespace` 放在一个代码块中，它只在这个块内有效。

```cpp
namespace first {
    int x = 5;
}

namespace second {
    double x = 3.1416;
}

int main()
{
    {
        using namespace first;
        std::cout << x << std::endl; // 5
    }

    {
        using namespace second;
        std::cout << x << std::endl; // 3.1416
    }
}
```

两个块中的 `x` 不冲突，因为前一个块结束后，前一次汇入的名字已经失效。

## `using Name::x`：汇入指定名字

另一种写法是只汇入某个名字：

```cpp
using first::x;
```

这表示把 `first` 名字空间中名为 `x` 的实体汇入当前作用域。这里的 `x` 仍然是“名字”，可能对应变量名、函数名、类名，也可能因为函数重载对应多个函数。

如果同一作用域中已经有一个 `x`，再写：

```cpp
using second::x;
```

就可能直接出现重名错误。

## 两种写法对比

| 写法 | 作用 | 风险 |
|---|---|---|
| `using namespace N;` | 汇入名字空间 `N` 中所有名字 | 冲突范围大 |
| `using N::x;` | 只汇入 `N` 中名为 `x` 的名字 | 冲突范围小，但仍可能重名 |
| `N::x` | 不汇入，只在使用处限定 | 最清晰，书写略长 |

实际工程中，头文件中应尽量避免 `using namespace`。在实现文件或局部代码块中，可以根据需要适度使用。

## 本节小结

- [[名字汇入]]的对象是名字，不是“变量本身”或“函数本身”。
- `using namespace N;` 汇入整个名字空间中的名字。
- `using N::x;` 只汇入指定名字。
- 汇入受作用域影响，离开作用域后失效。
- 汇入太多名字会提高命名冲突概率。
