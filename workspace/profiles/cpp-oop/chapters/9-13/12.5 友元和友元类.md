# 12.5 友元和友元类

这一节讨论[[友元函数]]和[[友元类]]。友元的目的，是在不把成员完全公开的前提下，让指定函数或指定类访问本类的私有成员。

## 友元为什么出现

类外普通函数不能直接访问类的私有成员：

```cpp
class A {
private:
    int field;
};

void useA(A& a)
{
    a.field = 5; // 错误：field 是 A 的私有成员
}
```

一种做法是提供 public 的 getter/setter，但这会让所有外部代码都能通过接口访问或修改数据。某些场景下，我们只希望极少数函数或类拥有特殊权限，这时可以使用友元。

## 友元函数

在类中使用 `friend` 声明一个普通函数，可以让这个函数访问本类私有成员。

```cpp
class A {
    friend void useA(A& a);

private:
    int field;
};

void useA(A& a)
{
    a.field = 5; // 正确：useA 是 A 的友元函数
}
```

注意：`friend` 声明只是授予访问权限，并不把 `useA` 变成 `A` 的成员函数。

## 成员函数作为友元

也可以只把另一个类的某个成员函数声明为友元：

```cpp
class A;

class B {
public:
    void f(A& a);
};

class A {
    friend void B::f(A& a);

private:
    int value;
};
```

这样只有 `B::f` 能访问 `A` 的私有成员，`B` 的其他成员函数没有这个权限。

## 友元类

如果希望某个类的所有成员函数都能访问本类私有成员，可以声明友元类：

```cpp
class Parent;

class Wallet {
    friend class Parent;

private:
    int money;
    int history;
};
```

`friend class Parent;` 表示 `Parent` 类中的所有成员函数都能访问 `Wallet` 的私有成员。

友元类的授权比“某个成员函数作为友元”更宽，因此要更谨慎。

## 友元的几种形式

| 形式 | 例子 | 权限范围 |
|---|---|---|
| 自由函数友元 | `friend A operator+(...);` | 指定普通函数可访问 |
| 成员函数友元 | `friend void B::f(A&);` | 指定成员函数可访问 |
| 友元类 | `friend class C;` | 类 `C` 的所有成员函数可访问 |
| 嵌套类友元 | `friend class D;` | 指定嵌套类可访问 |

在[[运算符重载]]中，非成员形式的二元运算符经常会被声明为友元，因为它既需要保持左右操作数对称，又可能需要访问类的私有数据。

## 友元的性质

友元有几个重要性质：

1. 友元不是类的成员。
2. 友元声明放在 `public`、`private` 或 `protected` 区域，本质效果一样。
3. 友元关系是单向的。
4. 友元关系没有传递性。

单向性例子：如果 `Parent` 是 `Wallet` 的友元，表示 `Parent` 可以访问 `Wallet` 的私有成员；不表示 `Wallet` 可以访问 `Parent` 的私有成员。

无传递性例子：如果 `A` 是 `B` 的友元，`B` 是 `C` 的友元，不能推出 `A` 是 `C` 的友元。

> [!important] 高频考点
> 友元不是成员、关系单向、没有传递性。这三句话经常出选择题和判断题。

## 友元与封装

从局部看，友元确实让外部代码能访问私有成员，因此会削弱[[封装]]和[[信息隐蔽]]。

但从整体看，友元比完全公开 getter/setter 更精确：它只授权给少数明确对象，而不是让所有外部代码都能访问。

因此友元适合表达“少数特定函数或类确实需要特殊访问权”的情况，不适合当作逃避封装设计的常规手段。

> [!warning] 易错点
> 只有真正需要局部授权时才使用友元。滥用友元会让类的私有数据失去保护意义。

## 本节小结

- 友元用于让指定函数或类访问本类私有成员。
- [[友元函数]]不是类成员，只是被授权访问。
- [[友元类]]会让该类所有成员函数获得访问权。
- 友元关系单向、无传递性。
- 友元会削弱封装，应慎用。
