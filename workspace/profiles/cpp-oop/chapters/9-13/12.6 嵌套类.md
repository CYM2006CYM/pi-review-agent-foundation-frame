# 12.6 嵌套类

[[嵌套类]]就是在一个类内部再定义一个类。它的意义不是让语法变复杂，而是把只服务于外部类的辅助类型收进外部类的作用域中，从而控制这些类型是否对外可见。

## 基本定义

```cpp
class Outer {
public:
    class InnerPub {
    public:
        void f1();
        static T1* p1;
    };

    void f();

private:
    class InnerPri;
};
```

嵌套类可以直接定义在外部类中，也可以先声明，再在类外定义。

```cpp
class Outer::InnerPri {
public:
    void f2();

private:
    int value;
};
```

类外实现嵌套类成员时，要把每层作用域都写出来：

```cpp
void Outer::InnerPub::f1()
{
}

void Outer::InnerPri::f2()
{
}
```

## 实例化嵌套类

如果嵌套类是 `public` 的，类外可以通过 `Outer::InnerPub` 使用它：

```cpp
int main()
{
    Outer outer;
    Outer::InnerPub c1; // 正确
}
```

如果嵌套类是 `private` 的，外部不能直接实例化：

```cpp
int main()
{
    Outer::InnerPri c2; // 错误：InnerPri 是 Outer 的私有成员
}
```

但是外部类自己的成员函数可以使用自己的私有嵌套类：

```cpp
void Outer::f()
{
    InnerPri obj; // 正确：Outer 的成员函数可访问 Outer 的私有成员
}
```

> [!important] 高频考点
> 嵌套类本身也受外部类的访问控制影响。`public` 嵌套类可在类外通过 `Outer::Inner` 使用；`private` 嵌套类只能由外部类内部使用。

## 嵌套类和普通类的关系

嵌套类本质上仍然是类，它内部的 `public`、`private`、`protected` 访问控制照常生效。

关键区别是：嵌套类这个“类型名”本身也成为外部类的一个成员名字，因此它会受到外部类访问控制限制。

```cpp
class E {
    int x;

    class B {
    };

    class I {
        B b;     // 正确：同在 E 的内部范围中，可见 B
        int y;
        void f(E* p) {
            p->x = 1; // 可访问 E 的私有成员
        }
    };

    int g(I* p) {
        return p->y; // 错误：y 是 I 的私有成员
    }
};
```

这里要区分两件事：

- 嵌套类的成员函数在一定规则下可以访问外部类的私有成员。
- 外部类的成员函数不能随便访问嵌套类对象的私有成员。

嵌套类仍然是独立类，不等于把所有私有权限互相打通。

## 嵌套类的使用意义

嵌套类常用于把辅助类型隐藏在主类内部。

如果把所有辅助类都平铺在外部作用域中，所有代码都能看到这些类；如果把它们设计成嵌套类，尤其是 `private` 嵌套类，就可以表达“这个类型只是当前类的实现细节”。

适合使用嵌套类的情况：

- 辅助类只为外部类服务。
- 不希望外部代码依赖该辅助类。
- 希望降低全局命名空间污染。
- 希望把相关类型组织在同一个类的语义范围内。

> [!warning] 易错点
> C++ 的嵌套类和 Java 的内部类不是同一个概念。学习 Java 时不要直接套用这里的理解。

## 本节小结

- [[嵌套类]]是在类内部定义的类。
- 嵌套类的类型名受外部类访问控制影响。
- 类外使用 public 嵌套类要写 `Outer::Inner`。
- private 嵌套类不能在外部直接实例化。
- 嵌套类内部仍按普通类规则进行访问控制。
- 嵌套类常用于隐藏实现细节和组织辅助类型。
