# 12.7 流

这一节介绍[[流]]的基本概念，以及常用的 `cin`、`cout`、`endl`。流可以理解为一条按顺序传输数据的管道，数据按进入顺序流出。

## 流的基本概念

流来自“管道”的类比：数据从一端进入，只能按顺序从另一端流出。

流的特点：

- 按顺序传输。
- 进入顺序和流出顺序保持一致。
- 常用于输入、输出和文件读写。
- 随机访问能力弱，更适合顺序处理。

```text
数据进入 -> [ 流 ] -> 数据流出
```

## 字节流与字符流

从底层看，流通常以字节为单位传输；但从应用角度，又常区分[[字节流]]和[[字符流]]。

| 类型 | 关注点 | 例子 |
|---|---|---|
| 字节流 | 一个字节一个字节传输，不解释内容含义 | 二进制文件读写 |
| 字符流 | 按字符语义处理数据 | 文本文件、控制台文本输入输出 |

一个字符不一定等于一个字节。比如汉字在不同编码中可能占 2 个、3 个甚至更多字节。因此读取文本时必须知道编码格式。

## 文本文件与二进制文件

文件读写常见两种方式：

| 文件方式 | 处理思路 |
|---|---|
| [[二进制文件]] | 按字节读取和写入，不关心内容表示 |
| [[文本文件]] | 按字符或文本编码解释内容 |

例如源代码文件可能使用 UTF-8、UTF-16、GBK 等编码。如果文件保存编码和编译器/运行环境理解的编码不一致，就可能出现乱码。

> [!warning] 易错点
> 乱码常常不是 `cout` 本身的问题，而是源文件编码、编译选项、运行终端编码之间不一致。

## 输入流和输出流

C++ 标准库中有[[输入流]]和[[输出流]]相关类。`cin` 和 `cout` 是其中最常用的两个对象：

| 对象 | 含义 | 所在名字空间 |
|---|---|---|
| [[cin]] | 标准输入流对象，通常表示键盘输入 | `std` |
| [[cout]] | 标准输出流对象，通常表示控制台输出 | `std` |

注意：`cin` 和 `cout` 是对象，不是类，也不是函数。

```cpp
int n;
std::cin >> n;
std::cout << n;
```

## `endl` 的含义

`endl` 是标准名字空间中的函数式操纵符，不是普通对象。它通常完成两件事：

1. 向输出流写入换行。
2. 刷新输出缓冲区，也就是执行 flush。

```cpp
std::cout << i << std::endl;
```

与只输出 `'\n'` 相比，`endl` 会刷新缓冲区，因此用户更容易看到即时输出；但频繁刷新也会降低性能。

```cpp
std::cout << i << '\n';   // 通常更快，不强制刷新
std::cout << i << std::endl; // 换行并刷新
```

> [!important] 高频考点
> `endl` 不只是换行，还会刷新输出缓冲区。循环中频繁使用 `endl` 可能明显变慢。

## 本节小结

- [[流]]是按顺序传输数据的管道模型。
- [[字节流]]强调按字节传输，[[字符流]]强调按字符语义处理。
- 文本处理离不开编码格式，编码不一致可能导致乱码。
- [[cin]] 和[[cout]] 是 `std` 中的标准流对象。
- [[endl]] 是函数式操纵符，作用是换行并刷新。
