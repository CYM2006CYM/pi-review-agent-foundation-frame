# 13.3 关联

[[关联]] 是类之间较强的水平逻辑关系。它通常表现为：一个类把另一个类的对象、指针或引用作为自己的数据成员。

```cpp
class A;

class B {
private:
    A* mpA;
};
```

这里 `B` 中保存了 `A*`，说明 `B` 对象在自己的生命周期内有一个“知道 A 对象”的位置。即使 `mpA` 目前是 `nullptr`，也仍然说明 `B` 的结构设计中保留了和 `A` 发生联系的可能性。

## 关联的基本含义

关联强调的是非偶然性的“知道”。

如果 `B` 偶然在某个函数里临时使用了一次 `A`，通常只是[[依赖]]。如果 `B` 对象长期保存了一个 `A` 的成员，那么 `B` 在结构上就和 `A` 发生了关系，这就是关联。

关联可以是：

| 维度 | 说明 |
|---|---|
| 可导航方向 | 可以是[[单向联系]]，也可以是[[双向联系]] |
| 数量关系 | 可以是一对一、一对多、多对一、多对多 |
| 语义类型 | 可以是[[一般关联]]，也可以是[[聚集关联]] |

> [!important] 高频考点
> 判断是否是关联，先看一个类是否把另一个类作为数据成员保存；再进一步判断它只是“知道”，还是表达整体-部分关系。

## 一般关联

[[一般关联]] 只强调对象之间的非偶然联系，不强调整体和部分。

例如学生知道所在大学和选修课程：

```cpp
class University;
class Course;

class Student {
private:
    University* univ;
    Course* courses[30];
};
```

这里 `Student` 保存了 `University*` 和 `Course*`，表示学生长期知道自己的大学和课程。它不必表示“大学是学生的一部分”或“课程由学生创建和销毁”，所以这是一般关联。

UML 图中，学生到大学、学生到课程可以体现不同数量关系：

| 关系 | 含义 |
|---|---|
| 一对一 | 一个对象对应一个对象 |
| 一对多 | 一个对象对应多个对象 |
| 多对一 | 多个对象对应一个对象 |
| 多对多 | 多个对象之间互相对应 |

注意：UML 图画的是类之间的结构关系，不是某一个具体对象之间的即时状态。

## 自关联

[[自关联]] 是一个类和它自身发生关联。

```cpp
class Student {
private:
    Student* mates[3];
    Student* super;
};
```

这个例子中，一个学生可以记录室友，也可以记录负责人或优秀学生代表。`Student` 的成员仍然是 `Student*`，所以这是类和自身的关联。

> [!warning] 易错点
> 自关联不是“某个对象和自己相等”，而是“同一个类的不同对象之间存在联系”。例如学生甲和学生乙是室友，二者都属于 `Student` 类。

## 关联类

当“关系本身”也有属性和行为时，可以把关系抽象成一个单独的类，这就是[[关联类]]。

典型例子是学生、教师和课程。

```text
Student  <-- Course -->  Teacher
```

学生和教师之间可能是多对多关系。如果直接让学生保存教师列表、教师保存学生列表，会很快变复杂。更合理的做法是抽象出 `Course`：

- `Course` 记录课程名、学分、上课时间、考核方式。
- `Student` 通过课程知道自己学了什么。
- `Teacher` 通过课程知道自己教了什么。

这样，学生和教师之间的复杂关系被课程类承载，模型更清楚。

婚姻关系也可以用类似方式建模。丈夫和妻子之间的关系本身可能有结婚日期、登记地点、证书编号等信息，这些信息不天然属于丈夫或妻子单独一方，而属于“婚姻”这个关系。

## 本节小结

- 关联通常表现为一个类把另一个类作为数据成员保存。
- 一般关联强调长期、非偶然的“知道”。
- 自关联是同一个类的不同对象之间发生联系。
- 当关系本身有信息时，可以抽象出关联类。
- 关联比依赖更强，但是否进一步是聚合或组合，要看是否表达整体-部分及生命周期控制。
