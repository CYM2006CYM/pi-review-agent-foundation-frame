# 13.7 宿舍到学生的关系例

上一节从学生到宿舍方向分析关系。本节换成宿舍到学生方向，观察同一现实问题在不同关注点下如何变化。

## 宿舍到学生：三种可能

如果宿舍只是知道住了哪些学生，这是[[一般关联]]。

如果宿舍被看成整体，学生被看成部分，就进入[[聚集关联]]。此时还要继续判断：

| 情况 | 关系 |
|---|---|
| 宿舍负责学生对象的创建与销毁 | [[组合]] |
| 宿舍只保存外部学生对象，不负责创建与销毁 | [[聚合]] |

关键问题仍然是：宿舍是否控制学生对象的生命周期。

## 宿舍到学生：组合实现

如果模型规定宿舍创建时创建其中的学生，宿舍销毁时也销毁这些学生，那么是组合。

```cpp
class Student;

class Dorm {
public:
    Dorm()
    {
        for (int i = 0; i < 4; ++i) {
            mStudents[i] = new Student;
        }
    }

    ~Dorm()
    {
        for (int i = 0; i < 4; ++i) {
            delete mStudents[i];
        }
    }

private:
    Student* mStudents[4];
};
```

这里每个 `Student` 对象由 `Dorm` 创建，也由 `Dorm` 释放。学生对象的创建与消亡由宿舍控制，所以宿舍和学生是组合关系。

> [!warning] 易错点
> 这种例子是为了说明组合的生命周期责任，不代表现实中的宿舍真的“创建学生”。程序模型服务于问题域，不等于现实世界本身。

## 宿舍到学生：聚合实现

如果学生对象由外部创建，宿舍只保存这些学生指针，那么是聚合。

```cpp
class Dorm {
public:
    Dorm(Student** students, int count)
        : maxCount(count), mStudents(new Student*[maxCount])
    {
        for (int i = 0; i < count; ++i) {
            mStudents[i] = students[i];
        }
    }

    ~Dorm()
    {
        delete[] mStudents;
    }

    void addStudent(Student* s, int index)
    {
        if (index >= 0 && index < maxCount && mStudents[index] == nullptr) {
            mStudents[index] = s;
        }
    }

    void removeStudent(int index)
    {
        if (index >= 0 && index < maxCount) {
            mStudents[index] = nullptr;
        }
    }

private:
    int maxCount;
    Student** mStudents;
};
```

这里 `new Student*[maxCount]` 创建的是“指针数组”，不是学生对象本身。析构时 `delete[] mStudents` 释放的也是指针数组，而不是数组中指向的学生对象。

`removeStudent` 只是把床位设为空，不删除学生对象。学生对象从宿舍外部来，离开宿舍后仍然可以存在。因此宿舍不控制学生生命周期，这是聚合。

## 本节小结

- 从宿舍到学生方向，关系可以是一般关联、聚合或组合。
- 是否是聚集关联，取决于是否强调宿舍是整体、学生是部分。
- 聚合和组合的区别，取决于宿舍是否负责学生对象的创建与销毁。
- `new Student*[n]` 只创建指针数组，不创建学生对象。
- 把指针设为 `nullptr` 只是解除联系，不等于删除对象。
