# 9.1 对象的拷贝

## 本节核心

[[对象拷贝]] 指根据一个已有对象创建另一个内容相同的新对象。

这个过程需要一种特殊的 [[构造函数]]：[[拷贝构造函数]]。

它仍然是构造函数，因为它也是“从无到有”创建新对象；只是它创建对象时使用的样本是另一个同类型对象。

> [!important] 高频考点
> 只要新对象在当前语句之前不存在，本语句根据已有对象创建它，就是拷贝构造；如果左右两个对象在语句之前都已经存在，就是赋值。

## 为什么需要拷贝构造函数

普通构造函数可以从普通参数创建对象：

```cpp
A a;
```

但有些场景需要根据已有对象创建新对象。

例如函数按值传参：

```cpp
void func(A a) {
}

A obj;
func(obj);
```

调用 `func(obj)` 时，形参 `a` 是函数栈帧中的一个新对象。它不是原来的 `obj`，只是内容应当和 `obj` 一样。

因此，函数调用过程需要根据 `obj` 创建一个新的 `A` 对象，这就需要拷贝构造函数。

## 按值返回也可能需要拷贝

另一个典型场景是按值返回对象：

```cpp
A makeA() {
    A a;
    return a;
}
```

局部变量 `a` 位于函数栈区。函数返回后，`a` 的生命周期结束，不能把 `a` 本身直接交给调用者继续使用。

因此，返回值需要是一个新的 `A` 对象，它的内容来自局部变量 `a`。

这也是拷贝构造函数的典型使用场景。

## 拷贝构造函数的含义

拷贝构造函数的任务是：

> 根据一个已有的本类对象，创建一个新的本类对象。

常见格式：

```cpp
class A {
public:
    A(const A& other);
};
```

参数通常写成 `const A&`：

- `const` 表示不会修改作为样本的对象。
- `&` 避免传参时再次触发拷贝。

## 拷贝构造和赋值的区别

判断一条语句是拷贝构造还是赋值，关键看目标对象在语句之前是否已经存在。

```cpp
A a1;
A a2;

a2 = a1; // 赋值：a1 和 a2 在语句前都已存在
```

这里是 [[对象赋值]]，不是拷贝构造。

再看：

```cpp
B b1;
B b2(b1); // 拷贝构造：b2 在语句前不存在
```

`b2` 是从无到有创建出来的，调用拷贝构造函数。

## 等号初始化也可能是拷贝构造

下面这条语句虽然出现了 `=`，但仍然是拷贝构造：

```cpp
B b1;
B b3 = b1;
```

原因是：在这条语句之前，`b3` 不存在。

因此它不是“把已有 b1 赋给已有 b3”，而是“根据 b1 创建 b3”。

## 推荐写法

对于用户自定义类，建议用括号形式表达拷贝构造：

```cpp
B b2(b1);
```

这样能更清楚地表示“正在构造一个新对象”。

对于内置类型，使用 `=` 初始化更符合传统写法：

```cpp
int x = 10;
double y = 3.14;
```

## 本节考点整理

- [[拷贝构造函数]] 是构造函数的一种。
- 拷贝构造函数根据已有对象创建新对象。
- 按值传参可能触发拷贝构造。
- 按值返回对象可能触发拷贝构造。
- 拷贝构造和赋值的区别在于目标对象是否已经存在。
- `B b2(b1);` 是拷贝构造。
- `B b3 = b1;` 也是拷贝构造，因为 `b3` 之前不存在。
- `b2 = b1;` 是赋值，因为 `b1`、`b2` 都已经存在。

## 本节速记

> 新对象从旧对象来：拷贝构造；  
> 旧对象给旧对象改值：赋值。
