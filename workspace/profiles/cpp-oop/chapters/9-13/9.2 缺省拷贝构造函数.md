# 9.2 缺省拷贝构造函数

## 本节核心

如果用户没有自定义 [[拷贝构造函数]]，编译器会提供 [[缺省拷贝构造函数]]。

缺省拷贝构造函数的格式与自定义拷贝构造函数类似，并且通常是公有的。

它的拷贝方式通常被称为 [[浅拷贝]]。

> [!important] 高频考点
> 缺省拷贝构造函数不是“智能理解对象语义”，它通常按成员进行直接拷贝。遇到指针成员时，它拷贝的是指针值，不会自动复制指针指向的堆区对象。

## 编译器会自动提供

如果类中没有写拷贝构造函数：

```cpp
class A {
public:
    int value;
};
```

仍然可以写：

```cpp
A a1;
A a2(a1);
```

这里会使用编译器提供的缺省拷贝构造函数。

## 缺省拷贝构造函数做什么

缺省拷贝构造函数的基本思路是：把已有对象的数据成员复制到新对象中。

对于普通数据成员，例如整数、浮点数、指针值等，它一般做直接的成员拷贝。

这种拷贝不深入理解成员背后的资源关系，因此称为浅拷贝。

## 浅拷贝的含义

浅拷贝可以粗略理解为：

> 把对象自身存储区中的成员值复制一份。

例如：

```cpp
class A {
private:
    int value_;
    int* p_;
};
```

缺省拷贝构造函数会复制：

- `value_` 的数值
- `p_` 这个指针变量中保存的地址值

但它不会自动创建一个新的 `int`，也不会让两个对象各自拥有不同的堆区资源。

## 编译器不会替你判断所有语义

类的数据成员可能有很多种：

- 普通内置类型
- 指针
- 引用
- 对象类型成员
- `const` 成员

编译器提供的缺省拷贝构造函数不能自动知道“指针指向的资源是否也应该复制”。

因此，当类管理资源时，不能盲目依赖缺省拷贝构造函数。

## 本节考点整理

- 用户不写拷贝构造函数时，编译器会提供 [[缺省拷贝构造函数]]。
- 缺省拷贝构造函数用于根据已有对象创建新对象。
- 缺省拷贝构造函数通常是公有的。
- 缺省拷贝构造函数一般执行 [[浅拷贝]]。
- 浅拷贝会复制指针值，而不是复制指针指向的对象。
- 类中有指针成员或资源管理职责时，缺省拷贝构造函数可能不够。

## 本节速记

> 编译器能复制成员值，  
> 但不一定懂资源所有权。
