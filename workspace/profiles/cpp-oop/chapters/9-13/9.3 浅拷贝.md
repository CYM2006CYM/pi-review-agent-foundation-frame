# 9.3 浅拷贝

## 本节核心

[[浅拷贝]] 是缺省拷贝构造函数常见的拷贝方式。

它拷贝的是对象自身存储区中的成员值，不会自动复制指针指向的堆区资源。

当类中有指针成员并且析构函数会释放指针指向资源时，浅拷贝很容易导致 [[重复释放]]。

> [!warning] 易错点
> 两个对象中的指针值相同，表示它们指向同一块堆区资源；析构时如果两个对象都 `delete` 这块资源，就会出现重复释放。

## 浅拷贝拷贝哪些内容

浅拷贝针对的是当前对象的存储区域。

它拷贝的是对象中的 [[实例变量]]，不包括 [[类变量]]。

原因是：

- 实例变量存放在对象内部。
- 类变量属于类，不属于某个对象的存储空间。
- 拷贝一个对象时，只需要复制这个对象自身的状态。

## 不同成员的拷贝方式

对于普通成员：

```cpp
int value_;
int* p_;
```

浅拷贝通常直接复制其存储内容：

- `int` 成员复制数值。
- 指针成员复制地址值。

对于对象类型成员：

```cpp
A member_;
```

不会简单地按字节复制整个成员对象，而是调用该成员对象所属类的拷贝构造函数。

因此，“按位拷贝”这个说法需要加引号理解：它描述的是普通成员的直接复制效果，但对象成员仍会按对象语义调用拷贝构造函数。

## 指针成员导致的问题

考虑一个类 `B`：

```cpp
class B {
public:
    B() {
        pa_ = new A;
    }

    ~B() {
        delete pa_;
    }

private:
    A* pa_;
};
```

创建对象：

```cpp
B b1;
B b2(b1);
```

如果 `B` 没有自定义拷贝构造函数，`b2` 会使用缺省拷贝构造函数。

浅拷贝会让：

```cpp
b2.pa_ == b1.pa_
```

也就是说，两个 `B` 对象的 `pa_` 指向同一个堆区 `A` 对象。

## 重复释放的过程

当作用域结束时，栈区对象按逆序析构。

```cpp
{
    B b1;
    B b2(b1);
}
```

析构顺序通常是：

1. 析构 `b2`
2. 析构 `b1`

`b2` 析构时：

```cpp
delete pa_;
```

它释放了那块堆区 `A` 对象。

随后 `b1` 析构时，也执行：

```cpp
delete pa_;
```

但此时这块堆区内存已经被释放，再次释放就会造成运行时错误。

这就是浅拷贝在资源管理类中最典型的问题。

## 为什么需要自定义拷贝构造函数

浅拷贝的问题不是“拷贝语法错误”，而是“拷贝结果不符合资源所有权语义”。

如果每个 `B` 对象都应该独立拥有自己的 `A` 对象，那么拷贝时就应创建新的堆区对象：

```cpp
B::B(const B& other) {
    pa_ = new A(*other.pa_);
}
```

这属于 [[深拷贝]]。

## 本节考点整理

- [[浅拷贝]] 拷贝对象自身存储区中的成员值。
- 对象拷贝不包括 [[静态数据成员]]。
- 指针成员的浅拷贝只复制地址值。
- 浅拷贝后，两个指针可能指向同一块堆区资源。
- 如果两个对象析构时都释放同一块资源，会产生 [[重复释放]]。
- 对象类型成员会调用其所属类的拷贝构造函数。
- 管理堆区资源的类通常需要自定义 [[拷贝构造函数]]。
- 需要独立资源副本时，应实现 [[深拷贝]]。

## 本节速记

> 浅拷贝复制指针值，  
> 不复制指针指向的资源；  
> 指针共享加析构释放，最容易重复 delete。
