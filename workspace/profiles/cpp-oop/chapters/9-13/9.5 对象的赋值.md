# 9.5 对象的赋值

## 本节核心

[[对象赋值]] 与 [[对象拷贝]] 不同。

对象拷贝是根据已有对象创建一个新对象；对象赋值是在两个对象都已经存在的情况下，把一个对象的状态赋给另一个对象。

```cpp
A a1;
A a2(a1); // 拷贝构造：a2 原来不存在

A a3;
a3 = a1;  // 对象赋值：a1 和 a3 原来都存在
```

> [!important] 高频考点
> 拷贝构造解决“新对象如何由旧对象创建”；赋值函数解决“已有对象如何接收另一个已有对象的状态”。

## 判断拷贝还是赋值

判断规则仍然是：看目标对象在语句执行前是否存在。

```cpp
A a1;
A a2(a1);
```

`a2` 在这条语句之前不存在，所以是拷贝构造。

```cpp
A a3;
a3 = a1;
```

`a3` 在赋值语句之前已经存在，所以是对象赋值。

## 缺省赋值函数

如果用户没有自定义 [[赋值函数]]，编译器会提供 [[缺省赋值函数]]。

缺省赋值函数通常是公有的，并执行类似 [[浅赋值]] 的操作。

```cpp
class A {
public:
    int value;
};

A a1;
A a2;
a2 = a1; // 使用缺省赋值函数
```

浅赋值与浅拷贝类似：通常复制成员值，而不是理解资源所有权。

## 引用成员会限制赋值

如果类中有引用成员：

```cpp
class M {
private:
    AA& ra_;
};
```

对象赋值会遇到问题。

引用初始化后不能重新绑定到另一个对象。赋值语义要求把 `m2` 的状态赋给 `m1`，但 `m1.ra_` 不能改绑成 `m2.ra_` 所引用的对象。

因此，含引用成员的类，编译器提供的缺省赋值函数往往不可用或会被删除。

## 浅赋值如何工作

如果类中没有引用成员，缺省赋值函数通常会按成员赋值。

普通成员：

```cpp
int value_;
A* pa_;
```

浅赋值会直接复制成员值：

- `value_` 复制整数值。
- `pa_` 复制地址值。

对象成员：

```cpp
AA member_;
```

会调用对象成员所属类的赋值函数：

```cpp
member_ = other.member_;
```

因此，和浅拷贝一样，“浅”也不是绝对的逐字节处理；对象成员仍会按其类型的赋值规则处理。

## 指针成员的浅赋值问题

考虑一个类：

```cpp
class B {
public:
    B() {
        pa_ = new AA;
    }

    ~B() {
        delete pa_;
    }

private:
    AA* pa_;
};
```

创建两个对象：

```cpp
B b1;
B b2;
b1 = b2;
```

赋值前：

- `b1.pa_` 指向一块堆区 `AA` 对象。
- `b2.pa_` 指向另一块堆区 `AA` 对象。

执行浅赋值后：

```cpp
b1.pa_ = b2.pa_;
```

结果是：

- `b1.pa_` 和 `b2.pa_` 指向同一块堆区对象。
- `b1` 原来指向的那块堆区对象丢失，没有指针再能访问它。

## 浅赋值的两个后果

第一，产生 [[内存泄漏]]。

`b1` 原来拥有的堆区对象没有被释放，指针却被覆盖掉了。

这块内存还占着，但程序已经找不到它。

第二，产生 [[重复释放]]。

作用域结束时，`b2` 和 `b1` 都会析构。

如果它们的 `pa_` 指向同一块堆区对象：

1. `b2` 析构，`delete pa_`，释放这块资源。
2. `b1` 析构，再次 `delete pa_`，试图释放同一块资源。

这会导致运行时错误。

## 为什么需要自定义赋值函数

浅赋值的问题说明：类一旦自己管理资源，就不能只考虑构造和析构，还要考虑赋值。

一个合理的自定义赋值函数至少要处理：

- 释放当前对象原有资源。
- 根据右侧对象创建或复制新资源。
- 避免自赋值问题。
- 返回当前对象引用，以支持链式赋值。

这些内容会在下一节 [[9.6 自定义赋值函数]] 继续展开。

## 本节考点整理

- [[对象赋值]] 发生在两个对象都已存在时。
- 拷贝构造是创建新对象，赋值是修改已有对象。
- 用户不写赋值函数时，编译器可能提供 [[缺省赋值函数]]。
- 缺省赋值函数通常执行 [[浅赋值]]。
- 含引用成员的类通常不能使用默认赋值语义。
- 指针成员的浅赋值只复制地址值。
- 浅赋值可能导致原资源丢失，形成 [[内存泄漏]]。
- 浅赋值还可能导致两个对象共享同一资源，析构时 [[重复释放]]。
- 管理资源的类通常需要自定义赋值函数。

## 本节速记

> 拷贝构造：新对象诞生；  
> 对象赋值：旧对象改值；  
> 指针浅赋值：先泄漏，再可能重复释放。
