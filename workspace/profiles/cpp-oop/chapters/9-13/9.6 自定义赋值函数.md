# 9.6 自定义赋值函数

## 本节核心

当 [[缺省赋值函数]] 的 [[浅赋值]] 不能满足类的资源管理要求时，就需要编写 [[自定义赋值函数]]。

典型形式如下：

```cpp
class T {
public:
    T& operator=(const T& rhs);
};
```

它的含义是：把右侧对象 `rhs` 的状态赋给左侧对象，也就是当前对象 `*this`。

> [!important] 高频考点
> 标准写法通常是 `T& operator=(const T& rhs);`。考题常问返回类型为什么是 `T&`、参数为什么通常是 `const T&`、函数体为什么要判断自我赋值。

## 函数名：operator=

[[赋值函数]] 本质上是对赋值运算符 `=` 的函数化定义。

在 C++ 中，`operator` 是关键字，`operator=` 整体构成赋值函数的函数名。

```cpp
T& operator=(const T& rhs);
```

这里可以拆成三部分理解：

| 部分 | 含义 |
|---|---|
| `T&` | 返回当前对象的引用 |
| `operator=` | 赋值运算符函数名 |
| `const T& rhs` | 右侧被赋值对象 |

赋值表达式是二元操作：

```cpp
left = right;
```

在成员函数写法中：

- `right` 通过显式形参 `rhs` 表示。
- `left` 通过隐含的 [[this指针]] 表示。
- `*this` 就是当前正在接收赋值的左侧对象。

## 参数为什么通常是 const 引用

赋值的正常语义是“用右侧对象的内容更新左侧对象”，右侧对象本身不应被修改。

因此参数通常写成：

```cpp
const T& rhs
```

这样有两个好处：

- `const` 保证函数内部不能随意修改右侧对象。
- `&` 避免按值传参引发额外拷贝。

如果写成按值传参：

```cpp
T& operator=(T rhs);
```

形参 `rhs` 本身还要由实参拷贝构造而来。对于正在讨论资源管理的类，这通常不是本节需要的基础形式。

## 返回类型为什么是 T&

赋值表达式在 C++ 中也有值。普通内置类型支持链式赋值：

```cpp
int a;
int b;
int c;

a = b = c;
```

执行顺序可以理解为：

```cpp
a = (b = c);
```

`b = c` 完成后，还要把这个赋值表达式的结果继续赋给 `a`。因此，对象赋值也应该保持类似行为。

如果赋值函数返回 `void`：

```cpp
void operator=(const T& rhs);
```

那么 `b = c` 没有可继续使用的结果，链式赋值无法成立。

如果赋值函数返回对象本身的值：

```cpp
T operator=(const T& rhs);
```

虽然某些表达式可能能通过编译，但会产生临时对象，复杂赋值表达式可能修改的是临时对象而不是原对象，语义不自然，也会增加额外拷贝。

所以常用正确写法是：

```cpp
T& operator=(const T& rhs);
```

函数末尾返回当前对象引用：

```cpp
return *this;
```

> [!warning] 易错点
> `this` 是指针，表示当前对象的地址；`*this` 才是当前对象本身。赋值函数要返回当前对象引用，所以返回 `*this`，不是返回 `this`。

## 深赋值的一般思路

对于拥有堆资源的类，赋值函数不能只复制指针地址，而要执行 [[深拷贝]] 语义。

例如一个类内部管理一个字符指针：

```cpp
class B {
public:
    B() {
        pch = new char;
    }

    B(const B& rhs) {
        pch = new char(*rhs.pch);
    }

    ~B() {
        delete pch;
    }

    B& operator=(const B& rhs);

private:
    char* pch;
};
```

自定义赋值函数最直接的写法是：

```cpp
B& B::operator=(const B& rhs) {
    delete pch;
    pch = new char(*rhs.pch);
    return *this;
}
```

这段代码表达了深赋值的基本意图：

1. 先释放当前对象原来拥有的资源。
2. 再根据右侧对象重新申请资源。
3. 把右侧对象资源中的值复制过来。
4. 返回当前对象引用。

但是这个版本还不完整，因为它没有处理 [[自我赋值]]。

## 自我赋值问题

自我赋值指一个对象赋值给自己：

```cpp
B b1;
b1 = b1;
```

更隐蔽的形式是引用或别名：

```cpp
B b1;
B& b2 = b1;
b1 = b2;
```

从变量名看是 `b1 = b2`，但 `b1` 和 `b2` 实际上是同一个对象。

如果赋值函数先执行：

```cpp
delete pch;
```

而右侧 `rhs` 恰好就是当前对象，那么 `rhs.pch` 也同时变成了已经释放的指针。接着再执行：

```cpp
pch = new char(*rhs.pch);
```

就会读取已经释放的内存，产生未定义行为。

> [!warning] 易错点
> 判断自我赋值不能只看变量名是否相同，因为引用可能让不同名字代表同一个对象。可靠方法是比较对象地址。

## 正确处理自我赋值

自我赋值判断应比较当前对象地址和右侧对象地址：

```cpp
B& B::operator=(const B& rhs) {
    if (this != &rhs) {
        delete pch;
        pch = new char(*rhs.pch);
    }

    return *this;
}
```

含义是：

- `this` 是当前对象地址。
- `&rhs` 是右侧对象地址。
- 地址不同，说明确实是两个对象，才释放旧资源并复制新资源。
- 地址相同，说明是自我赋值，直接返回当前对象即可。

这样既避免了浅赋值的 [[内存泄漏]] 和 [[重复释放]]，也避免了自我赋值时先释放后读取的问题。

## 与 swap 写法的关系

课程中强调的基础写法是“先判断自我赋值，再释放旧资源并深拷贝”。

在更现代的 C++ 写法中，也常用 [[swap函数]] 辅助实现赋值逻辑，把资源交换和异常安全封装起来。但对本节而言，首先要掌握的是：

- 为什么缺省赋值函数不够。
- `operator=` 的函数声明格式。
- 为什么返回 `T&`。
- 为什么返回 `*this`。
- 为什么要判断 `this != &rhs`。

## 本节考点整理

- 自定义赋值函数用于替代不合适的 [[缺省赋值函数]]。
- 常见声明为 `T& operator=(const T& rhs);`。
- `operator=` 是赋值运算符函数名。
- 右侧对象通常用 `const T& rhs` 表示。
- 左侧对象是当前对象，由 [[this指针]] 隐含表示。
- 返回 `T&` 是为了让对象赋值像内置类型一样支持 [[链式赋值]]。
- 函数体末尾通常写 `return *this;`。
- 管理堆资源的类需要深赋值，不能只复制指针地址。
- 自我赋值要用 `this != &rhs` 判断。
- 自我赋值判断可以避免释放当前资源后又从同一资源读取。
