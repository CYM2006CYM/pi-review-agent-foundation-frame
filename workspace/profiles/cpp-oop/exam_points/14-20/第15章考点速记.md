# 第15章考点速记

## 一页总览

| 主题 | 必背结论 |
|---|---|
| [[黑盒复用]] | 通过公开接口使用功能，常见于依赖、关联、聚合、组合 |
| [[白盒复用]] | 通过继承吸收基类成员，C++ 中主要由继承实现 |
| [[继承列表]] | `class Derived : public Base { ... };`，省略方式时 `class` 默认 private |
| 派生类成员 | 特殊函数、自定义成员、继承来的基类成员 |
| [[派生类访问控制]] | public/protected/private 继承改变基类 public/protected 成员权限，private 成员不可直接访问 |
| 构造析构顺序 | 构造先基类后派生类；析构先派生类后基类 |
| 拷贝和赋值 | 拷贝构造用 `A(other)`；赋值函数用 `A::operator=(other)` |
| [[同名隐藏]] | 派生类同名函数会隐藏基类所有同名函数 |
| 继承含义 | public 表示 is-a；private/protected 更接近实现复用 |
| 继承 vs 组合 | 能说“是一种”才 public 继承；“有一个/用一个”优先组合 |

## 访问控制表

| 基类成员原权限 | public 继承 | protected 继承 | private 继承 |
|---|---|---|---|
| `public` | `public` | `protected` | `private` |
| `protected` | `protected` | `protected` | `private` |
| `private` | 不可直接访问 | 不可直接访问 | 不可直接访问 |

> [!warning] 易错点
> 基类 `private` 成员不是没有进入派生类对象，而是派生类代码不能直接访问。基类自己的成员函数仍然能访问它。

## 高频代码模板

派生类构造函数：

```cpp
class Derived : public Base {
public:
    Derived(int n1, int n2) : Base(n1), value(n2) {
    }
private:
    int value;
};
```

派生类拷贝构造：

```cpp
Derived(const Derived& other)
    : Base(other), value(other.value) {
}
```

派生类赋值函数：

```cpp
Derived& operator=(const Derived& other) {
    if (this != &other) {
        Base::operator=(other);
        value = other.value;
    }
    return *this;
}
```

同名隐藏与 using：

```cpp
class Derived : public Base {
public:
    using Base::f;
    void f(double);
};
```

## 高频辨析

| 易混点 | 正确理解 |
|---|---|
| 黑盒复用 vs 白盒复用 | 黑盒看接口；白盒看内部并继承成员 |
| 基类/派生类 vs 父类/子类 | 前者是语法术语；后者更适合 public 继承语义 |
| redefine vs override | redefine 是普通函数重定义；override 是虚函数重写 |
| overload vs hide | overload 是重载；hide 是派生类同名函数隐藏基类同名函数 |
| private 继承 vs 组合 | private 继承常可改成组合 |
| is-a vs has-a | is-a 用 public 继承；has-a 优先组合 |
