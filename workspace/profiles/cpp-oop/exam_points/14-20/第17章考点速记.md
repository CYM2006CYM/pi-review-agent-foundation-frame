# 第17章考点速记

## 一页总览

| 主题 | 必背结论 |
|---|---|
| [[多重继承]] | 一个派生类有多个直接基类 |
| 构造顺序 | 按继承列表顺序构造基类，再构造派生类 |
| 析构顺序 | 与构造顺序相反 |
| [[多重继承名字冲突]] | 同名成员导致二义性 |
| 作用域限定 | `A::f()` 指定成员来源 |
| `using` | 把某个基类名字引入派生类作用域 |
| [[菱形结构]] | 共同基类经两条路径进入最终派生类 |
| [[虚基类]] | 让共同基类在最终派生类中只保留一份 |
| 虚基类代价 | 对象布局复杂，类型转换可能需要指针调整 |
| [[接口类]] | 无实例数据，主要描述公共行为契约 |
| [[接口继承]] | 通过多个接口表达多种能力，避免多份状态数据 |

## 高频代码

多重继承：

```cpp
class D : public B1, public B2 {
};
```

名字冲突：

```cpp
class C : public A, public B {
public:
    void k() {
        A::f();
        B::number = 10;
    }
};
```

using 解决默认名字选择：

```cpp
class C : public A, public B {
public:
    using A::f;
    using B::number;
};
```

菱形结构与虚基类：

```cpp
class A {
};

class B : virtual public A {
};

class C : virtual public A {
};

class D : public B, public C {
};
```

接口类：

```cpp
class IComputer {
public:
    virtual ~IComputer() = default;
    virtual void runApp() = 0;
};
```

## 高频辨析

| 易混点 | 正确理解 |
|---|---|
| 多重继承 vs 单继承 | 多重继承有多个直接基类 |
| 初始化列表顺序 vs 继承列表顺序 | 基类构造看继承列表，不看初始化列表书写顺序 |
| 函数冲突 vs 数据冲突 | 数据冲突更麻烦，因为数据占对象空间 |
| `using` vs 真正消除重复数据 | `using` 只解决访问选择，不删除对象里的重复成员 |
| 菱形结构 vs 普通重名 | 菱形是共同基类从两条路径重复进入 |
| 虚基类 vs 普通基类 | 虚基类在最终派生类中共享一份 |
| 工具类 vs 接口类 | 工具类多为静态函数集合；接口类定义公共行为契约 |
