# 第18章考点速记

## 一页总览

| 主题 | 必背结论 |
|---|---|
| [[静态编联]] | 编译期确定调用目标 |
| [[动态编联]] | 运行期根据对象动态类型确定调用目标 |
| [[虚函数]] | 用 `virtual` 声明，支持动态编联 |
| [[虚函数表]] | 保存虚函数入口地址的函数指针数组 |
| [[虚函数表指针]] | 对象中指向虚表的隐藏指针 |
| [[静态类型]] | 编译器看到的表达式类型 |

## 高频代码

虚函数与重写：

```cpp
class Shape {
public:
    virtual ~Shape() = default;
    virtual double area() = 0;
};

class Rectangle : public Shape {
public:
    double area() override;
};
```

虚调用判断：

```cpp
Shape* p = new Rectangle;
p->area();   // 静态类型 Shape* 中有虚函数 area，运行期调用 Rectangle::area
delete p;    // Shape 析构为虚，先析构 Rectangle，再析构 Shape
```

构造函数中的虚调用：

```cpp
class A {
public:
    A() { f(); }
    virtual void f();
};
```

`A::A()` 中的 `f()` 调用 `A::f()`，不会调用派生类版本。

纯虚函数：

```cpp
class IArea {
public:
    virtual ~IArea() = default;
    virtual double area() = 0;
};
```

RTTI：

```cpp
A* p = new B;

typeid(p);    // A*
typeid(*p);   // B，前提是 A 为多态类型

B* pb = dynamic_cast<B*>(p);  // 成功返回 B*，失败返回 nullptr
```

## 高频辨析

| 易混点 | 正确理解 |
|---|---|
| 虚函数调用是否跳过编译检查 | 不跳过，先在静态类型中找函数 |
| 静态类型 vs 动态类型 | 静态类型编译期确定，动态类型运行期确定 |
| 非虚函数重定义 vs 虚函数重写 | 非虚函数按静态类型绑定；虚函数可动态绑定 |
| 指针/引用向上转换 vs 对象向上转换 | 指针/引用保留动态类型；对象转换会[[对象切片]] |
| 构造函数能否 virtual | 不能 |
| 析构函数能否 virtual | 可以，多态基类通常应当 virtual |
| 纯虚函数有定义 | 仍然是纯虚函数，所在类仍抽象 |
| `typeid(p)` vs `typeid(*p)` | 前者看指针，后者看对象 |
| `dynamic_cast` 指针失败 | 返回 `nullptr` |
| `dynamic_cast` 引用失败 | 抛异常 |
