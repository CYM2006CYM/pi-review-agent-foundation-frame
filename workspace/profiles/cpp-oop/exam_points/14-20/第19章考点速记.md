# 第19章考点速记

## 一页总览

| 主题 | 必背结论 |
|---|---|
| [[多态]] | 同一消息请求执行不同代码体 |
| [[静态多态]] | 编译期决定目标，典型是重载和模板 |
| [[动态多态]] | 运行期根据目标对象动态类型决定目标 |
| C++ 虚机制 | 目标对象看动态类型，参数匹配看静态类型 |
| [[子类型化]] | 把变化封装到不同派生类中 |
| 虚机制价值 | 接口不变，实现可变 |
| [[模板方法]] | 父类定流程，子类定步骤 |
| [[虚拟拷贝构造函数]] | 用 `clone()` 模拟按动态类型复制 |
| 虚机制不足 | 不适合多个独立变化方向 |
| [[子类爆炸]] | 多个变化方向组合导致子类数迅速增长 |
| 解决方向 | 用关联/依赖拆分变化方向 |

## 高频代码

静态多态：

```cpp
void f(int);
void f(char*);
```

动态多态：

```cpp
Base* p = new Derived;
p->func();
```

模板方法：

```cpp
class Software {
public:
    void develop() {
        design();
        code();
        test();
        maintain();
    }
protected:
    virtual void design() = 0;
    virtual void code() = 0;
    virtual void test() = 0;
    virtual void maintain() = 0;
};
```

虚拟拷贝构造函数：

```cpp
class Base {
public:
    virtual ~Base() = default;
    virtual Base* clone() const {
        return new Base(*this);
    }
};

class Derived : public Base {
public:
    Derived* clone() const override {
        return new Derived(*this);
    }
};
```

## 高频辨析

| 易混点 | 正确理解 |
|---|---|
| 多态 vs 虚机制 | 虚机制只是动态多态的一种实现 |
| 静态多态 vs 动态多态 | 前者编译期决定，后者运行期参与 |
| 目标对象 vs 参数 | C++ 虚机制看目标对象动态类型，参数重载看静态类型 |
| 拷贝构造函数能否 virtual | 不能 |
| `clone()` 为什么可行 | 它是普通虚函数，内部调用本类拷贝构造函数 |
| 虚机制是否越多越好 | 不是，多方向变化会导致子类爆炸 |
| 多方向变化怎么处理 | 拆成多个水平协作对象，每个方向一棵小继承树 |
