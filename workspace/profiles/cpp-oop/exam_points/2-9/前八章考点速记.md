## 1. 工程与编译

- C++ 项目通常由 [[头文件]]、[[实现文件]]、资源文件和构建配置组成。
- 头文件主要放声明，实现文件主要放定义。
- `#include` 是预处理阶段的文本包含。
- [[包含警戒]] 用于避免同一个头文件被重复包含。
- [[声明]] 告诉编译器“有这个名字”，[[定义]] 提供实体或存储。
- [[单一定义规则]] 要求同一实体在程序中不能有冲突定义。

## 2. 类型、变量、作用域

- [[抽象数据类型]] 关注“能做什么”，不暴露内部表示。
- C++ 内置类型、枚举、类类型都可以看作类型系统的一部分。
- [[作用域]] 决定名字在哪里可见，[[生命周期]] 决定对象活多久。
- 静态局部变量局部可见，但生命周期持续到程序结束。
- 表达式通常包含操作数、运算符、类型转换和求值结果。

## 3. 指针、引用、const

- [[指针]] 保存地址，可以改变指向，也可以为空。
- [[引用]] 是已有对象的别名，初始化后通常不能改绑到其他对象。
- `const int* p`：不能通过 `p` 改值。
- `int* const p`：`p` 本身不能改指向。
- `const int* const p`：值和指向都不能改。
- [[const和引用]] 常用于函数参数，避免复制并保证不修改实参。

## 4. 函数

- 函数声明说明接口，函数定义提供实现。
- [[函数重载]] 依赖参数列表区分，不能只靠返回类型区分。
- [[默认参数]] 应避免与重载组合出二义性。
- 按值返回通常安全，但可能有复制成本。
- 返回指针或引用要保证被返回对象仍然活着。
- 返回局部变量引用会形成 [[悬空引用]]。

## 5. 类和对象

- [[类]] 是类型，[[对象]] 是类的实例。
- [[数据成员]] 描述对象状态，[[成员函数]] 描述对象行为。
- 对象大小通常由非静态数据成员、对齐、虚函数表指针等因素决定。
- [[静态数据成员]] 属于类，不计入单个对象大小。
- 空类对象也需要有非零大小，以保证不同对象有不同地址。

## 6. 成员函数与 this

- 普通成员函数有隐含的 [[this指针]]。
- `obj.f()` 可以理解为把 `obj` 作为当前对象传入。
- 成员函数可在类内实现，也可在类外用 [[作用域解析符]] 实现。
- 类内短函数通常形成 [[内联实现]]。
- 类外实现有利于分离接口和实现。

## 7. 访问控制与封装

- `public` 是外部接口，`private` 是内部实现，`protected` 主要用于继承。
- [[封装]] 是把数据和行为组织在类中。
- [[信息隐蔽]] 是隐藏内部实现细节，只暴露稳定接口。
- 好的类设计应让外部“针对接口编程”，减少对内部表示的依赖。

## 8. const 成员函数

- [[常成员函数]] 在参数列表后写 `const`。
- 常成员函数中的 `const` 修饰隐含的 `this`。
- 普通成员函数的 `this` 类似 `T* const`。
- 常成员函数的 `this` 类似 `const T* const`。
- [[常对象]] 只能调用常成员函数。
- 只读取对象状态的成员函数应主动写成常成员函数。

## 9. 实例成员与类成员

- [[实例变量]] 每个对象各有一份。
- [[实例常量]] 每个对象各有一份，但创建后不应变化。
- [[实例方法]] 是普通成员函数，有 `this`。
- [[类变量]] 归整个类共享，C++ 中通常是静态数据成员。
- [[类方法]] 归整个类，C++ 中通常是静态成员函数。
- [[静态成员函数]] 没有 `this`，不能直接访问实例变量或实例方法。

## 10. 构造函数

- [[构造函数]] 与类名同名，没有返回值。
- 构造函数用于对象创建时初始化对象。
- 构造函数可以重载，也可以有默认参数。
- 单参数构造函数可能参与隐式类型转换。
- [[explicit关键字]] 可禁止这种隐式构造转换。
- [[私有构造函数]] 可限制类外直接创建对象。
- 私有构造函数配合静态方法和静态局部对象，可形成 [[单例模式]]。

## 11. 缺省构造函数与对象初始化

- 用户没有声明任何构造函数时，编译器才提供 [[缺省构造函数]]。
- 只要用户声明了任何构造函数，编译器就不再自动补无参构造。
- 构造函数体执行时，对象已经创建完成。
- 构造函数体内 `x = value` 更准确说是赋值，不是真正初始化。
- [[实例常量]] 不能靠构造函数体内赋值。
- [[类内初始值]] 可为非静态数据成员提供默认初值。
- 普通静态数据成员通常需要类外定义。

## 12. 析构函数与生命周期

- [[析构函数]] 名为 `~类名()`，无参、无返回值、不能重载。
- 析构函数不能是 `const`。
- 用户不写析构函数时，编译器提供 [[缺省析构函数]]。
- 对象成员会在外层对象析构时自动析构。
- [[栈区]] 对象离开作用域自动析构。
- [[堆区]] 对象需要 `delete` 才会析构并释放空间。
- 成员按声明顺序构造，按逆序析构。

## 13. 最容易混淆的点

| 易混点 | 正确理解 |
|---|---|
| 构造函数体内赋值就是初始化 | 进入函数体时对象已创建，严格说是赋值 |
| 静态成员在每个对象里都有一份 | 静态成员属于类，不属于单个对象 |
| 常成员函数不能改类里任何数据 | 它限制的是当前对象的实例状态 |
| 静态成员函数能直接访问普通成员 | 不能，因为没有 `this` |
| 用户写了有参构造后还能自动无参构造 | 不能，编译器不再提供缺省构造 |
| `delete p` 只释放内存 | 它先调用析构函数，再释放堆区空间 |
