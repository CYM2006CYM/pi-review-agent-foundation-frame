# 第9章考点速记

## 一句话主线

第 9 章解决的是：当对象内部有指针、堆内存等资源时，不能只让编译器按成员浅复制，必须自己设计 [[拷贝构造函数]] 和 [[赋值函数]]。

## 拷贝构造 vs 对象赋值

| 对比点 | 拷贝构造 | 对象赋值 |
|---|---|---|
| 发生时机 | 用已有对象创建新对象 | 两个对象都已存在 |
| 典型语句 | `A a2(a1);` | `a2 = a1;` |
| 对应函数 | `A(const A& other);` | `A& operator=(const A& rhs);` |
| 重点问题 | 新对象资源怎么来 | 旧资源如何处理，再如何接收新状态 |

## 高频判断

```cpp
A a1;
A a2(a1); // 拷贝构造

A a3;
a3 = a1;  // 对象赋值
```

判断口诀：目标对象原来不存在，就是拷贝构造；目标对象原来已经存在，就是赋值。

## 浅拷贝和浅赋值

[[浅拷贝]]、[[浅赋值]] 的核心问题是只复制指针值。

```cpp
pa = other.pa;
```

后果：

- 两个对象指向同一块堆内存。
- 原对象或左侧对象原来拥有的资源可能丢失，产生 [[内存泄漏]]。
- 析构时两个对象可能释放同一块资源，产生 [[重复释放]]。

## 深拷贝和深赋值

深拷贝不是复制地址，而是重新申请资源并复制资源内容。

```cpp
pch = new char(*rhs.pch);
```

资源类对象通常至少要考虑：

- 析构函数释放资源。
- 拷贝构造函数复制资源。
- 赋值函数释放旧资源并复制新资源。

## 自定义拷贝构造函数

常见形式：

```cpp
B::B(const B& rhs) {
    pch = new char(*rhs.pch);
}
```

考点：

- 参数一般是 `const B&`。
- 如果写成按值传参，会递归触发拷贝构造。
- 指针成员不能只复制地址。

## 自定义赋值函数

常见形式：

```cpp
B& B::operator=(const B& rhs) {
    if (this != &rhs) {
        delete pch;
        pch = new char(*rhs.pch);
    }

    return *this;
}
```

必须能解释：

- `operator=` 是赋值函数名。
- `rhs` 表示右侧对象。
- `this` 表示左侧当前对象地址。
- 返回 `B&` 是为了支持 [[链式赋值]]。
- 返回 `*this` 是返回当前对象本身。
- `this != &rhs` 用来处理 [[自我赋值]]。

## 易错点

> [!warning] 易错点
> 拷贝构造和赋值函数不是一回事。`A a2 = a1;` 在定义新对象时仍然是拷贝构造，不是赋值。

> [!warning] 易错点
> 自我赋值不能只看变量名。`B& b2 = b1; b1 = b2;` 中，`b1` 和 `b2` 是同一个对象。

> [!warning] 易错点
> `this` 是地址，`*this` 是对象本身。赋值函数返回 `*this`，不是返回 `this`。

## 典型简答题模板

题目：为什么有指针成员的类通常需要自定义拷贝构造函数和赋值函数？

答题要点：

1. 编译器提供的缺省拷贝构造函数和缺省赋值函数通常执行浅复制。
2. 浅复制只复制指针地址，导致多个对象共享同一块堆资源。
3. 这会造成内存泄漏、重复释放、对象之间意外互相影响等问题。
4. 自定义拷贝构造函数应重新申请资源并复制内容。
5. 自定义赋值函数还要先处理当前对象已有资源，并判断自我赋值。
