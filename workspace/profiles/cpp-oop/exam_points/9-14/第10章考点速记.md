# 第10章考点速记

## 一句话主线

[[运算符重载]] 的目标是让自定义类型像内置类型一样自然地使用已有运算符，但不能改变语法规则，也不能破坏运算符原本的直觉语义。

## 基本限制

- 不能创造新运算符。
- 不能改变操作数个数。
- 不能改变优先级和结合性。
- 至少一个操作数必须是自定义类型。
- 不能重载 `.`、`.*`、`::`、`?:`。

## 二元运算符两种形式

自由函数形式：

```cpp
A operator+(const A& lhs, const A& rhs);
```

成员函数形式：

```cpp
A operator+(const A& rhs) const;
```

对应关系：

```cpp
a + b
operator+(a, b)    // 自由函数
a.operator+(b)     // 成员函数
```

## 返回类型口诀

| 运算 | 是否修改左侧对象 | 常见返回 |
|---|---|---|
| `+`、`-` | 不修改 | 按值返回新对象 |
| `+=`、`-=` | 修改 | 返回 `A&` |
| `=` | 修改 | 返回 `A&` |
| 前置 `++` | 修改 | 返回 `A&` |
| 后置 `++` | 修改但返回旧值 | 返回 `A` |
| `[]` 非 const | 可能修改元素 | 返回元素引用 |
| `<<`、`>>` | 修改流状态 | 返回流引用 |

## 成对重载

常见成对关系：

```cpp
+   +=
-   -=
*   *=
/   /=
```

建议先实现复合赋值，再复用它实现普通运算。

```cpp
A operator+(A lhs, const A& rhs) {
    lhs += rhs;
    return lhs;
}
```

## 前置 ++ 与后置 ++

前置：

```cpp
A& operator++();
```

含义：先加，再返回修改后的当前对象。

后置：

```cpp
A operator++(int);
```

含义：先保存旧值，再加，返回旧值。`int` 只用于区分前置和后置。

## 特殊运算符

`[]`：

```cpp
int operator[](int index) const;
int& operator[](int index);
```

非 const 版本返回引用，支持 `a[i] = value`。

`()`：

```cpp
int operator()(int a, int b) const;
```

让对象成为 [[函数对象]]。

`->`：

```cpp
A* operator->();
```

返回结果必须能继续进行 `->` 操作。

`<<`、`>>`：

```cpp
ostream& operator<<(ostream& os, const A& obj);
istream& operator>>(istream& is, A& obj);
```

通常写成非成员函数，因为左操作数是流对象。

## 成员还是非成员

| 运算符 | 建议或要求 |
|---|---|
| 一元运算符 | 成员函数 |
| `=`、`[]`、`()`、`->` | 必须是成员函数 |
| `+=`、`-=` 等复合赋值 | 成员函数 |
| 普通二元运算符 | 非成员函数常见 |

## 易错点

> [!warning] 易错点
> `operator++(int)` 里的 `int` 不是业务参数，只是区分后置 `++`。

> [!warning] 易错点
> 普通 `operator+` 返回局部对象引用是错误的；它通常按值返回新对象。

> [!warning] 易错点
> `&&`、`||`、`,` 一般不建议重载，因为容易破坏内置类型的短路求值或求值习惯。
