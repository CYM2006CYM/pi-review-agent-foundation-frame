# 第11章考点速记

## 一句话主线

第 11 章围绕[[动态内存管理]]展开：程序的不同[[存储空间]]有不同生命周期，`new/delete` 管理单对象和数组，`operator new/operator delete` 负责底层分配释放，[[智能指针]]通过对象生命周期自动管理资源，[[共享指针]]通过[[引用计数]]支持共享所有权。

## 存储空间

| 区域 | 典型内容 | 生命周期特点 |
|---|---|---|
| [[代码区]] | 程序指令 | 程序运行期间存在 |
| [[数据区]] | 全局变量、静态变量 | 程序运行期间存在 |
| [[常量数据区]] | 字符串字面量、常量数据 | 通常不可修改 |
| [[栈区]] | 局部变量、函数调用信息 | 自动分配、自动释放 |
| [[堆区]] / [[自由存储区]] | 动态对象 | 手动或由资源管理对象控制 |

## `new` 和 `delete`

普通 `new` 做两件事：

1. 调用[[operator new]] 分配原始内存。
2. 在这块内存上调用[[构造函数]]。

普通 `delete` 做三件事：

1. 判断指针是否为空。
2. 调用对象的[[析构函数]]。
3. 调用[[operator delete]] 回收内存。

> [!warning] 易错点
> `delete p;` 不会把 `p` 自动变成 `nullptr`。对象释放后，`p` 仍保存旧地址，继续使用会形成[[悬空指针]]。

## 单对象与数组

| 分配方式 | 释放方式 | 说明 |
|---|---|---|
| `new A` | `delete p` | 构造和析构一个对象 |
| `new A[n]` | `delete[] p` | 构造和析构一组对象 |

`new[]` 必须配 `delete[]`，`new` 必须配 `delete`。混用会导致析构次数错误或未定义行为。

## 重载 `operator new/delete`

重载 `operator new` 和 `operator delete` 改变的是“内存如何分配和释放”，不直接替代构造函数和析构函数。

常见考法：

- 区分 `new` 表达式和 `operator new` 函数。
- 说明为什么构造函数不能直接返回对象地址。
- 说明 `new_handler` 在分配失败时的作用。

## 智能指针

[[智能指针]]的核心是把资源释放放入对象析构函数中，利用[[RAII]]自动释放资源。

```cpp
{
    SmartPtr p(new A);
} // p 离开作用域时自动释放 A
```

它解决的核心问题不是“让指针更像普通变量”，而是让资源释放有明确归属和自动时机。

## 共享指针

[[共享指针]]允许多个智能指针共同管理同一个对象。典型实现依赖[[引用计数]]：

- 构造新共享关系时计数加 1。
- 共享指针析构或退出共享关系时计数减 1。
- 计数减到 0 时释放真实对象和计数对象。

> [!important] 高频考点
> 共享指针的拷贝构造函数、赋值函数、析构函数都要维护引用计数；赋值函数还要处理自我赋值和旧资源释放。

## 其它易错点

- 不要 `delete void*`，因为会丢失真实对象类型，可能无法调用正确析构函数。
- `new` 和 `delete` 相距太远时容易产生[[内存泄漏]]。
- [[写时复制]]用于解决共享对象被修改时相互影响的问题。
- [[定位分配]]是在已有内存上构造对象，常用 placement new，通常需要手动调用析构函数。

1. `new A` 的完整执行过程是什么？
2. `delete p` 的完整执行过程是什么？
3. 为什么 `new[]` 要配 `delete[]`？
4. `operator new` 和构造函数的分工是什么？
5. 智能指针如何体现 [[RAII]]？
6. 共享指针的引用计数何时加、何时减、何时释放对象？

8. placement new 为什么不能简单等同于普通 new？
