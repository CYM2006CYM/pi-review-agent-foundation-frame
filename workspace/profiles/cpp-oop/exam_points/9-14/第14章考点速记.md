# 第14章考点速记

## 一页总览

| 主题 | 必背结论 |
|---|---|
| 面向对象三大特征 | [[封装]]和[[信息隐蔽]]、[[继承]]、[[多态]] |
| 抽象与表示 | 先抽象类型、行为、数据，再用 C++ 类表示 |
| 信息隐蔽 | 只公开必要接口，隐藏内部数据和实现行为 |
| 分离使用和实现 | 使用者依赖接口，实现细节可独立变化 |
| 多个类设计 | 考虑拆分、合并、数据关系、行为关系 |
| 单个类设计 | 考虑行为可见性、数据访问、参数抽象、关系选择、实现/数据抽象 |
| 类方法 | 适合表达类整体行为、创建方法、单例 |
| 分页器例 | 行为和数据先抽象，再决定构造、拷贝、赋值、显示等成员 |

## 高频辨析

| 易混点 | 正确理解 |
|---|---|
| 封装 vs 信息隐蔽 | 封装是抽象并表示成类；信息隐蔽是隐藏外部不需要知道的细节 |
| 抽象 vs 表示 | 抽象不依赖语言；表示才涉及 C++ 类、成员函数、数据成员 |
| public vs private | public 是对外接口；private 是内部实现 |
| 依赖 vs 关联 | 参数形式通常是依赖；数据成员形式通常是关联 |
| 类拆分 vs 类合并 | 职责过多要拆；拆得过细且互相访问频繁要合 |
| 构造函数重载 vs 创建方法 | 构造函数名固定，创建方法可用函数名表达语义 |
| 静态局部对象单例 vs 静态指针单例 | 前者简单；后者灵活但要释放和置空 |

## 代码判断

```cpp
class Ball {
public:
    void stepMove();

private:
    bool hitBorder() const;
    void rebound();
};
```

`stepMove` 是公开接口；`hitBorder`、`rebound` 是内部实现，体现信息隐蔽。

```cpp
class Dog {
public:
    void bark(Dest& dest);
};
```

`Dog` 和 `Dest` 通常是依赖。

```cpp
class Dog {
private:
    Dest* dest;
};
```

`Dog` 和 `Dest` 通常是关联。

```cpp
class Balloon {
public:
    static Balloon createColorBalloon(int color);

private:
    explicit Balloon(int color);
};
```

这是用类方法作为创建方法，让创建语义更清楚。
