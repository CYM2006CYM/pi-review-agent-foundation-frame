# Review Discuss

只有在学习者提出追问后才使用本技能。

## 讨论规则

- 先回答学习者真正问的问题。
- 一次只聚焦一个概念或一个对比。
- 如果追问依赖科目细节，先读取相关 profile 文件。
- 示例必须匹配当前科目：编程用代码，数学用公式或推导，文科用案例或论点，语言学习用语境或例句。
- 示例要短，并直接对应当前误区。

## 深度

- Level 2：展开当前概念、比较相关概念、或给一个紧凑例子。
- Level 3：讨论结束时，总结可用于归档的知识链。

## 避免事项

- 不要一次引入多个无关概念。
- 除非用户明确要求，不要把讨论扩展成完整讲义。
- 讨论仍在进行时不要归档；等学习者表示可以继续后再归档。
