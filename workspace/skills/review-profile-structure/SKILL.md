# Profile Structure

你在 `/review-init` 的步骤 2-3 中被加载。任务：生成 `subject.md` 和 `chapters/` 下所有小节文件。

## subject.md

不要写空话。每个字段都要给后续出题 AI 提供实际指引。

```markdown
# {科目名称}

## 科目概述
{什么课、考什么、需要什么能力。2-3 句，具体。}

## 考试目标
- {具体可验证的能力。不是"掌握基础知识"，是"能区分 X 和 Y 的触发时机"。}
- {至少 3 条。}

## 出题风格
- 题型偏好：{各题型大致比例}
- 难度分布：{S-R/S-U/M-U/M-A/C-A 比例}
- 出题特点：{重代码/重推导/重辨析/重语境——按实际科目选}
- 代码语言：{cpp/python/java/.../none}

## 资料说明
- 源文件数：{N}
- 覆盖范围：{哪些章节/专题}
- 资料类型：{笔记/教材/考点/试题}
```

## chapters/ 生成规则

### 颗粒度控制

| 源文件情况 | 操作 |
|-----------|------|
| ≤ 300 行，1-3 个知识点 | 一个小节，保留原名 |
| > 500 行 | 按二级标题拆为多个小节 |
| 多个文件讲同一知识点 | 合并为一个小节 |
| 小节 < 20 行 | 合并到相邻小节 |
| 小节 > 300 行 | 检查能否再拆 |

### 文件格式

路径：`chapters/{N}.{n}-{小节名}.md`

```markdown
# {小节标题}

## 核心概念
- **{概念}**：{1-2 句定义 + 为什么重要}

## 知识点详解
{结构化展开。定义 → 原理 → 使用场景 → 注意事项。}

## 关键示例
{代码/推导/案例。代码必须可运行。非编程科目用适合的方式。}

## 常见误区
- ❌ {错误} → ✅ {正确}

## 来源
- {源文件}#{标题或行号}
```

### 不确定的章节归属

不要硬猜。在小节文件末尾加：
```
<!-- UNCERTAIN_STRUCTURE: 本节可能属于第X章，因为{原因} -->
```
同时在质量报告中记录。
